# Convex schema definition (all tables, all fields)

The database schema is the single source of truth for the entire Conflict Coach data model. Every Convex query, mutation, and action derives its type safety from `convex/schema.ts` — if a field is missing or wrong here, runtime errors cascade across the whole stack. This task pins all 11 tables, their fields, their union literals, and their indexes to the normative spec before any feature code is written. Three fields not present in the original tech spec are added per planning assumptions: `cases.inviteeName` (the pre-join display name typed by the initiator), `cases.tokenCount` (accumulated AI token usage for cost tracking), and `partyStates.closureProposalSummary` (the resolution summary text captured at closure proposal time). The `COST_LIMITED` case status is also added to support the AI budget enforcement work in T17.

## Acceptance Criteria

- All 11 tables match tech spec §3.1 field names, types, and union literals exactly
- cases.status union includes COST_LIMITED in addition to the 7 spec values
- cases table includes: inviteeName: v.optional(v.string()), tokenCount: v.optional(v.number())
- partyStates table includes: closureProposalSummary: v.optional(v.string())
- All indexes present
- `npx convex dev` applies the schema without errors
- All 6 key invariants from §3.2 documented as inline comments
- `tsc --noEmit` passes

## Implementation Notes

The file is `convex/schema.ts`. It is the first file in `convex/` and no other Convex function file should be written until this task is complete — all generated types (in `convex/_generated/`) depend on it.

**Table inventory (11 tables):**

1. `users` — managed partly by Convex Auth; one row per authenticated user; `role` union `USER | ADMIN`; indexed `by_email`.
2. `cases` — the core mediation record; `status` is the state machine field (8 values including `COST_LIMITED`); `schemaVersion: v.literal(1)` is intentional for migration support; FK refs use `v.id('tableName')` not `v.string()`; indexed `by_initiator` and `by_invitee`.
3. `partyStates` — one row per party per case; holds per-party form fields, phase completion timestamps, `synthesisText`, closure flags, and the new `closureProposalSummary`; indexed `by_case` and `by_case_and_user`.
4. `privateMessages` — strictly isolated by `userId`; no query must return another user's rows; indexed `by_case_and_user` (primary) and `by_case` (server-side AI context assembly only, never client-exposed).
5. `jointMessages` — visible to both parties; includes `isIntervention` flag for coach behavior; indexed `by_case`.
6. `draftSessions` — per-user within a case; holds `finalDraft` text before send gate; indexed `by_case_and_user`.
7. `draftMessages` — child of `draftSessions`; same status pattern as other message tables; indexed `by_draft_session`.
8. `inviteTokens` — 32-char URL-safe token; `status` has three literals: `ACTIVE`, `CONSUMED`, and `REVOKED`; the `ACTIVE → CONSUMED` transition is atomic within the redemption mutation; `REVOKED` is set by the initiator via case cancellation; indexed `by_token` (primary lookup) and `by_case`.
9. `templates` — admin-managed; `currentVersionId` is an optional ID reference to `templateVersions` (circular ref is valid in Convex); indexed `by_category`.
10. `templateVersions` — immutable once published; content frozen at publish time; indexed `by_template`.
11. `auditLog` — append-only; `metadata: v.optional(v.any())` is the one intentional `v.any()` in the schema; indexed `by_actor`.

**Planning-assumption additions** (not in the original spec, confirmed in `plan-design-enriched.json`):

- `cases.inviteeName: v.optional(v.string())` — the invitee's display name typed by the initiator at case creation ("What's their name?"). Not derivable from auth since the invitee has not yet registered.
- `cases.tokenCount: v.optional(v.number())` — accumulated token count across all AI actions for this case; used by T17 budget enforcement.
- `partyStates.closureProposalSummary: v.optional(v.string())` — the resolution summary text typed by the proposing party; stored on their `partyStates` row, not on `cases`, so it is private until the case resolves.
- `cases.status` union must include `COST_LIMITED` as the 8th literal.

**Inline invariant comments** — add all 6 from tech spec §3.2 as comments directly above the relevant field or table definition:

1. `cases.templateVersionId` is set at creation and never mutated.
2. `privateMessages` rows are never exposed to a user who doesn't match `userId`.
3. `jointMessages` are visible only to users with a `partyStates` row for that case.
4. `inviteTokens.status` transitions `ACTIVE → CONSUMED` atomically within the redemption mutation; `REVOKED` is set by initiator-side case cancellation.
5. `templateVersions` rows are immutable once inserted.
6. `cases.schemaVersion` is always `1` in v1; present for forward migration support.

**What `npx convex dev` does:** on first run it pushes the schema, creates indexes, and generates `convex/_generated/`. The generated directory must be gitignored — add it to `.gitignore` if T1 hasn't already. This task verifies the push succeeds without errors.

**Exports**

**Signatures**
```ts
// convex/schema.ts
import { defineSchema, defineTable } from "convex/server";
import { v } from "convex/values";

export default defineSchema({
  users: defineTable({
    email: v.string(),
    displayName: v.optional(v.string()),
    role: v.union(v.literal("USER"), v.literal("ADMIN")),
    createdAt: v.number(),
  }).index("by_email", ["email"]),

  cases: defineTable({
    schemaVersion: v.literal(1),
    status: v.union(
      v.literal("DRAFT_PRIVATE_COACHING"),
      v.literal("BOTH_PRIVATE_COACHING"),
      v.literal("READY_FOR_JOINT"),
      v.literal("JOINT_ACTIVE"),
      v.literal("CLOSED_RESOLVED"),
      v.literal("CLOSED_UNRESOLVED"),
      v.literal("CLOSED_ABANDONED"),
      v.literal("COST_LIMITED"),       // planning assumption: T17 budget enforcement
    ),
    isSolo: v.boolean(),
    category: v.string(),
    templateVersionId: v.id("templateVersions"),
    initiatorUserId: v.id("users"),
    inviteeUserId: v.optional(v.id("users")),
    inviteeName: v.optional(v.string()),         // planning assumption: pre-join display name
    tokenCount: v.optional(v.number()),          // planning assumption: AI cost tracking
    createdAt: v.number(),
    updatedAt: v.number(),
    closedAt: v.optional(v.number()),
    closureSummary: v.optional(v.string()),
  })
    .index("by_initiator", ["initiatorUserId"])
    .index("by_invitee", ["inviteeUserId"]),

  partyStates: defineTable({
    caseId: v.id("cases"),
    userId: v.id("users"),
    role: v.union(v.literal("INITIATOR"), v.literal("INVITEE")),
    mainTopic: v.optional(v.string()),
    description: v.optional(v.string()),
    desiredOutcome: v.optional(v.string()),
    formCompletedAt: v.optional(v.number()),
    privateCoachingCompletedAt: v.optional(v.number()),
    synthesisText: v.optional(v.string()),
    synthesisGeneratedAt: v.optional(v.number()),
    closureProposed: v.optional(v.boolean()),
    closureConfirmed: v.optional(v.boolean()),
    closureProposalSummary: v.optional(v.string()),  // planning assumption
  })
    .index("by_case", ["caseId"])
    .index("by_case_and_user", ["caseId", "userId"]),

  privateMessages: defineTable({
    caseId: v.id("cases"),
    userId: v.id("users"),
    role: v.union(v.literal("USER"), v.literal("AI")),
    content: v.string(),
    status: v.union(v.literal("STREAMING"), v.literal("COMPLETE"), v.literal("ERROR")),
    tokens: v.optional(v.number()),
    createdAt: v.number(),
  })
    .index("by_case_and_user", ["caseId", "userId"])
    .index("by_case", ["caseId"]),

  jointMessages: defineTable({
    caseId: v.id("cases"),
    authorType: v.union(v.literal("USER"), v.literal("COACH")),
    authorUserId: v.optional(v.id("users")),
    content: v.string(),
    status: v.union(v.literal("STREAMING"), v.literal("COMPLETE"), v.literal("ERROR")),
    isIntervention: v.optional(v.boolean()),
    replyToId: v.optional(v.id("jointMessages")),
    createdAt: v.number(),
  }).index("by_case", ["caseId"]),

  draftSessions: defineTable({
    caseId: v.id("cases"),
    userId: v.id("users"),
    status: v.union(v.literal("ACTIVE"), v.literal("SENT"), v.literal("DISCARDED")),
    createdAt: v.number(),
    completedAt: v.optional(v.number()),
    finalDraft: v.optional(v.string()),
  }).index("by_case_and_user", ["caseId", "userId"]),

  draftMessages: defineTable({
    draftSessionId: v.id("draftSessions"),
    role: v.union(v.literal("USER"), v.literal("AI")),
    content: v.string(),
    status: v.union(v.literal("STREAMING"), v.literal("COMPLETE"), v.literal("ERROR")),
    createdAt: v.number(),
  }).index("by_draft_session", ["draftSessionId"]),

  inviteTokens: defineTable({
    caseId: v.id("cases"),
    token: v.string(),
    status: v.union(v.literal("ACTIVE"), v.literal("CONSUMED"), v.literal("REVOKED")),
    createdAt: v.number(),
    consumedAt: v.optional(v.number()),
    consumedByUserId: v.optional(v.id("users")),
  })
    .index("by_token", ["token"])
    .index("by_case", ["caseId"]),

  templates: defineTable({
    category: v.string(),
    name: v.string(),
    currentVersionId: v.optional(v.id("templateVersions")),
    archivedAt: v.optional(v.number()),
    createdAt: v.number(),
    createdByUserId: v.id("users"),
  }).index("by_category", ["category"]),

  templateVersions: defineTable({
    templateId: v.id("templates"),
    version: v.number(),
    globalGuidance: v.string(),
    coachInstructions: v.optional(v.string()),
    draftCoachInstructions: v.optional(v.string()),
    publishedAt: v.number(),
    publishedByUserId: v.id("users"),
    notes: v.optional(v.string()),
  }).index("by_template", ["templateId"]),

  auditLog: defineTable({
    actorUserId: v.id("users"),
    action: v.string(),
    targetType: v.string(),
    targetId: v.string(),
    metadata: v.optional(v.any()),
    createdAt: v.number(),
  }).index("by_actor", ["actorUserId"]),
});
```

**Queries used**
- None — this file has no runtime logic; it is consumed by `npx convex dev` to push the schema and generate types.

**Invariants**
- Every table that stores user-owned data (`privateMessages`, `partyStates`, `draftSessions`, `draftMessages`) has an index that enables efficient per-user or per-case-and-user lookups without a full table scan.
- `v.any()` is used exactly once: `auditLog.metadata`. Every other field is explicitly typed.
- `v.id('tableName')` is used for all cross-table references; `v.string()` is never used as a surrogate FK.
- `cases.schemaVersion` is `v.literal(1)` — not `v.number()` — enforcing the current version as the only valid value.
- The `COST_LIMITED` status literal is present in the union; its absence would cause T17 to fail at the Convex type level.

**Non-goals**
- No query, mutation, or action code in this task — schema only.
- No seed data (covered in T11).
- No auth configuration (covered in T3).
- No migration tooling — Convex handles schema migrations automatically via `npx convex dev`.
- Do not validate field business rules (e.g., "status must be JOINT_ACTIVE before sending joint messages") — those live in mutation/action code, not in the schema validator.

## Stack Practices

- Use `defineSchema` and `defineTable` from `convex/server`; use `v` from `convex/values` for all field validators.
- ID references use `v.id('tableName')` not `v.string()`.
- Union literals use `v.union(v.literal('A'), v.literal('B'))`.
- Anti-pattern: do NOT add `v.any()` fields except where explicitly needed (auditLog.metadata).
- After editing schema.ts, `npx convex dev` must be running to push the schema.
- The `schemaVersion: v.literal(1)` pattern on `cases` is intentional.
