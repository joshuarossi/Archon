# Joint Chat View — Reactive Messages, Coach Facilitation, Cost & Closure States

The joint chat view is the centerpiece of Conflict Coach: the first screen where both parties communicate directly, mediated in real-time by an AI coach. This view must surface shared messages reactively to both clients simultaneously, render Coach interventions with distinct visual identity, and gracefully handle the COST_LIMITED degradation state (AI unavailable, peer chat still works) and CLOSED_* read-only states. Because messages live in a shared Convex reactive query, both parties' clients update automatically — no polling required.

## Acceptance Criteria

- Reactive messages auto-update both clients
- Party avatar colors correct; ⟡ glyph + coach-accent on Coach messages
- isIntervention messages: warning-tint background
- 'Coach is thinking...' indicator for STREAMING Coach row
- Closure proposal banner with Confirm/Reject buttons
- 'My guidance' Sheet with synthesisText
- COST_LIMITED: banner + hidden Draft Coach + manual messages still work
- CLOSED_*: input and Draft Coach hidden; 'case is closed' banner

## Implementation Notes

`JointChatView.tsx` lives under `src/routes/` (or `src/pages/`) and is rendered by the CaseDetail layout route (T24) as the `/cases/:caseId/joint` sub-route. It receives the resolved `caseData` via `useOutletContext<CaseDetailContext>()` — import `CaseDetailContext` from `src/routes/cases/types.ts`. Do not re-fetch `cases/get` here.

**Real-time messages.** `useQuery(api.jointChat.messages, { caseId })` is the single data source. Convex WebSocket subscriptions mean both parties see the same data within milliseconds. The query already filters by case; no further client-side filtering by party is needed for the message list (all joint messages are visible to both).

**Party identification.** `msg.authorUserId === currentUserId` (from `useQuery(api.users.getCurrentUser)`) distinguishes USER from OTHER messages. Coach messages are identified by `msg.role === 'COACH'` — this is the canonical discriminator. The `authorUserId` is `null` for Coach rows as a consequence but `role` is the source of truth.

**Message shape.** The `api.jointChat.messages` query returns rows with these fields:
```typescript
type JointMessage = {
  _id: Id<'jointMessages'>;
  caseId: Id<'cases'>;
  authorUserId: Id<'users'> | null;  // null for Coach messages
  content: string;
  role: 'USER' | 'COACH';
  status: 'SENT' | 'STREAMING' | 'COMPLETE';
  isIntervention?: boolean;  // only set on COACH rows; true when Haiku classified INFLAMMATORY
  _creationTime: number;
};
```

**Coach message anatomy.** Coach messages carry `isIntervention?: boolean` (set by the Haiku classifier in T14). Standard Coach messages get `--coach-accent` left-border + `--accent-subtle` background + ⟡ glyph. Intervention messages swap `--accent-subtle` for a `--warning` tint background. The `isIntervention` flag comes directly off the message row — no client-side classification.

**@-mention detection** happens inside the `jointChat/sendUserMessage` mutation body (T14). The client sends the raw text as-is; the mutation detects `@coach` and schedules `generateCoachResponse`. Do NOT implement client-side @-mention detection.

**Draft Coach panel.** `DraftCoachPanel` (T28) is a sibling component rendered conditionally. On desktop it is an `absolute/fixed` 420 px right panel. On mobile (`useMediaQuery('(max-width: 768px)')`) it is a shadcn/ui `Sheet side='bottom'`. COST_LIMITED hides the panel entirely (`caseData.case.status === 'COST_LIMITED'`). The `useMediaQuery` hook lives in `src/hooks/useMediaQuery.ts` and is created in this ticket (T27); T28 reuses it.

**Closure proposal banner.** When `caseData.case.closureProposed === true` and `caseData.case.closureProposedBy !== currentUserId`, show the banner above `MessageInput`. The proposer (where `closureProposedBy === currentUserId`) sees a "waiting for confirmation" state instead. The banner provides `Confirm` and `Reject and keep talking` buttons calling `jointChat/confirmClosure` and `jointChat/rejectClosure` respectively (T16 mutations). `closureProposedBy` is a `v.optional(v.id('users'))` field on the `cases` table (T2).

**CLOSED_* state.** When `caseData.case.status` starts with `CLOSED_`, hide `MessageInput` and `DraftCoachPanel`, and render a `'This case is closed'` informational banner. The message list remains visible and scrollable in read-only mode.

**COST_LIMITED state.** Show an informational banner above the message list. Keep `MessageInput` enabled for peer-to-peer messages. Hide `DraftCoachPanel` entirely. `generateCoachResponse` already enforces the hard cap server-side (T17); the client just needs to surface the status.

**MessageInput disabled during STREAMING.** If any message in the list has `status === 'STREAMING'`, disable (but do not hide) `MessageInput`. The input is hidden only when status is `CLOSED_*`.

**'My guidance' Sheet.** A ghost button in the top nav (Sparkles icon, "My guidance") opens a `Sheet side='right'` (480 px desktop) rendering the `text` field from `api.jointChat.mySynthesis` as react-markdown prose. `mySynthesis` returns `{ text: string } | null` — access as `synthesis?.text`. The Sheet uses shadow-3 elevation.

## Stack Practices

- Real-time: useQuery(api.jointChat.messages, { caseId }) — both clients share reactive subscription.
- Party identification: `msg.role === 'COACH'` (canonical discriminator); `msg.authorUserId === currentUserId` for USER vs OTHER.
- `mySynthesis` return: `{ text: string } | null` — read as `synthesis?.text`, NOT `synthesis?.synthesisText`.
- Mobile sheet for Draft Coach: useMediaQuery('(max-width: 768px)') hook from `src/hooks/useMediaQuery.ts` (created in this ticket).
- COST_LIMITED: caseData.case.status === 'COST_LIMITED' → show banner, hide DraftCoachPanel, keep MessageInput enabled.
- STREAMING gate: any message with status === 'STREAMING' → disable MessageInput (do not hide).
- Anti-pattern: do NOT have the client detect @mentions — mutation body handles it.

## Design Practices

- Layout (design doc §4.9): max-width 1080px for joint chat (more breathing room). Desktop: single column message list, Draft Coach as 420px right panel (not part of the message list). Mobile <768px: Draft Coach as bottom Sheet full-screen.
- Top nav (design doc §3.2): '← Back', case name, phase label, 'My guidance' button (ghost, Sparkles icon), 'Close' button (ghost, --text-secondary). No hamburger on desktop.
- Party avatars (design doc §4.9): Alex/Initiator = #BFDBFE soft blue circle with initial; Jordan/Invitee = #FECDD3 soft rose; Coach = --coach-accent (#8B7AB5) at 20% opacity with --coach-accent initial/glyph. 32px circles. Consistent per party throughout the session.
- Coach messages (design doc §4.9): left border 2px --coach-accent, bg --accent-subtle, ⟡ glyph prefix, --coach-accent text. isIntervention messages: swap --accent-subtle for a --warning (#B58B4D) tinted background to visually signal inflammatory-content response.
- 'Coach is thinking...' STREAMING state: inline indicator below last message, --text-secondary 13px italic. STREAMING Coach bubble has blinking cursor, role='status' aria-label='Coach is replying'.
- Closure proposal banner (design doc §4.11): bg-surface, border --border-default, 16px padding, positioned above MessageInput. Icon: 📬 emoji or Lucide Mail. Two shadcn/ui buttons: 'Confirm' (primary --accent) and 'Reject and keep talking' (secondary). Banner must not block the message list — insert it between list and input.
- 'My guidance' Sheet: shadcn/ui Sheet side='right', width 480px on desktop. Header: 'My Guidance' 500 weight + Lock icon. Content: react-markdown synthesis text with prose classes. shadow-3 elevation.
- COST_LIMITED banner: bg-warning (#B58B4D) at 10% opacity, border --warning, --warning text. 'AI Coach is temporarily unavailable. You can still message each other.' Hides DraftCoachPanel component entirely.
- CLOSED_* state: 'case is closed' banner above the message list (not a modal). --bg-surface-subtle background. MessageInput hidden. Draft Coach hidden.
- Accessibility: message list container role='log' aria-live='polite' aria-relevant='additions'. Each message authored by 'Coach' gets aria-label='Coach: [content]' on the bubble element. Closure banner uses role='alert' so both parties receive the notification announcement. 'My guidance' Sheet focus-trapped, Esc closes and returns focus to trigger button.
