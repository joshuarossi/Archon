# CI Pipeline (GitHub Actions: lint, typecheck, unit, E2E)

This task configures the GitHub Actions CI pipeline that enforces all quality gates on every pull request targeting `main`. It is the final integration point of the project: CI runs lint, type-check, and Vitest unit tests as independent parallel jobs, then runs the full Playwright E2E suite against a live Convex dev deployment with `CLAUDE_MOCK=true`. Without this pipeline, none of the quality work in T33‚ÄìT35 is enforceable at merge time. This task exists to ensure the repo's default branch is always in a green, deployable state.

## Acceptance Criteria

- Four jobs on push to PR branches targeting main
- lint, typecheck, unit pass independently
- e2e: deploy to Convex dev; seed; Playwright with CLAUDE_MOCK=true; multi-browser
- node_modules cached

## Implementation Notes

The CI workflow lives at `.github/workflows/ci.yml`. It triggers on `pull_request` events targeting `main`:

```yaml
on:
  pull_request:
    branches: [main]
```

**Concurrency ‚Äî cancel redundant runs on the same PR:**

```yaml
concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true
```

This prevents two push events on the same PR branch from both holding CI runners simultaneously.

**Common job scaffold (all four jobs share this pattern):**

```yaml
jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version-file: '.nvmrc'   # or node-version: '20' if no .nvmrc
          cache: 'npm'                  # built-in npm cache (equivalent to actions/cache on ~/.npm)
      - run: npm ci
      - run: npm run lint && npm run format:check
```

Using `actions/setup-node@v4` with `cache: 'npm'` is the modern equivalent of a manual `actions/cache@v4` on `~/.npm`. Both satisfy the "node_modules cached" AC; prefer `setup-node`'s built-in cache because it reduces YAML boilerplate and automatically keys on `package-lock.json`.

**Job structure:**

1. **lint** ‚Äî `npm run lint` and `npm run format:check`. No `needs:`.
2. **typecheck** ‚Äî `tsc --noEmit`. No `needs:`.
3. **unit** ‚Äî `npx vitest run` with `CI=true`. No `needs:`.
4. **e2e** ‚Äî deploy ‚Üí seed ‚Üí Playwright. Uses `needs: [lint, typecheck, unit]` to fail fast before spending E2E runtime on a statically-broken PR. Set `timeout-minutes: 20` to guard against runaway Playwright jobs.

**e2e job ‚Äî full step sequence:**

```yaml
e2e:
  runs-on: ubuntu-latest
  needs: [lint, typecheck, unit]
  timeout-minutes: 20
  env:
    CLAUDE_MOCK: 'true'
    CONVEX_DEPLOY_KEY: ${{ secrets.CONVEX_DEPLOY_KEY }}
    VITE_CONVEX_URL: ${{ secrets.VITE_CONVEX_URL }}
  steps:
    - uses: actions/checkout@v4
    - uses: actions/setup-node@v4
      with:
        node-version-file: '.nvmrc'
        cache: 'npm'
    - run: npm ci
    - run: npx convex deploy
    - run: npx convex run seed
    - run: npx playwright install --with-deps
    - run: npx playwright test
```

**`VITE_CONVEX_URL` is a static secret, not derived from deploy output.** A Convex project URL (e.g., `https://happy-otter-123.convex.cloud`) is a stable property of the project, fixed at project creation time. It does not change between deployments. Store it as a GitHub repo secret `VITE_CONVEX_URL`. There is no `--print-url` flag on `npx convex deploy`; do NOT attempt to parse the URL from deploy stdout.

**Convex deployment in CI:** Requires the `CONVEX_DEPLOY_KEY` secret set in the repository. Use `npx convex deploy` (with the key in env); do NOT use `npx convex dev` ‚Äî it is an interactive watch process.

**Seeding across multiple CI runs:** `convex/seed.ts` (from T11) uses `patch` (upsert) semantics for the admin user and template records, so it is safe to run repeatedly against the same Convex dev deployment without accumulating duplicate data. The seed step will overwrite existing records rather than fail.

**Playwright browser caching (optional optimisation):** `npx playwright install --with-deps` downloads ~300 MB of browser binaries on every run. For faster CI, add a second `actions/cache@v4` keyed on the Playwright version:

```yaml
- uses: actions/cache@v4
  with:
    path: ~/.cache/ms-playwright
    key: playwright-${{ runner.os }}-${{ hashFiles('package-lock.json') }}
```

Place this step before `npx playwright install --with-deps`. This is an optimisation, not a correctness requirement ‚Äî the suite works correctly either way.

**Playwright project coverage:** Running `npx playwright test` with no `--project` flag picks up all three browser projects (`chromium`, `firefox`, `webkit`) defined in `playwright.config.ts` (T34). Do not add a `--project` filter.

**Secrets required:**
- `CONVEX_DEPLOY_KEY` ‚Äî Convex project deploy key
- `VITE_CONVEX_URL` ‚Äî Stable Convex project URL (e.g., `https://happy-otter-123.convex.cloud`); stored as a secret so it isn't committed to the repo
- `ANTHROPIC_API_KEY` ‚Äî NOT needed for the `CLAUDE_MOCK=true` suite; only required if a live Anthropic integration test is ever added

- Implementation: `.github/workflows/ci.yml`
- No Vitest tests for this task (YAML-only, no TypeScript)
- Validation: manual inspection of GitHub Actions run log + green status badge

### Exports
- No TypeScript exports. Artifact is the YAML workflow file.

### Signatures
- N/A (YAML configuration, not TypeScript)

### Queries used
- `npx convex deploy` ‚Äî pushes schema and functions to Convex cloud using `CONVEX_DEPLOY_KEY`
- `npx convex run seed` ‚Äî executes `convex/seed.ts` on the deployed backend (idempotent ‚Äî safe on every run)
- `npx playwright test` ‚Äî runs all Playwright specs from `playwright.config.ts` (chromium + firefox + webkit)
- `npx vitest run` ‚Äî runs all Vitest unit tests in non-watch mode

### Invariants
- `npm ci` (not `npm install`) must be used in all jobs ‚Äî reproducible installs from `package-lock.json`.
- `CLAUDE_MOCK=true` must be set for the e2e job ‚Äî the suite must never make live Anthropic API calls in CI.
- `npx convex dev` must NOT appear in any CI job ‚Äî it is a foreground watch process, not a deploy command.
- `VITE_CONVEX_URL` is a static repo secret; do NOT attempt to derive it from deploy command output.
- Cache key is based on `package-lock.json` hash; a lock-file change invalidates the cache automatically.
- Playwright browser installation uses `--with-deps` to install system dependencies on the GitHub-hosted runner.
- `actions/checkout@v4` and `actions/setup-node@v4` (with `node-version-file` or `node-version`) must appear in every job before any `npm` or `npx` command.
- `concurrency.cancel-in-progress: true` must be set to prevent redundant parallel runs on the same PR branch.
- `e2e` job must have `timeout-minutes: 20` to prevent runaway Playwright jobs from consuming CI quota.

## Stack Practices

- GitHub Actions cache: `actions/setup-node@v4` with `cache: 'npm'` (preferred) or `actions/cache@v4` on `~/.npm`, keyed on `hashFiles('package-lock.json')`. Use `npm ci` not `npm install`.
- Convex deploy in CI: `npx convex deploy` with `CONVEX_DEPLOY_KEY` secret. `VITE_CONVEX_URL` is a separate static secret.
- Playwright in CI: `npx playwright install --with-deps` before `npx playwright test`.
- Anti-pattern: do NOT run `npx convex dev` in CI ‚Äî use `npx convex deploy`.
- Anti-pattern: do NOT attempt to parse `VITE_CONVEX_URL` from deploy stdout ‚Äî it is a stable project property stored as a repo secret.
- Add `concurrency.cancel-in-progress: true` on all PR-triggered workflows.
- Add `timeout-minutes:` to any job that invokes browsers or external services.
