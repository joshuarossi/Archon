The case creation form is the initiator's first substantive action: describing the conflict, framing their private perspective, and choosing who they're inviting. This 5-step progressive form bridges the landing experience into the actual mediation by gathering structured data (category, shared topic, private context, desired outcome, invitee name) before calling `cases/create` and routing to an invite-sharing screen. It is the entry point for every real case in the system, so its privacy signaling, category-driven template selection, and post-submit invite UX directly determine whether the initiator successfully brings a second party into the session.

## Acceptance Criteria

- Five steps with Back/Next navigation; Submit on step 5
- Step 1: category radio cards for all 5 categories
- Step 2: mainTopic with 140-char counter, labeled 'visible to other person'
- Steps 3+4: lock icon + tooltip on private fields
- Step 5: other party name; Advanced disclosure with solo mode
- Crisis language prominently displayed
- Submit calls cases/create; routes to /cases/:id/invite
- /cases/:id/invite: copyable invite URL; email mailto pre-fill; 'Start coaching now' link

## Implementation Notes

The form lives at `/cases/new`. Its state is a single `CaseFormData` object held in local component state, managed through `useCaseForm`. Each "Next" press validates only the current step's fields before advancing. "Back" is unconditional and the Back button is hidden (not just disabled) on step 1.

**Template ↔ category derivation:** `CaseNew` fetches `api.templates.list` (filtered to non-archived, per T11). The five unique `category` values are derived by deduplicating the template list: `[...new Set(templates.map(t => t.category))]`. This deduplicated array is passed to `CategoryRadioCards` as `categories: Array<{ id: string; name: string }>` where `id === name === category`. When the user selects a category in step 1, `setField('category', selectedCategory)` is called on the hook. At submit time (step 5), `CaseNew` resolves `templateId` by finding `templates.find(t => t.category === formData.category)?.._id` — the first matching template for the selected category. `templateId` is NOT stored in `CaseFormData`; it is derived at submit time.

**`cases/create` mutation input shape:**
```typescript
// Input sent to api.cases.create
interface CreateCaseArgs {
  templateId: Id<'templates'>;
  mainTopic: string;
  description: string;
  desiredOutcome: string;
  inviteeName?: string; // omitted entirely (not sent as '') when isSolo === true
  isSolo: boolean;
}
// Return from api.cases.create
interface CreateCaseResult {
  caseId: Id<'cases'>;
  inviteUrl: string;
}
```

**Mutation call site:** `CaseNew` calls `useMutation(api.cases.create)` directly (not via `useCaseForm`). The hook provides only form state and navigation logic. On success, `CaseNew` calls `navigate('/cases/${caseId}/invite', { state: { inviteUrl } })`.

**`inviteUrl` delivery:** `CaseInvite` reads the URL exclusively from `useLocation().state.inviteUrl`. If `state` is absent (direct navigation, page reload), render a friendly error: "This link is only valid immediately after creating a case. Go back to your dashboard." The invariant that `inviteUrl` is never derived client-side is enforced by treating absent state as an error state, not a loading state.

Step 2's `mainTopic` textarea deliberately avoids a `maxLength` HTML attribute: the character counter turns `--danger` above 140 but does not block submission, giving the user awareness without hard truncation. The label "In one sentence, what's this about?" and the helper text "This will be visible to the other person" are both rendered above/below the textarea so the shared-visibility contract is unmistakable.

Steps 3 and 4 collect fields that are private to the initiator (description/perspective, desired outcome). Each label carries a Lucide `Lock` icon (16px, 1.5px stroke) and a shadcn/ui `Tooltip` ("Only you and the AI coach will see this."). These fields also receive a 4px `--private-tint` left-border accent to reinforce T18's "Privacy is Visible" design principle. Steps 3 and 4 fields are optional; `canNext()` returns `true` unconditionally at those steps regardless of field content.

Step 5 collects `inviteeName` (the invitee's first name). An "Advanced" `Collapsible` section below exposes the solo-mode checkbox (`isSolo`). When `isSolo` is checked, the invitee name field is hidden and `canNext()` returns `true` without requiring `inviteeName`. On submit when `isSolo === true`, the `inviteeName` key is omitted from the mutation args (not passed as an empty string).

Crisis language is rendered as a full-width banner (not inline) above the Next/Submit button on every step — `--danger-subtle` background, Lucide `AlertTriangle` in `--danger`, with a link to `/crisis-resources` (static placeholder). This is non-negotiable per the Epic's safe-use guardrails.

The invite-sharing screen renders the URL in a read-only `Input` with a "Copy link" primary button (using `navigator.clipboard.writeText`). The email share ghost button links to:
```
mailto:?subject=I'd like to work through something with you&body=I've started a Conflict Coach session and would love your perspective. Join here: ${inviteUrl}
```
A text share ghost button links to `sms:?body=Join my Conflict Coach session: ${inviteUrl}`. A "Start coaching now →" secondary link routes to `/cases/${caseId}/private`.

Both routes depend on the case CRUD backend (T9) for `cases/create`, the templates backend (T11) for `templates/list`, and the shared components from T19a and T19b.

## Stack Practices

- Multi-step form state: single FormData object with setFormData(prev => ({ ...prev, field: value })).
- Character counter: {mainTopic.length}/140; text-danger when >140. Do NOT use maxLength HTML attr.
- Radio cards for category: shadcn/ui RadioGroup + RadioGroupItem. NOT a <select> dropdown.
- shadcn/ui Tooltip for private field explainer; Collapsible for 'Advanced' section.
- Clipboard API: navigator.clipboard.writeText(inviteUrl).
- `templateId` is resolved at submit time from `api.templates.list` — it is NOT stored in `CaseFormData` or `useCaseForm` state.
- `inviteUrl` travels from mutation response → `navigate` state → `useLocation().state` — never reconstructed client-side.
- Back button: hidden (not disabled) at step 1. Use conditional rendering, not `disabled` attribute.
- `isSolo` omits `inviteeName` from mutation args entirely — do not pass empty string.

## Design Practices

- Form container: single-column, max-width 720px, bg-canvas, centered. Form card uses bg-surface, shadow-1, 32px padding. Step indicator at top: small pills or numbered steps in --text-secondary with current step in --accent.
- Step 1 — Category radio cards (design doc §4.5): each category is a full-width card with border --border-default, bg-surface, 16px padding. Selected state: border 2px --accent, bg-accent-subtle. Hover: bg-surface-subtle, transition 150ms ease-out. Labels in 15px --text-primary 500 weight. Do NOT use a dropdown — tactile card choice is intentional. Wrapped in `<fieldset><legend>Choose a category</legend>…</fieldset>`.
- Step 2 — Main Topic: Textarea label 'In one sentence, what's this about?' in 500 weight --text-primary. Helper text 'This will be visible to the other person…' in 14px --text-secondary. Character counter right-aligned 12px --text-tertiary; switches to --danger when >140 chars. No maxLength attribute — allow over-limit with visual warning per spec.
- Steps 3+4 — Private fields (design doc §4.5): Lucide Lock icon (16px, 1.5px stroke) adjacent to label, in --text-secondary. Tooltip (shadcn/ui) on hover: 'Only you and the AI coach will see this.' Private field sections have --private-tint (#F0E9E0) left-border accent (4px) to visually reinforce privacy (design doc §1.3 Privacy is Visible).
- Step 5 — Invitee name: plain Input, no privacy indicator (name is shared). 'Advanced' Collapsible section below in --text-tertiary 13px; solo mode checkbox inside styled subtly.
- Crisis language: prominent banner (not inline) above the submit button. --danger-subtle background (#F2DCD8), --danger text, Lucide AlertTriangle. 'Not for crisis or emergency situations.' with link to `/crisis-resources` (static placeholder). Must be on every step.
- Navigation buttons: Back = shadcn/ui Button variant='ghost', hidden on step 1; Next/Submit = primary --accent filled. Single primary action per step (design doc §1.4).
- Post-create invite screen (design doc §4.6): bg-canvas page, centered card bg-surface shadow-1. Invite URL in a read-only Input with 'Copy link' primary --accent button adjacent. Email/text share options as secondary ghost buttons. 'Start coaching now →' secondary link in --accent — visually subordinate to 'Copy link'.
- Accessibility: fieldset + legend for radio card groups. Each Input/Textarea has an associated <label>. Step indicator uses aria-current='step' on active step. Form errors inline below inputs with role='alert'. Tooltip trigger is focusable with keyboard.
