# Design tokens and global styles

T18 establishes the visual foundation that every Conflict Coach screen is built on. The product's calm, warm aesthetic — described in the Epic as "Calm over Clever" — depends on a single authoritative set of CSS custom properties, Tailwind utility mappings, and typographic choices being applied consistently from day one. Without this layer in place, subsequent UI tickets (T19a, T19b, and all view tasks) would either introduce ad-hoc color values or be blocked waiting for the palette. This ticket creates that shared foundation: 16 semantic color tokens, four elevation shadows, typography loading, dark mode via `prefers-color-scheme`, and the global reduced-motion guard.

## Acceptance Criteria

- All 16 color tokens defined as CSS custom properties in src/index.css
- Dark mode via @media (prefers-color-scheme: dark)
- Tailwind config maps each token to a semantic utility class
- Inter variable font loaded (400/500/600); JetBrains Mono loaded
- Four shadow utilities matching design doc values
- @media (prefers-reduced-motion: reduce): all transitions 0ms, animations paused

## Implementation Notes

This task has no JavaScript logic — it is entirely CSS and Tailwind configuration. All work lives in two files: `src/index.css` and `tailwind.config.js`, plus font side-effect imports in `src/main.tsx`.

**Color tokens** are defined as CSS custom properties on `:root` and overridden in a `@media (prefers-color-scheme: dark)` block. Do NOT define color values directly in `tailwind.config.js` — the Tailwind config must reference the CSS variables (e.g., `canvas: 'var(--bg-canvas)'`). This keeps dark mode automatic and avoids duplicated values across two files.

**shadcn/ui** uses its own CSS variable naming convention (e.g., `--background`, `--foreground`, `--primary`). Those variables are written by `npx shadcn-ui@latest init` (T1) and must not be overwritten. The 16 design-doc tokens are additions, not replacements. Map them alongside the shadcn palette in `tailwind.config.js` under `theme.extend.colors`.

**Typography**: Use the `@fontsource-variable/inter` npm package for Inter — it provides the full variable font (weight axis 100–900) in a single import. Do NOT use `@fontsource/inter`, which installs separate static-weight CSS files and does not include the variable font ranges. Install: `npm install @fontsource-variable/inter`. Add the side-effect import in `src/main.tsx`:
```ts
import '@fontsource-variable/inter';
```
For JetBrains Mono (weight 400 only), install `@fontsource/jetbrains-mono` and import in `src/main.tsx`:
```ts
import '@fontsource/jetbrains-mono/400.css';
```
Body text is 15px / 1.6 line-height set on the `html` or `body` selector in `src/index.css`. Headings at 500 weight — not 700.

**Shadows**: Define four named shadow utilities in `tailwind.config.js` under `theme.extend.boxShadow`: `shadow-0` (flat/borders only — i.e., `none`), `shadow-1` (`0 1px 2px rgba(0,0,0,0.04)`), `shadow-2` (`0 4px 12px rgba(0,0,0,0.06)`), `shadow-3` (`0 12px 32px rgba(0,0,0,0.10)`). These map exactly to the elevation system used in the design doc §2.4 and will be referenced by component tickets.

**Reduced motion**: Add a global `@media (prefers-reduced-motion: reduce)` block in `src/index.css` that sets `transition-duration: 0ms !important`, `animation-duration: 0ms !important`, and `animation-play-state: paused !important`. All three declarations are required. This is the single place where all motion is globally disabled — individual component tickets add animations inside a `@media (prefers-reduced-motion: no-preference)` guard so they are already opt-in.

**Tailwind v3 dark mode**: Set `darkMode: 'media'` in `tailwind.config.js`. This activates the `dark:` variant driven by `prefers-color-scheme` rather than a class toggle, which is correct for v1 (no manual theme toggle required).

**Content glob**: The Tailwind `content` array must include `./src/**/*.{ts,tsx}` and `./convex/**/*.ts` so utility classes used in Convex action files are not purged.

### Invariants

- Every one of the 16 design-doc color tokens must appear as both a CSS custom property in `src/index.css` (`:root` block) and as a Tailwind color key in `tailwind.config.js`.
- The Tailwind color values must be `var(--<token>)` strings — never hex literals — so dark mode overrides flow through automatically.
- `darkMode` must equal `'media'` (not `'class'`).
- `boxShadow` keys `'0'`, `'1'`, `'2'`, `'3'` must all be present with the exact RGBA values from the design doc.
- The `@media (prefers-reduced-motion: reduce)` block must set all three of: `transition-duration: 0ms !important`, `animation-duration: 0ms !important`, and `animation-play-state: paused !important`.
- Inter variable font must be loaded via `@fontsource-variable/inter` (side-effect import in `src/main.tsx`); JetBrains Mono must be loaded at weight 400 via `@fontsource/jetbrains-mono/400.css`.
- The Tailwind `content` array must contain at least `./src/**/*.{ts,tsx}` and `./convex/**/*.ts`.

### Non-goals

- This ticket does NOT implement any React components or layout.
- It does NOT implement a manual dark mode toggle (no `class`-based dark mode, no user preference storage).
- It does NOT define shadcn/ui's own CSS variables (`--background`, `--foreground`, etc.) — those are written by `npx shadcn-ui@latest init` in T1.
- It does NOT verify WCAG contrast ratios programmatically — that audit is T33's responsibility.
- It does NOT add per-component CSS files or CSS Modules.

## Stack Practices

- CSS custom properties go in `src/index.css` under `:root` and `@media (prefers-color-scheme: dark)`.
- Tailwind config extension maps tokens: `canvas: 'var(--bg-canvas)'` etc.
- shadcn/ui CSS variable names must match the shadcn convention.
- Tailwind v3 dark mode: set `darkMode: 'media'` in tailwind.config.js.
- Font packages: `@fontsource-variable/inter` for the variable font (one import, full weight axis); `@fontsource/jetbrains-mono` for JetBrains Mono. Import both as side effects in `src/main.tsx`.
- Anti-pattern: do NOT define color values in both tailwind.config.js and src/index.css separately.
- Anti-pattern: do NOT use `@fontsource/inter` (static weight variant) — it does not provide variable font ranges and requires a separate file per weight.

## Design Practices

- Implement all 16 color tokens verbatim from design doc §2.1: --bg-canvas (#FAF8F5), --bg-surface (#FFFFFF), --bg-surface-subtle (#F3EFE9), --text-primary (#1F1D1A), --text-secondary (#5C5952), --text-tertiary (#8A8680), --accent (#6B8E7F), --accent-hover (#5A7A6C), --accent-subtle (#DCE7E0), --danger (#B5594D), --danger-subtle (#F2DCD8), --warning (#B58B4D), --border-default (#E5E0D8), --border-strong (#CBC4B8), --coach-accent (#8B7AB5), --private-tint (#F0E9E0).
- Typography (design doc §2.2): Inter variable font at 400/500/600 weights; body text 15px / 1.6 line-height (chat bubbles 16px for readability); chat timestamps 12px in --text-tertiary; headings at 500 weight — NOT 700 (this is not a marketing page); JetBrains Mono for invite links and audit log IDs.
- Spacing & Layout (design doc §2.3): 8px base grid with paddings at 8, 16, 24, 32, 48px. Content max-width 720px for reading-heavy screens (forms, synthesis); 1080px for joint chat. Mobile breakpoint: < 768px.
- Elevation (design doc §2.4): shadow-0 = flat/borders only; shadow-1 = '0 1px 2px rgba(0,0,0,0.04)' for cards at rest; shadow-2 = '0 4px 12px rgba(0,0,0,0.06)' for popovers/dropdowns; shadow-3 = '0 12px 32px rgba(0,0,0,0.10)' for modals and Draft Coach panel.
- Motion (design doc §2.6): all transitions 150–200ms ease-out; new message arrival = fade-in + 8px upward translate (150ms); route crossfade 100ms; NO slide animations; NO celebration animation on case closure. Under prefers-reduced-motion: reduce → set all three: `transition-duration: 0ms`, `animation-duration: 0ms`, and `animation-play-state: paused` (all with `!important`).
- Dark mode: invert luminance of all 16 tokens while preserving warm, low-saturation character — no stark white-on-black. Reference design doc §2.1 dark mode note.
- Accessibility: ensure every --text-* / --bg-* combination meets WCAG 2.1 AA 4.5:1 contrast ratio in both light and dark modes. Use the are-you-ok CLI or browser DevTools to verify before T33.
