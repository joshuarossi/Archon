# Ready-for-Joint View

The ready-for-joint screen (`/cases/:id/ready`) is the pivotal pause point between private coaching and the joint session. After both parties have completed private coaching and the AI has generated their personalized synthesis, this view presents each party's private guidance in a calm, reading-friendly layout and offers a single primary action to enter the joint session. Its purpose is to give users a moment to absorb the AI's insights before the more intense joint conversation begins — unhurried, reflective, and clearly private.

## Acceptance Criteria

- Fetches jointChat/mySynthesis; skeleton while loading
- Synthesis card with privacy label
- Synthesis rendered as markdown
- 'Enter Joint Session →' primary button
- Calls jointChat/enter; routes to /joint on success
- Handles case already JOINT_ACTIVE

## Implementation Notes

`ReadyForJointView.tsx` sits under the `CaseDetail` layout route (T24), which provides the `PhaseHeader` and the `caseData` outlet context. The component should not re-fetch `cases/get` directly — pull `caseData` from `useOutletContext`.

```typescript
// Import from the paths established in T24
import { useOutletContext, useNavigate, useParams } from 'react-router-dom';
import type { CaseDetailContext } from './cases/types';
import { CaseDetailSkeleton } from './cases/CaseDetailSkeleton';
import ReactMarkdown from 'react-markdown';
import rehypeSanitize from 'rehype-sanitize';
```

**Obtaining `caseId`:** Read it from `useParams<{ caseId: string }>()`. Do not extract it from outlet context — the route param is the authoritative source.

**Data fetching:** `jointChat/mySynthesis` returns `{ text: string } | null`. `undefined` means still loading (Convex convention); `null` means synthesis is not yet generated (pending AI action from T13). Both states need distinct UI treatment:
- `undefined` → show `<CaseDetailSkeleton variant="card" />` (card variant — this is a centered-card layout, not a full-height chat column; see T24 `CaseDetailSkeleton` docs). Apply a 300ms `setTimeout` before revealing the skeleton to avoid flash (visual-only; not unit-tested via fake timers).
- `null` → show a pending state ("Synthesis is being generated…") with a subtle shimmer — no spinner
- populated string → render the synthesis card

**Entering the session:** Call `useMutation(api.jointChat.enter)` on button click. `jointChat/enter` is idempotent — if the case is already `JOINT_ACTIVE` (other party entered first), it succeeds normally. On any success, call `navigate('../joint')` (no `{ replace: true }` — normal forward navigation). Do not gate on the current case status client-side; let the server handle idempotency.

**Reactive case status:** Add a `useEffect` that watches `caseData.case.status`. If status is already `JOINT_ACTIVE` on mount (the user navigated back here after entering), call `navigate('../joint', { replace: true })` immediately without user interaction.

**Markdown rendering:** Use `react-markdown` with `rehype-sanitize` and the `@tailwindcss/typography` plugin:
```tsx
<ReactMarkdown
  className="prose prose-sm max-w-none"
  rehypePlugins={[rehypeSanitize]}
>
  {mySynthesis.text}
</ReactMarkdown>
```
Headings in the synthesis text (e.g., **Areas of likely agreement**, **Points that will need real discussion**, **Suggested approach**) render as `h3` or bold paragraphs — the prose plugin handles styling. Never use `dangerouslySetInnerHTML`.

**Privacy header:** The synthesis card must display a clear "Private to you" strip at the top using `--private-tint` background and a Lucide Lock icon. The exact copy is: `"Private to you — {inviteeName} has their own version."` Use `caseData.case.inviteeName ?? 'the other party'` when the name is unavailable.

**Component dependencies:** Reuses `CaseDetailSkeleton` from `src/routes/cases/CaseDetailSkeleton.tsx` (T24) for loading state, using `variant="card"`. No new shared components needed.

## Stack Practices

- Markdown rendering: react-markdown with @tailwindcss/typography plugin for prose classes; `rehypePlugins={[rehypeSanitize]}` required — never `dangerouslySetInnerHTML`.
- Idempotent enter: `navigate('../joint')` (no `replace`) on any success from `jointChat/enter`.
- Skeleton: `synthesis === undefined` → `<CaseDetailSkeleton variant="card" />`; `synthesis === null` → pending state.
- `caseId`: from `useParams<{ caseId: string }>()`, not outlet context.
- Anti-pattern: do NOT render markdown via `dangerouslySetInnerHTML` — use `react-markdown + rehype-sanitize`.
- Anti-pattern: do NOT re-fetch `cases/get` inside this view — read from `useOutletContext<CaseDetailContext>()`.
- Import paths: `CaseDetailContext` from `./cases/types`; `CaseDetailSkeleton` from `./cases/CaseDetailSkeleton`.

## Design Practices

- Page layout (design doc §4.8): centered column, max-width 720px, bg-canvas. Heading 'You've both completed private coaching.' in 500 weight --text-primary. Helper text in --text-secondary 15px. This screen is a moment of pause — ample vertical whitespace (48px between sections).
- Synthesis card: bg-surface, shadow-1, border --border-default, 24px padding, border-radius 12px. Header strip: --private-tint background, Lock icon + 'Private to you — Jordan has their own version' in 13px --text-secondary.
- Synthesis markdown rendering (design doc §4.8): use react-markdown with className='prose prose-sm max-w-none'. Typography plugin renders headings (**Areas of likely agreement**, **Points that will need real discussion**, **Suggested approach**) in 500 weight --text-primary. Body prose in --text-secondary 15px / 1.6 line-height.
- 'Enter Joint Session →' button (design doc §4.8): large primary --accent, centered, min-width 240px. This IS the main action — make it visually prominent. Helper text below: 'Jordan will see you've entered when they enter too.' in 13px --text-tertiary.
- Synthesis loading/pending states: skeleton (300ms delay) while undefined. When null: 'Synthesis is being generated…' with a subtle shimmer or Pause icon — no spinner (too mechanical for the calm tone).
- Accessibility: synthesis card content has `aria-label='Your private guidance'`. 'Enter Joint Session' button — after clicking, announce 'Entering joint session…' via a live region or button disabled state with loading text. Route change moves focus to /joint h1.
