# Private coaching view (/cases/:id/private) with crisis card integration

The private coaching view is the first phase of the mediation process — the screen where each party works one-on-one with the AI coach before the joint session begins. It is deliberately isolated: each user sees only their own messages and AI responses, never the other party's. This ticket wires the `PrivateCoachingView` component to the reactive `privateCoaching/myMessages` backend, integrates the shared chat primitives built in T19a/T19b, adds the mark-complete flow, and attaches `CrisisCard` detection so that crisis-keyword responses surface an immediate safety prompt.

## Acceptance Criteria

- Loads privateCoaching/myMessages reactively
- PrivacyBanner always visible
- Streaming/Complete/Error message states
- Send disabled while STREAMING
- Mark-complete footer CTA + confirmation modal
- Routes to /ready if case advances to READY_FOR_JOINT
- CrisisCard appears on crisis keywords; dismissable; re-appears on new crisis message

## Implementation Notes

`PrivateCoachingView` is a sub-route rendered inside the `CaseDetail` layout route created in T24. It receives the case data via `useOutletContext<CaseDetailContext>()` — import `CaseDetailContext` from `src/routes/cases/types.ts` (defined in T24). Do not re-fetch `cases/get` here.

The reactive query `api.privateCoaching.myMessages` returns only the calling user's messages and their corresponding AI coach responses. This is enforced server-side (T12); the frontend simply renders what it receives. The `status` field on each message (`STREAMING | COMPLETE | ERROR`) drives which `MessageBubble` variant to render.

Streaming detection: derive `isStreaming` by checking `messages?.some(m => m.status === 'STREAMING')`. Pass this boolean to `MessageInput` to disable the send button and textarea while the coach is responding. The backend (T12) guarantees only one STREAMING row exists at a time, so this check is safe.

Mark-complete flow: the footer CTA is intentionally de-emphasized (small text link or ghost button, not a primary action) because the design doc warns against rushing users through private coaching. On click, open a shadcn/ui `AlertDialog` showing the count of COMPLETE messages (not STREAMING or ERROR rows). The two actions are "Continue Coaching" (cancel) and "Mark Complete" (calls `api.privateCoaching.markComplete`). The mutation is idempotent. After it resolves, the case status will eventually advance to `READY_FOR_JOINT` via the backend synthesis trigger — watch this in a `useEffect` on `caseData.case.status` and navigate to `../ready` with `replace: true`. Do NOT show the mark-complete AlertDialog while `isStreaming === true`; the CTA should be hidden or disabled in that state.

Route transitions: watch `caseData.case.status` in a `useEffect`. The full mapping is:
- `READY_FOR_JOINT` → `navigate('../ready', { replace: true })`
- `JOINT_ACTIVE` → `navigate('../joint', { replace: true })`
- `COST_LIMITED` → `navigate('../joint', { replace: true })`
- `CLOSED_RESOLVED`, `CLOSED_UNRESOLVED`, `CLOSED_ABANDONED` → `navigate('../closed', { replace: true })`

Guard against duplicate navigations with a `useRef` flag set to `true` on first trigger; reset on unmount.

CrisisCard integration: the `CrisisCard` component (T19b) takes `isVisible` and `onDismiss` props and manages its own dismissed state internally. To make it re-appear when a new crisis message arrives after a prior dismissal, assign a `key` prop equal to the count of user messages in the array: `key={messages?.filter(m => m.authorRole === 'USER').length ?? 0}`. Crisis detection happens server-side — the presence of a coach message with `isCrisis: true` should control `isVisible`.

File location: `src/routes/PrivateCoachingView.tsx`. Uses ChatWindow, MessageBubble, MessageInput, StreamingIndicator (T19a), PrivacyBanner, CrisisCard (T19b), and shadcn/ui AlertDialog.

## Stack Practices

- Reactive streaming: useQuery(api.privateCoaching.myMessages) — no manual polling.
- isStreaming: messages?.some(m => m.status === 'STREAMING'). Pass to MessageInput.
- Route transition on case advance: watch caseData.case.status in useEffect.
- CrisisCard key prop: `messages?.filter(m => m.authorRole === 'USER').length ?? 0` — resets dismissed state when new user message sent.
- Mark-complete: shadcn/ui AlertDialog for confirmation. CTA hidden/disabled while isStreaming.
- Navigate guard: useRef flag to prevent duplicate navigation calls on re-render.

## Design Practices

- Full-height layout (design doc §4.7): PhaseHeader (sticky top), PrivacyBanner (persistent below header), ChatWindow (flex-grow, overflow-y-auto), MessageInput (fixed bottom), Footer CTA below input.
- PrivacyBanner: full-width strip, --private-tint (#F0E9E0) background, Lucide Lock icon, 14px --text-secondary, always visible — not collapsible. Clicking opens shadcn/ui Dialog (shadow-3) with full privacy explanation.
- Coach messages: --accent-subtle (#DCE7E0) background, left-aligned, Sparkles icon (Lucide 16px 1.5px stroke) at top-left. User messages: --bg-surface, right-aligned. Chat bubbles at 16px Inter for readability (design doc §2.2).
- Streaming bubble: same --accent-subtle background with blinking cursor at end (500ms, step-end). No copy button while STREAMING. aria-label 'Coach is replying' on the STREAMING bubble's wrapper (role='status'). Remove role='status' when COMPLETE.
- Mark-complete footer CTA (design doc §4.7): subtle treatment — text-based link or very small secondary button in --text-secondary, not a large prominent button. Design intent: don't rush users through private coaching. Footer area: bg-surface, border-top --border-default, 16px padding.
- Mark-complete AlertDialog: shadcn/ui AlertDialog. Title: 'Ready to move on?' Body shows count of COMPLETE messages. Two actions: 'Continue Coaching' (cancel, ghost) and 'Mark Complete' (primary --accent). No celebration styling on confirmation.
- CrisisCard: slides in below PrivacyBanner (150ms ease-out), --danger-subtle background, Lucide AlertTriangle in --danger. Dismissable ✕ button. Re-appears (via key prop reset) on new crisis message.
- Accessibility: ChatWindow has role='log' aria-live='polite'. CrisisCard uses role='alert' so it's announced immediately. Mark-complete AlertDialog uses shadcn/ui's built-in focus trap and Esc close. Footer CTA keyboard-accessible.
