# Dashboard page (/dashboard) with unilateral-close notification indicator

The `/dashboard` route is the home base users land on after authentication — the primary overview of all their active and closed cases. Its job is to surface status at a glance and route users to the right phase without friction. The unilateral-close notification indicator is a critical fairness feature: when the other party closes a case without agreement, the non-acting party must see a visible signal so they are not left confused about why the case disappeared. This screen also exposes the Nudge button for stale invites and the entry point to case creation.

## Acceptance Criteria

- Fetches cases/list; renders skeleton while loading
- Active/Closed sections with correct status glyph mapping
- Unilateral close indicator: ◼ with red dot + message for CLOSED_UNRESOLVED non-actor
- Nudge button on DRAFT_PRIVATE_COACHING cases >7 days old without invitee
- Empty state with link to /cases/new
- Case rows keyboard-navigable; Enter activates

## Implementation Notes

`useQuery(api.cases.list)` is reactive — Convex pushes updates in real time, so there is no polling loop. The loading state is `undefined` (Convex convention); `null` would indicate a query returning no result, but `cases/list` always returns an array so the pattern is `cases === undefined → skeleton`.

### `getStatusGlyph` as the canonical pure function

The status-to-glyph mapping is extracted into `export function getStatusGlyph(caseItem: CaseSummary, currentUserId: Id<'users'>): StatusGlyph` in `src/lib/caseStatus.ts`. **This is the single source of truth for all glyph logic.** `StatusPill` (T19b, `src/components/StatusPill.tsx`) must import and delegate to `getStatusGlyph` rather than maintaining its own mapping — coordinate with the T19b implementer to update `StatusPill` to accept a pre-computed `StatusGlyph` or call `getStatusGlyph` internally. Do NOT let two independent switch statements diverge.

The unilateral-close case is a subtype of `CLOSED_UNRESOLVED` where the current user is NOT the actor who called `jointChat/unilateralClose`. The backend must include `closedByUserId` in `cases/list` output (see Non-goals — this ticket is blocked if that field is absent).

### Status mapping (all 9 statuses — exhaustive)

| status | glyph | color | label |
|--------|-------|-------|-------|
| `DRAFT_PRIVATE_COACHING` | ● | accent | Your Turn |
| `ACTIVE_PRIVATE_COACHING` | ● | accent | Your Turn |
| `DRAFT_INVITE_SENT` | ○ | secondary | Waiting for Invite |
| `JOINT_ACTIVE` | ○ | secondary | Waiting |
| `READY_FOR_JOINT` | ◐ | warning | Ready for Joint |
| `CLOSED_RESOLVED` | ◼ | tertiary | Closed |
| `CLOSED_UNRESOLVED` | ◼ | tertiary (base) | Closed — see unilateral variant below |
| `CLOSED_ABANDONED` | ◼ | tertiary | Closed |
| `COST_LIMITED` | ◼ | warning | Cost Limit Reached |

Unilateral-close variant: `CLOSED_UNRESOLVED` AND `closedByUserId !== currentUserId` → override `color` to `danger`, append `unilateralClose: { closedByName: string }` to the returned `StatusGlyph`.

### Nudge button

Condition: `case.status === 'DRAFT_PRIVATE_COACHING' && !case.inviteeUserId && Date.now() - case.createdAt > 7 * 24 * 60 * 60 * 1000`. The Nudge mutation is an out-of-scope backend feature for v1 (call `useMutation(api.cases.nudge)` — placeholder if the mutation is not yet built; do not `alert()`). Its visual presence is the AC here; the button must not crash if the mutation is a no-op.

### Closed section collapse / auto-expand

Local `useState<boolean>(false)` controls `closedExpanded`. On mount collapsed. Auto-expand via `useEffect`: compare previous closed-case IDs with current; if a new ID appears in the current closed list that was not in the previous render's closed list, call `setClosedExpanded(true)`. Implement previous-value tracking with `useRef<Id<'cases'>[]>`.

### Case row interaction

Click navigation: `navigate('/cases/${caseId}')`. Keyboard: `role="button"` + `tabIndex={0}` + `onKeyDown` (Enter triggers navigate). Accessible label: `aria-label={'Open case with ' + caseItem.counterpartName}` on the clickable row element.

### Skeletons

Use `shadcn/ui Skeleton` primitive. Show 3 skeletons at 72px height. Guard with a 300 ms `setTimeout` via local `useEffect` state (`showSkeleton`) to avoid flash on fast connections — render nothing until `showSkeleton === true`, and only then render `<Skeleton>` elements while `cases === undefined`.

### Queries used

- `api.cases.list` — via `useQuery`; returns `CaseSummary[]` including `status`, `createdAt`, `inviteeUserId`, `closedByUserId`, `closedByName`, `counterpartName`, `mainTopic`, `category`, `lastActivityAt`
- `api.cases.nudge` — via `useMutation`; accepts `{ caseId: Id<'cases'> }`; called on Nudge button click (placeholder if mutation not yet implemented — must not throw on undefined)

### Invariants

- Skeleton guard: `showSkeleton` local state starts `false`; only set to `true` after 300 ms `setTimeout` fires AND `cases === undefined`. Skeleton elements must NOT render before the 300 ms guard fires.
- `cases === undefined` (Convex loading) → show skeleton rows; `cases === []` (empty array) → show empty state; `cases.length > 0` → render rows.
- Closed section must be collapsed (`aria-expanded="false"`) on first render regardless of case count.
- Closed section must auto-expand when a case that was not in the closed list on the previous render appears in the current closed list.
- `getStatusGlyph` must be a pure function — no side effects, no Convex calls.
- `getStatusGlyph` must handle all 9 `CaseStatus` values without falling through to an unhandled branch (TypeScript exhaustive check via `never` at the default branch).
- Unilateral-close indicator (`data-testid="unilateral-close-indicator"`) must only appear when `case.status === 'CLOSED_UNRESOLVED'` AND `case.closedByUserId !== currentUserId`.
- Unilateral-close indicator must NOT appear when `case.closedByUserId === currentUserId` (actor who closed it).
- Nudge button must only appear on rows where `status === 'DRAFT_PRIVATE_COACHING'` AND `inviteeUserId` is falsy AND `Date.now() - createdAt > 7 days`.
- Each case row must have `aria-label={'Open case with ' + counterpartName}`.
- Active Cases `<section>` and Closed Cases `<section>` must each have `aria-labelledby` pointing to their respective heading `id`.

- This ticket does NOT implement the `+ New Case` form — it only links to `/cases/new`.

## Stack Practices

- Data fetching: `const cases = useQuery(api.cases.list)` — reactive, no polling needed.
- Closed section default collapsed; auto-expand when case transitions to CLOSED_* via `useEffect` + `useRef` tracking previous closed IDs.
- React Router: case row click → `navigate('/cases/${caseId}')`. Keyboard: `role='button'` `tabIndex={0}` `onKeyDown` Enter.
- Anti-pattern: do NOT derive status glyph logic in JSX directly — delegate to `getStatusGlyph()`. Do NOT duplicate the mapping in `StatusPill` (T19b) — `StatusPill` must consume `getStatusGlyph`.
- Use `useMutation(api.cases.nudge)` even if the mutation is a placeholder; do NOT use `alert()` as a fallback.
- Skeleton: use 300ms `setTimeout` guard via local `useEffect` state before rendering `<Skeleton>` elements (prevents flash on fast connections).

## Design Practices

- Page layout: --bg-canvas background. Top-right: '+ New Case' primary --accent button (design doc §4.4). Max-width content wrapper with 32px horizontal padding at desktop, 16px on mobile.
- Section headings 'Active Cases' / 'Closed Cases': 500 weight, --text-primary, 16px. 'Closed Cases' collapsed by default with a ChevronDown icon (Lucide 1.5px stroke).
- Case row (design doc §4.4): bg-surface, shadow-1 on hover (transition 150ms ease-out), border-bottom --border-default between rows. Padding 16px. Layout: icon column left → case title + metadata stack → status glyph + 'Enter →' right.
- Case title: 15px 500 weight --text-primary. Sub-metadata (category · date): 12px --text-secondary. Status line: 13px --text-secondary with glyph. Last activity: 12px --text-tertiary.
- Status glyphs (design doc §4.4): ● --accent green = Your Turn; ○ --text-secondary gray = Waiting; ◐ --warning amber = Ready for Joint; ◼ --text-tertiary neutral = Closed. Unilateral close: ◼ with a 6px --danger dot overlay + 'Jordan closed this case.' in 13px --danger. COST_LIMITED: ◼ --warning amber + 'Cost Limit Reached' label.
- Nudge button: shadcn/ui Button variant='outline', 13px, --text-secondary. Appears on stale DRAFT_PRIVATE_COACHING rows without invitee, >7 days old.
- Empty state: centered in content area, --text-secondary 15px, friendly non-cutesy copy per design doc §4.4. Link to /cases/new in --accent underline.
- Skeletons: 3 case row skeletons at 72px height with 300ms delay. Use shadcn/ui Skeleton with --bg-surface-subtle shimmer.
- Accessibility: case rows use `role='button'` `tabIndex={0}`. Enter key activates. Each row has `aria-label='Open case with [counterpartName]'` to differentiate rows for screen readers. `<section>` elements with `aria-labelledby` pointing to Active/Closed heading IDs.
