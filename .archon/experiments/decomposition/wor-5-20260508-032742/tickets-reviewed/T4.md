# Authorization Helper Module and ConvexError Codes

This task creates the shared authorization infrastructure that every Convex function in the project will rely on. Rather than repeating auth boilerplate inline, `convex/lib/auth.ts` provides a single `requireAuth(ctx)` helper that resolves a Convex identity to a typed `{ identity, user }` pair (or throws a normalized `ConvexError`). Two additional helpers — `assertPartyToCase` and `assertAdmin` — enforce the two most common permission boundaries in the app. A companion file `convex/lib/pricing.ts` centralizes AI cost constants so they can be updated in one place when Anthropic pricing changes.

## Acceptance Criteria

- `requireAuth(ctx)` returns `{ identity, user }` or throws `ConvexError` with code `UNAUTHENTICATED`
- All 9 error codes typed as constants
- `ConvexError` shape: `{ code, message, httpStatus }`
- `assertPartyToCase` throws `FORBIDDEN` if caller is not a party to the case
- `assertAdmin` throws `FORBIDDEN` if `user.role !== 'ADMIN'`
- `convex/lib/pricing.ts` exports soft/hard caps and per-token pricing

## Implementation Notes

The authorization pattern in the tech spec (§4.3) is the direct source of truth. Every query, mutation, and action is expected to open with a call to `requireAuth(ctx)`. The pattern it replaces is:

```ts
const identity = await ctx.auth.getUserIdentity();
if (!identity) throw new Error("UNAUTHENTICATED");
const user = await getUserByEmail(ctx, identity.email);
if (!user) throw new Error("USER_NOT_FOUND");
```

`requireAuth` wraps this with typed `ConvexError` throws so callers get structured errors, not raw strings.

**Error code table** (tech spec §7.4 — all 9 must be defined as typed string literal constants):

| Code | httpStatus | Meaning |
|------|-----------|---------|
| `UNAUTHENTICATED` | 401 | No valid session |
| `FORBIDDEN` | 403 | Authed but not permitted |
| `NOT_FOUND` | 404 | Resource missing |
| `CONFLICT` | 409 | Illegal state transition |
| `INVALID_INPUT` | 400 | Validation failed |
| `TOKEN_INVALID` | 400 | Invite token not active |
| `RATE_LIMITED` | 429 | Upstream rate limit hit |
| `AI_ERROR` | 502 | Upstream AI error |
| `INTERNAL` | 500 | Unexpected server failure |

`ConvexError` is imported from `convex/values`. The shape passed to its constructor is `{ code: ErrorCode, message: string, httpStatus: number }`.

**`assertPartyToCase`** reads the `cases` table, verifies `user._id === case.initiatorUserId || user._id === case.inviteeUserId`, and throws `FORBIDDEN` with a clear message if not. It accepts the resolved `user` object (from `requireAuth`) and a `Case` document already fetched by the caller — it does **not** re-fetch the case itself (avoids a second DB read) and takes no `ctx` parameter. It is a synchronous function.

**`assertAdmin`** takes the resolved `user` object and checks `user.role === 'ADMIN'`; throws `FORBIDDEN` if not.

**`convex/lib/pricing.ts`** centralizes all three cost constants referenced by T12–T17:
- `SOFT_CAP_USD = 2` — threshold for boilerplate degradation
- `HARD_CAP_USD = 10` — threshold for `COST_LIMITED`
- `COST_PER_INPUT_TOKEN` and `COST_PER_OUTPUT_TOKEN` — Anthropic Claude Sonnet 4.5 rates at time of writing; each constant **must carry an inline comment** noting the model name and date so future maintainers know when to update

The `tokenCountToUsd(inputTokens: number, outputTokens: number): number` helper in `pricing.ts` makes the per-action cost calculation a one-liner for callers. Its formula is `inputTokens * COST_PER_INPUT_TOKEN + outputTokens * COST_PER_OUTPUT_TOKEN`.

Both files are pure TypeScript with zero side effects — no Convex `query`/`mutation` wrappers, just exported functions and constants.

- `tokenCountToUsd` — named, pure function

**Signatures**

```ts
// convex/lib/auth.ts
import { ConvexError } from "convex/values";
import type { UserIdentity } from "convex/server";
import type { QueryCtx, MutationCtx, ActionCtx } from "./_generated/server";
import type { Doc } from "./_generated/dataModel";

export type ErrorCode =
  | "UNAUTHENTICATED"
  | "FORBIDDEN"
  | "NOT_FOUND"
  | "CONFLICT"
  | "INVALID_INPUT"
  | "TOKEN_INVALID"
  | "RATE_LIMITED"
  | "AI_ERROR"
  | "INTERNAL";

export type AppError = { code: ErrorCode; message: string; httpStatus: number };

export const ERROR_CODES: Record<ErrorCode, ErrorCode> = { ... };

export async function requireAuth(
  ctx: QueryCtx | MutationCtx | ActionCtx
): Promise<{ identity: UserIdentity; user: Doc<"users"> }>;

export function assertPartyToCase(
  user: Doc<"users">,
  caseDoc: Doc<"cases">
): void;  // sync; throws ConvexError FORBIDDEN if not party; never reads ctx.db

export function assertAdmin(user: Doc<"users">): void;  // throws ConvexError FORBIDDEN

// convex/lib/pricing.ts
// Anthropic Claude Sonnet 4.5 rates as of [date written] — update when Anthropic reprices.
export const COST_PER_INPUT_TOKEN: number;
export const COST_PER_OUTPUT_TOKEN: number;
export const SOFT_CAP_USD: number;
export const HARD_CAP_USD: number;
export function tokenCountToUsd(inputTokens: number, outputTokens: number): number;
// formula: inputTokens * COST_PER_INPUT_TOKEN + outputTokens * COST_PER_OUTPUT_TOKEN
```

**Queries used**
- `ctx.db.query("users").withIndex("by_email", q => q.eq("email", identity.email)).unique()` — inside `requireAuth`
- No external API calls

**Invariants**
- `requireAuth` never returns without a valid `user` — either returns both fields or throws
- `assertPartyToCase` never mutates the database and never reads from `ctx`
- `assertAdmin` never mutates the database
- `ERROR_CODES` contains exactly 9 entries matching the spec table
- `HARD_CAP_USD > SOFT_CAP_USD` — always true
- `tokenCountToUsd(0, 0) === 0`
- `tokenCountToUsd` is a pure function (same input → same output, no side effects)

**Non-goals**
- This ticket does NOT implement any Convex query, mutation, or action wrappers
- Does NOT define the `users` table schema (that is T2)
- Does NOT implement any route handler or HTTP endpoint
- Does NOT add admin promotion UI (manual dashboard action in v1)
- Does NOT define `getUserByEmail` as a separate exported helper — it can be inlined in `requireAuth`

## Stack Practices

- Import `ConvexError` from `convex/values`.
- Import `UserIdentity` from `convex/server` (not from `_generated/server`).
- Helper function signature: `async function requireAuth(ctx): Promise<{ identity; user }>`.
- File location: `convex/lib/auth.ts` and `convex/lib/pricing.ts`.
- Anti-pattern: do NOT put authorization logic inline in each function.
