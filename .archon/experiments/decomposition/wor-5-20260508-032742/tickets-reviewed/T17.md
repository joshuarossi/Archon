# AI cost tracking hardening: budget enforcement, soft-cap degradation, Anthropic error handling

T12–T15 each accumulate token counts into `cases.tokenCount` as they run. T17 is a cross-cutting hardening pass that reads those counts and acts on them: enforcing the $2 soft cap (Coach response degrades to boilerplate), the $10 hard cap (case marked `COST_LIMITED`, AI fully disabled), and handling upstream Anthropic failures (429 rate limits and network timeouts) uniformly across all four AI actions — `generateAIResponse` (private coaching), `synthesis/generate`, `generateCoachResponse` (joint chat), and `generateResponse` (Draft Coach). This task exists because AI cost control and error resilience are cross-cutting concerns that cannot be fully owned by any single feature task; adding them as a final hardening pass ensures consistent behavior across the whole AI surface.

## Acceptance Criteria

- Soft cap: generateCoachResponse posts boilerplate at $2 threshold
- Hard cap: all four AI actions return early at $10; set COST_LIMITED
- 429: retry once after 2000ms; second 429 → RATE_LIMITED
- Network timeout >30s: message status=ERROR; throw AI_ERROR

## Implementation Notes

`convex/lib/pricing.ts` was introduced in T4 and already exports `SOFT_CAP_USD`, `HARD_CAP_USD`, `COST_PER_INPUT_TOKEN`, `COST_PER_OUTPUT_TOKEN`, and `tokenCountToUsd(inputTokens, outputTokens)`. T17 extends that file with two additions:

- `COST_PER_TOKEN` — a blended per-token constant (arithmetic mean of `COST_PER_INPUT_TOKEN` and `COST_PER_OUTPUT_TOKEN`) used when the caller has only a single accumulated token count rather than separate input/output counts. Must carry an inline comment: `// blended rate — use tokenCountToUsd(i, o) when split counts are available`.
- `checkCostCap(tokenCount: number): 'ok' | 'soft_cap' | 'hard_cap'` — a pure synchronous function that converts a single `tokenCount` to USD using `tokenCount * COST_PER_TOKEN` and compares against `SOFT_CAP_USD` and `HARD_CAP_USD`. This helper is called at the top of each of the four internal AI actions immediately after fetching the case document and before any Anthropic API call.

```typescript
// Usage inside each AI internalAction (not exported):
import { ERROR_CODES } from '../lib/auth';
import { checkCostCap } from '../lib/pricing';
import * as stateMachine from '../lib/stateMachine';

const caseDoc = await ctx.db.get(caseId);
const capStatus = checkCostCap(caseDoc.tokenCount ?? 0);
if (capStatus === 'hard_cap') {
  await ctx.runMutation(internal.cases.setCostLimited, { caseId });
  return;
}
if (capStatus === 'soft_cap') { /* degrade or early-return, see below */ }
```

**Soft-cap behaviour (≥$2, <$10):** Only `generateCoachResponse` degrades — it posts a templated boilerplate message to `jointMessages` ("I'm limited right now — consider summarizing where you are and whether you've reached agreement.") and returns without calling Anthropic. The other three AI actions (`generateAIResponse`, `synthesis/generate`, `generateResponse`) return early without modifying message state. The `cases.status` is NOT changed at the soft cap — the case remains active.

**Hard-cap behaviour (≥$10):** All four AI actions call `ctx.runMutation(internal.cases.setCostLimited, { caseId })` and return immediately. The `setCostLimited` internal mutation (see Signatures below) reads the current `cases.status`, calls `stateMachine.hitBudgetCap(currentStatus)` to validate the JOINT_ACTIVE → COST_LIMITED transition, catches any CONFLICT thrown by the state machine, and then patches the row. No Anthropic call is made. The UI (T27) renders a banner and hides Draft Coach; parties can still exchange plain messages via `jointChat/sendUserMessage`.

**429 handling:** Wrap the Anthropic streaming call in a try/catch. On a 429 response, wait exactly 2000ms (`await new Promise(r => setTimeout(r, 2000))`), then retry once. If the second attempt also throws a 429, catch it and throw `new ConvexError({ code: ERROR_CODES.RATE_LIMITED, message: 'Rate limit hit after retry', httpStatus: 429 })`. Do NOT use exponential backoff or recursion.

**Timeout handling (>30s):** Use `Promise.race` with a local timeout helper:

```typescript
const timeoutPromise = <T>(ms: number): Promise<T> =>
  new Promise((_, reject) => setTimeout(() => reject(new Error('timeout')), ms));

const result = await Promise.race([anthropicCall, timeoutPromise(30_000)]);
```

If the timeout fires first, set the in-flight message row to `status: 'ERROR'` via `ctx.runMutation(internal.jointChat.appendMessage, { ..., status: 'ERROR' })` and throw `new ConvexError({ code: ERROR_CODES.AI_ERROR, message: 'Response timeout', httpStatus: 502 })`. The UI message bubble for `status: 'ERROR'` renders a Retry button (built in T19a).

**Token accumulation** was specified in T12–T15 acceptance criteria. T17 does not change how tokens are counted — it only adds enforcement on top of the existing `cases.tokenCount` value.

**CLAUDE_MOCK guard:** When `process.env.CLAUDE_MOCK === 'true'`, skip all cap and retry logic — return immediately with canned responses as defined in T12. The cap check must be gated behind `if (!process.env.CLAUDE_MOCK)`.

The `COST_LIMITED` status transition belongs to the state machine (`convex/lib/stateMachine.ts`, T8). T8 already exports `hitBudgetCap(status: CaseStatus): CaseStatus` for the JOINT_ACTIVE → COST_LIMITED transition. The `internal.cases.setCostLimited` internal mutation (added by T17 to `convex/cases.ts`) wraps this: it fetches the case, calls `stateMachine.hitBudgetCap(caseDoc.status)`, catches a plain `{ code: "CONFLICT" }` throw and re-throws as `ConvexError` via T4's error codes, then patches the status. Direct callers in the four AI actions do not need to import the state machine themselves.

Error codes `RATE_LIMITED` and `AI_ERROR` are imported from T4's constants: `import { ERROR_CODES } from '../lib/auth'`.

```typescript
// convex/lib/pricing.ts additions (alongside T4's existing exports)
export const COST_PER_TOKEN: number; // blended rate — use tokenCountToUsd(i, o) when split counts are available
export function checkCostCap(tokenCount: number): 'ok' | 'soft_cap' | 'hard_cap';
// formula: tokenCount * COST_PER_TOKEN >= HARD_CAP_USD → 'hard_cap'
//          tokenCount * COST_PER_TOKEN >= SOFT_CAP_USD → 'soft_cap'
//          otherwise → 'ok'

// convex/cases.ts — new internal mutation
import * as stateMachine from './lib/stateMachine';
import { ERROR_CODES } from './lib/auth';

export const setCostLimited = internalMutation({
  args: { caseId: v.id("cases") },
  handler: async (ctx, { caseId }): Promise<void> => {
    const caseDoc = await ctx.db.get(caseId);
    if (!caseDoc) throw new ConvexError({ code: ERROR_CODES.NOT_FOUND, message: 'Case not found', httpStatus: 404 });
    let newStatus: string;
    try {
      newStatus = stateMachine.hitBudgetCap(caseDoc.status);
    } catch (e) {
      const err = e as { code: string; message: string };
      throw new ConvexError({ code: ERROR_CODES.CONFLICT, message: err.message, httpStatus: 409 });
    }
    await ctx.db.patch(caseId, { status: newStatus as "COST_LIMITED" });
  },
});

// Usage inside each AI internalAction (not exported):
import { ERROR_CODES } from '../lib/auth';
import { checkCostCap } from '../lib/pricing';
const capStatus = checkCostCap(caseDoc.tokenCount ?? 0);
if (capStatus === 'hard_cap') { await ctx.runMutation(internal.cases.setCostLimited, { caseId }); return; }
if (capStatus === 'soft_cap') { /* degrade or early-return */ }
```

## Stack Practices

- Cap check at the top of each AI action using `checkCostCap` from `convex/lib/pricing.ts`; always fetch `caseDoc` before the check so `tokenCount` is current.
- Import error codes: `import { ERROR_CODES } from '../lib/auth'` — never define raw string literals inline.
- Retry: fixed 2s delay via `await new Promise(r => setTimeout(r, 2000))`; not exponential. Maximum one retry — no loop.
- Timeout: wrap Anthropic call in `Promise.race` with a local `timeoutPromise(30_000)` helper; do NOT use `AbortController` or vendor timeout options unless the Anthropic SDK exposes them cleanly.
- Anti-pattern: do NOT let AI errors bubble up as unhandled Convex action failures — every throw site must produce a `ConvexError` with a typed code.
- `setCostLimited` delegates the transition to `stateMachine.hitBudgetCap` and re-throws the state machine's plain `{ code: "CONFLICT" }` as a `ConvexError` — no direct `ctx.db.patch({ status: "COST_LIMITED" })` outside of `setCostLimited`.

## Design Practices

- **Pure `checkCostCap(tokenCount)` over a DB-reading helper.** The function takes a number, not `(ctx, caseId)`, so it is importable by Vitest without any Convex runtime mock. The caller is responsible for fetching `caseDoc.tokenCount` before calling it — this keeps the state machine layer (T8) and the enforcement layer (T17) both side-effect free and independently testable.
- **Blended `COST_PER_TOKEN` alongside T4's split rates.** T4 defines separate `COST_PER_INPUT_TOKEN` / `COST_PER_OUTPUT_TOKEN` for precise billing in `tokenCountToUsd(i, o)`. T17 adds `COST_PER_TOKEN` as a blended average because `cases.tokenCount` is a single accumulated field (T2 schema). If T12–T15 ever split the field, callers should migrate to `tokenCountToUsd`; the blended constant is explicitly a simplification for v1.
- **Fixed 2 s retry delay.** Anthropic 429s are typically capacity-related, not load-shedding cascades, so a single fixed delay is proportionate. Exponential backoff would require state across action invocations, which Convex actions do not naturally support; the YAGNI tradeoff favours the simpler approach in v1.
