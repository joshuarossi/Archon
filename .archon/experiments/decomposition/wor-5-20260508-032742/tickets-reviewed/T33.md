# T33 ‚Äî Accessibility pass (WCAG 2.1 AA, keyboard navigation, screen reader, reduced motion)

Conflict Coach must be usable by every party in a conflict, regardless of disability or assistive technology. This task performs a systematic accessibility audit and remediation across all 12+ routes built in T20‚ÄìT32, ensuring the product meets WCAG 2.1 AA for color contrast, keyboard navigation, screen reader announcements, and motion safety. Unlike typical greenfield accessibility work, this pass audits a fully built product and closes gaps before E2E testing (T34) and CI (T36) lock the interface.

## Acceptance Criteria

- Exactly one `<main>` per page; `<nav>` and `<aside>` landmarks present where appropriate
- Chat regions: `role='log'`, `aria-live='polite'`
- Streaming messages announce 'Coach is replying' once per response
- All modals trap focus; Esc closes; focus returns to trigger
- Route change moves focus to h1
- Draft Coach panel: focus moves to textarea on open
- `prefers-reduced-motion`: all animations disabled
- All text ‚â• 4.5:1 contrast in light and dark mode
- Keyboard-only critical path test passable
- All buttons have visible text or aria-label

## Implementation Notes

T33 is a cross-cutting remediation task, not a feature. Every route (landing, login, invite, dashboard, case creation, case detail, private coaching, ready-for-joint, joint chat, draft coach, closure, solo mode, profile, admin templates, audit log) must be reviewed.

**axe-core integration:** Install `@axe-core/react` as a dev dependency (`npm install -D @axe-core/react`). Add it to `main.tsx` under a `process.env.NODE_ENV === 'development'` guard using the **React 18‚Äìcompatible** pattern below (see Signatures). The old React 16/17 pattern using `ReactDOM.render` must NOT be used ‚Äî React 18 dropped `ReactDOM.render`. Run it against every page in dev to produce a machine-readable violations list before manually remediating.

**For automated unit/integration tests:** Install `jest-axe` (`npm install -D jest-axe @types/jest-axe`) and use `axe` + `toHaveNoViolations` from `jest-axe`. Import `{ axe, toHaveNoViolations } from 'jest-axe'` and call `expect.extend(toHaveNoViolations)` in test setup. This is the test-time axe runner distinct from the dev-mode `@axe-core/react` integration.

**Focus management pattern:** React Router v6 does not automatically move focus on navigation. Install a `useLocation` subscription in `App.tsx` that calls `document.querySelector('h1')?.focus()` after each route transition. Every page `<h1>` must have `tabIndex={-1}` so it receives programmatic focus without appearing in the tab order.

**Modal / Sheet focus traps:** shadcn/ui `Dialog` and `Sheet` use Radix UI `FocusScope` internally ‚Äî do not reimplement. Verify correct trigger-to-focus-return behavior for: PrivacyBanner dialog, MarkComplete `AlertDialog`, CrisisCard (not a modal ‚Äî no focus trap), `DraftCoachPanel` (Sheet), closure modal, version comparison modal, audit log drawer. The Esc key must close each and return focus to the trigger button.

**Chat announcements (role='log'):** `ChatWindow` must have `role='log' aria-live='polite' aria-relevant='additions'`. The STREAMING Coach bubble adds `role='status' aria-label='Coach is replying'` to announce the reply once; when the message reaches COMPLETE, remove `role='status'`. User-authored messages must NOT be announced (they originate from the user).

**Draft Coach panel (T28 aside):** The panel is `<aside>` (complementary content), not a `Dialog`. It must NOT globally trap focus ‚Äî users should be able to tab back to the joint chat behind it. When the panel opens, `useEffect` programmatically focuses the draft `<textarea>`. Esc closes and returns focus to the joint-chat `MessageInput`.

**Color contrast audit:** Check every `--text-*` / `--bg-*` combination defined in T18. Critical pairs: `--text-secondary` (#5C5952) on `--bg-canvas` (#FAF8F5), `--text-tertiary` (#8A8680) on `--bg-surface` (#FFFFFF), `--accent` (#6B8E7F) on `--bg-surface`, `--danger` (#B5594D) on `--bg-canvas`. Use the `are-you-ok` CLI (`npm install -D are-you-ok`) or browser DevTools contrast picker. Both light and dark mode palettes must pass. Any token failing 4.5:1 for normal text (or 3:1 for large text ‚â•18px or ‚â•14px bold) must be adjusted.

**Focus ring:** `src/index.css` must include a global `:focus-visible { outline: 2px solid var(--accent); outline-offset: 2px; }` rule. `outline: none` is only permissible on elements that supply a custom `:focus-visible` replacement.

**prefers-reduced-motion:** T18 set transition durations to 0ms under the `@media (prefers-reduced-motion: reduce)` guard. T33 verifies consistent application: message arrival animation, streaming cursor blink, panel slide-in, route crossfade, skeleton shimmer, DraftCoachPanel slide-in. Run the audit with the OS reduce-motion setting active.

**prefers-reduced-motion test limitation:** jsdom does not evaluate CSS `@media` queries, so `getComputedStyle` assertions against `animation-play-state` or `transition-duration` will always reflect the non-reduced values. Unit tests for this AC must use a different approach: mock `window.matchMedia` to return `matches: true` for `(prefers-reduced-motion: reduce)`, then assert that the relevant component applies the correct CSS class or inline style (e.g., `data-reduced-motion="true"` or a class like `motion-safe` that drives the animation). Alternatively, promote this AC to the Playwright E2E spec using `page.emulateMedia({ reducedMotion: 'reduce' })`.

**Keyboard critical path:** Tab through landing ‚Üí login ‚Üí dashboard ‚Üí new case (all 5 steps) ‚Üí invite page ‚Üí private coaching (send a message, mark complete) ‚Üí ready for joint (enter) ‚Üí joint chat (send message, open Draft Coach, send draft, propose closure). Every action must be completable without a mouse.

**Landmark structure:**
- `<main>`: exactly one per page, wrapping primary content
- `<nav>`: PhaseHeader navigation (back link + phase label)
- `<aside>`: DraftCoachPanel (complementary content)
- Admin tables: `<th scope='col'>` on all column headers
- Filter input on audit log page: `aria-label='Filter by actor email'`

**Route-change focus test constraint:** Do not wrap `<App />` in a second `<MemoryRouter>` ‚Äî `App` already contains the router from T1. For the route-change focus integration test, extract a bare `<AppRoutes />` component (or equivalent named export) from `App.tsx` and wrap that in `<MemoryRouter>` in the test. This avoids nested-router warnings and unstable state.

**Queries used**

**Invariants**

- Every rendered `<Dialog>` and `<Sheet>` must restore focus to its trigger element on close.
- `role='log'` must appear on exactly one element per chat view (ChatWindow), never on individual messages.
- `role='status'` on a STREAMING bubble must be removed once the message reaches COMPLETE status.
- `prefers-reduced-motion: reduce` must suppress all CSS transition durations and animation `play-state` across the entire app.

- `AppRoutes` (exported from `App.tsx`) must not include any `<BrowserRouter>` wrapper, enabling test-time `<MemoryRouter>` substitution.

## Stack Practices

- `axe-core` audit: run `@axe-core/react` (‚â•4.9.0, React 18 compatible) in development mode via `main.tsx` (dev-only). Use `jest-axe` for unit/integration test assertions ‚Äî these are separate packages with separate roles.
- Focus trap: shadcn/ui `Dialog` and `Sheet` via `@radix-ui/react-focus-scope` ‚Äî do NOT reimplement. In tests use `userEvent.setup({ pointerEventsCheck: PointerEventsCheckLevel.Never })` to avoid jsdom pointer-event stubs breaking Radix focus behavior.
- Route change focus: `useLocation` subscription in `App.tsx`; `document.querySelector('h1')?.focus()`. Export `AppRoutes` (no router wrapper) for test isolation.
- `aria-live` for chat: `role='log' aria-live='polite' aria-relevant='additions'` on message list.
- Focus ring: `:focus-visible { outline: 2px solid var(--accent); outline-offset: 2px; }` globally.
- Contrast check: use `are-you-ok` CLI or browser DevTools.
- `prefers-reduced-motion` unit tests: mock `window.matchMedia`; assert CSS class/attribute, not `getComputedStyle` (jsdom does not evaluate `@media` queries).
- Anti-pattern: do NOT use `outline: none` without a `:focus-visible` replacement.

## Design Practices

- Color contrast audit: check every `--text-*` on every `--bg-*` combination using design doc ¬ß2.1 tokens. All must meet WCAG 2.1 AA 4.5:1 for normal text, 3:1 for large text (18px+ or 14px+ bold). Check both light and dark mode. Critical pairs to verify: `--text-secondary` on `--bg-canvas`, `--text-tertiary` on `--bg-surface`, `--accent` on `--bg-surface`, `--danger` on `--bg-canvas`.
- Focus ring: global `:focus-visible` rule using `var(--accent)` (#6B8E7F) 2px solid, 2px offset. Apply `outline-none` ONLY to elements that provide a custom `:focus-visible` replacement (e.g., custom active state styling). Never remove outline entirely.
- `prefers-reduced-motion`: T18 sets all transition durations to 0ms and pauses CSS animations. T33 verifies this is consistently applied: message arrival animation, streaming cursor blink, panel slide-in, route crossfade, skeleton shimmer ‚Äî ALL must cease. Use `@media (prefers-reduced-motion: reduce)` guards.
- Landmark structure (design doc ¬ß9.5): exactly one `<main>` per page (not per route component). `<nav>` for PhaseHeader navigation. `<aside>` for Draft Coach panel (it is complementary content). Verify with `@axe-core/react` violations list.
- Chat accessibility (design doc ¬ß9.5): `role='log' aria-live='polite' aria-relevant='additions'` on ChatWindow container. STREAMING Coach bubble: `role='status' aria-label='Coach is replying'` ‚Äî announced once. COMPLETE: remove `role='status'`. New user messages are NOT announced (they are authored by the user, not new arrivals).
- Modal / Sheet accessibility: shadcn/ui `Dialog` and `Sheet` use Radix UI `FocusScope` and `aria-modal='true'`. Esc must close. Focus must return to the trigger element (button that opened it) on close. Verify this for: PrivacyBanner dialog, MarkComplete `AlertDialog`, CrisisCard (not a modal ‚Äî no focus trap), DraftCoachPanel (Sheet), closure modal, version comparison modal, audit log drawer.
- Route change focus: after each React Router navigation, programmatically focus the page's h1 (`tabIndex={-1}` on h1). This is critical for screen reader users who would otherwise be lost at the previous scroll position.
- Keyboard critical path (design doc ¬ß9.5): Tab through landing ‚Üí login ‚Üí dashboard ‚Üí new case (all 5 steps) ‚Üí invite page ‚Üí private coaching (send message, mark complete) ‚Üí ready for joint (enter) ‚Üí joint chat (send message, open Draft Coach, send draft, propose closure). All actions completable without mouse.
