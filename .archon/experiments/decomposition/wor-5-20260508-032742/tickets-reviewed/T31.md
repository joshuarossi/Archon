# Profile Page and Logout

The `/profile` route gives authenticated users one place to manage their account: update their display name and sign out. It is a minimal, low-ceremony screen intentionally — Conflict Coach has no settings panel or account dashboard beyond these two actions. The top-nav user avatar (visible on all case-less pages like Dashboard) provides the entry point. This task also ensures the global navigation renders correctly for logged-in users on case-less pages.

## Acceptance Criteria

- `/profile`: read-only email, editable displayName
- Saving calls users/updateDisplayName; success toast
- 'Sign out' calls Convex Auth signOut; redirects to /
- Logged-out access to /profile redirects to /login
- Profile accessible from top nav

## Implementation Notes

This is a small, focused route. Keep the component lean — resist the impulse to add settings panels or conditional content blocks.

**Top nav entry point:** The logged-in case-less top nav (design doc §3.2) shows `[Dashboard] [◉ You]`. The `◉ You` element is a 32×32 user avatar (initials, `--accent` background — the "you" avatar) that links to `/profile`. This avatar is the `avatar-you` variant from the design system (§6.5). Wire it into whatever top-nav layout component is shared across Dashboard and Profile.

**Data requirements:** `useQuery(api.users.getCurrentUser)` returns the current user's email and displayName. An `updateDisplayName` mutation on `convex/users.ts` needs to exist (or be created here). It receives `{ displayName: string }`, calls `requireAuth(ctx)`, and patches the user row.

**Route guard:** Wrap `/profile` in the same auth guard pattern used by `/dashboard`. If `useConvexAuth().isAuthenticated === false`, redirect to `/login`. Use React Router `<Navigate replace>` inside the route component or a `RequireAuth` wrapper.

**Display name mutation UX:** Use `useMutation(api.users.updateDisplayName)` with optimistic UI via the `optimisticUpdate` option. Disable the Save button until the local input value differs from the fetched `displayName`. On success, fire the shadcn/ui toast. On error, show an inline error below the input with `role="alert"`.

**Sign-out:** Call `const { signOut } = useAuthActions(); await signOut();` then `navigate('/')`. The sign-out button should be visually subordinate — `variant="ghost"` or `variant="outline"` — because signing out is routine, not dangerous.

**Non-editable fields:** Email is shown read-only. Do NOT render it as a form input (even `readOnly`). Use a static `<p>` or `<div>` styled to look like an input but with `--bg-surface-subtle` background to signal non-editability. Do not expose `role`, `createdAt`, or `_id` fields. Note: the Design Practices below say "read-only Input" to describe the visual appearance only — the DOM element must be `<p>` or `<div>`, not `<input>`.

**Auth source:** `useAuthActions` comes from `@convex-dev/auth/react`. The `ConvexAuthProvider` wrapping in `main.tsx` (established in T3) makes this available everywhere.

## Stack Practices

- Sign out: `const { signOut } = useAuthActions(); await signOut()` then navigate('/').
- Display name mutation: useMutation with optimistic UI (`optimisticUpdate` option).
- Current user: useQuery(api.users.getCurrentUser).
- Anti-pattern: do NOT expose non-editable fields (email, role, createdAt) as form fields.
- Anti-pattern: do NOT render email as `<input readOnly>` — use `<p>` or `<div>` styled to look like an input.

## Design Practices

- Page layout: centered card, max-width ~480px, bg-surface shadow-1, 32px padding. Background --bg-canvas. Simple single-column form — not a settings panel with sidebar.
- Email field: visually styled like an Input (border, bg-surface-subtle, same height/padding) but rendered as a `<p>` or `<div>`, NOT as `<input>`. `--text-secondary` text color. Label 'Email address' in 14px 500 weight. Use `aria-readonly='true'` on the element.
- Display name Input: border --border-strong, 15px Inter, --text-primary. Save button: primary --accent, 'Save changes'. Disabled (and `aria-disabled='true'`) until name has changed from current value OR mutation is in-flight.
- Success toast: shadcn/ui toast, neutral (not green celebration), 'Display name updated.' --text-primary. Auto-dismisses in 3s. Announced via shadcn/ui's built-in aria-live region.
- 'Sign out' button: shadcn/ui Button variant='ghost' or outline, --text-secondary, at bottom of card with top border divider --border-default. NOT a danger button — signing out is routine.
- Top nav link to /profile: user avatar circle (32px, initials, --accent bg) in top-right corner of the global nav. Clicking opens profile route.
- Accessibility: form has accessible `<label>` for each field. Read-only email uses `aria-readonly='true'`. Save button disabled state: `aria-disabled='true'`. Success toast announced via shadcn/ui's built-in aria-live.
