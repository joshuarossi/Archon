# AI Synthesis Backend (generate action, privacy filter, state transition)

Once both parties have marked private coaching complete, the system must bridge their separate experiences into a shared starting point for the joint session — without ever surfacing one party's raw words to the other. This ticket implements the `synthesis/generate` internal action that reads both parties' private messages, calls Claude Sonnet to produce two independent synthesis texts, runs the privacy filter on each, and writes the results to the database while advancing the case to `READY_FOR_JOINT`. This is the only moment in the product where both parties' private content is visible in the same execution context, making correct isolation and filter enforcement load-bearing for the privacy guarantee.

## Acceptance Criteria

- Calls claude-sonnet-4-5; validates { forInitiator, forInvitee } JSON
- Runs filterResponse on each half; retries up to 2x on failure
- On success: writes synthesisText to both partyStates; sets case READY_FOR_JOINT
- Idempotent: if synthesisText already populated, returns without re-generating

## Implementation Notes

`synthesis/generate` must be declared as an `internalAction` — it is only ever called from `privateCoaching/markComplete` (via `ctx.scheduler.runAfter` or `ctx.runAction`) when both parties' `privateCoachingCompletedAt` fields are set. It must never be a public action; the client has no direct path to invoke it.

**Reading both parties' context.** The action calls `ctx.runQuery(internal.synthesis._getBothPartiesContext, { caseId })` which returns each party's full private message history and their form fields. Both are passed to `assemblePrompt({ role: "SYNTHESIS", ... })` from `convex/lib/prompts.ts` (T6). The assembled prompt includes the verbatim instruction: *"You have access to both parties' private content for context. In your outputs, NEVER quote, closely paraphrase, or otherwise surface the other party's raw words."*

**Calling Claude.** Use the `@anthropic-ai/sdk` Anthropic client with a non-streaming call (`messages.create` without `stream: true`). Model: `claude-sonnet-4-5`. The prompt instructs the model to respond with ONLY the JSON object — no surrounding prose. Parse the response text as JSON; if JSON.parse throws, treat it as a format failure and retry.

**Privacy filter.** After parsing, call `filterResponse(forInitiator, inviteePrivateMessages)` and `filterResponse(forInvitee, initiatorPrivateMessages)` from `convex/lib/privacyFilter.ts` (T5). Each returns `{ passed, matchedSubstring? }`. Both must pass. If either fails, increment the retry counter and regenerate — do not mix a passing half with a failing half across retries.

**Retry loop.** Use a `for (let attempt = 0; attempt < 3; attempt++)` loop (0, 1, 2 = up to 3 attempts total, 2 retries after the first). Never recurse. On all three failures, call `ctx.runMutation(internal.synthesis._persistSynthesis, { caseId, forInitiator: FALLBACK_TEXT, forInvitee: FALLBACK_TEXT, isFallback: true })` which atomically writes the fallback text and inserts an `auditLog` row with `action: "SYNTHESIS_FALLBACK"` (see Persistence below).

**CLAUDE_MOCK guard.** If `process.env.CLAUDE_MOCK === 'true'`, skip the Anthropic call and return canned synthesis strings immediately, same as T12.

**Persistence.** Write results via `ctx.runMutation(internal.synthesis._persistSynthesis, { caseId, forInitiator, forInvitee, isFallback })`. This internal mutation atomically:
1. Writes `synthesisText` + `synthesisGeneratedAt` to both `partyStates` rows.
2. Sets `cases.status = "READY_FOR_JOINT"` (skipped when `isFallback: true` — the case stays at its current status so an admin can review before the joint session begins).
3. When `isFallback: true`, inserts an `auditLog` row: `{ caseId, action: "SYNTHESIS_FALLBACK", metadata: {} }`.

Keeping all writes in a single mutation preserves Convex's transactional guarantee — if the write fails partway, neither partyState is updated.

**Idempotency.** At the top of `synthesis/generate`, call `ctx.runQuery(internal.synthesis._isSynthesisComplete, { caseId })`. If it returns `true`, return immediately without calling Claude. This prevents double-billing if the action is accidentally re-scheduled.

**File locations.** All synthesis functions live in `convex/synthesis.ts`. The internal persistence mutation is `internal.synthesis._persistSynthesis`.

## Stack Practices

- `synthesis/generate` is an `internalAction` only — never a public action or mutation.
- Retry loop: `for (let attempt = 0; attempt < 3; attempt++)` — never use recursion or `while(true)`.
- Anti-pattern: do NOT call the Anthropic API inside a Convex mutation — mutations cannot make network calls.
- Anti-pattern: do NOT call `ctx.db` directly inside an `internalAction` — actions must use `ctx.runMutation` / `ctx.runQuery` for all database access.
- Both the auditLog write and the synthesisText writes must be in the same `_persistSynthesis` mutation to guarantee atomicity.
- Extract the fallback string as a named module-level constant (`FALLBACK_TEXT`) so tests can reference it without string literals.

## Design Practices

- **Non-streaming for synthesis**: The synthesis call uses a one-shot `messages.create` (no streaming) per tech spec TQ3. This simplifies the retry loop: a streaming failure mid-response would require buffering and partial-output detection; a non-streaming failure is a clean exception that triggers the next attempt.
- **`internalAction` not `internalMutation`**: The Anthropic API is a network call. Convex mutations execute in a single synchronous database transaction and cannot make network requests — only actions can. This is why `generate` must be an action and `_persistSynthesis` must be a separate mutation that the action dispatches.
- **Atomic persistence via a single mutation**: Writing `synthesisText` for initiator, `synthesisText` for invitee, and `cases.status` in one `_persistSynthesis` mutation call ensures they are committed atomically. A crash after writing one partyState but before writing the other would leave the case in a corrupt state that no state-machine transition could recover from.
- **Graceful fallback instead of throwing**: If all three synthesis attempts fail, the action writes a generic fallback text and flags the case for admin review rather than throwing an unhandled error. Throwing would leave the case stuck at `PRIVATE_COACHING_COMPLETE` indefinitely with no user-visible indication; the fallback text allows both parties to proceed to the joint session while the admin investigates.
