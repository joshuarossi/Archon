# T19b — Secondary UI Components

This task builds the remaining shared UI primitives that every feature view composes from. Where T19a covers the core chat interaction components, T19b provides the surrounding context: privacy indicators, party identity, phase orientation, status feedback, crisis safety, and loading states. These components are consumed by every route from T20 onward and must be correct and accessible before feature-level views are assembled.

## Acceptance Criteria

- PrivacyBanner: lock icon + Dialog explaining privacy; screen reader text
- PartyAvatar: role-colored circles; shows first initial
- CoachIntervention: coach-accent border, ⟡ glyph, accent-subtle background
- PhaseHeader: current phase label + Back to Dashboard link
- StatusPill: ●○◐◼ with correct color mapping
- CrisisCard: appears on crisis keywords; dismissable; links to crisis resources
- Dashboard skeleton: 3 case row skeletons
- Case detail skeleton: phase-appropriate layout

## Implementation Notes

All components in this task are presentational or lightly stateful. They receive props and render; they do not own data-fetching. The design system tokens established in T18 (`--private-tint`, `--coach-accent`, `--danger-subtle`, `--accent-subtle`, `--border-default`, etc.) must be referenced via CSS custom properties or the Tailwind config mappings — never hardcoded hex values.

**PrivacyBanner** is a persistent strip at the top of every private-scoped view (private coaching, Draft Coach panel). It must always be visible — not collapsible — because privacy legibility is a first-class design principle (design doc §1.3). The banner opens a shadcn/ui Dialog on click to provide a full explanation. Include `<span className="sr-only">` with that full text so screen reader users receive it without having to open the Dialog.

**PartyAvatar** receives a `role` prop (`INITIATOR | INVITEE | COACH`) and optionally a `name` string. Colors are role-fixed per design doc §4.9: Initiator = `#BFDBFE` soft blue, Invitee = `#FECDD3` soft rose, Coach = `--coach-accent` at 30% opacity background with `--coach-accent` foreground text. The initial is derived from `name[0]` uppercased; fall back to the role abbreviation if `name` is absent.

**CoachIntervention** is not a standalone message type — it is a wrapper applied around an existing MessageBubble (from T19a) to indicate a coach-generated, intervention-triggered message. It renders a styled `<div>` that wraps `children` and adds a 2px left border in `--coach-accent` and swaps the bubble background to `--accent-subtle`. When `isIntervention` is true (inflammatory-content response), the background becomes a `--warning`-tinted variant instead. The ⟡ glyph (U+27C1 or inline SVG) is prepended to the message text via a `<span>` inside the wrapper — it does not reach into `children`.

**PhaseHeader** is used as a layout-level persistent header in CaseDetail.tsx and all its sub-routes (T24). It receives `caseName`, `phaseName`, and `isSolo` as props. On `isSolo === true`, it renders a 'Solo' badge and the PartyToggle (implemented in T30 — render a `{children}` slot or `toggleSlot` prop so the header doesn't import T30 directly). It must be a `<nav>` landmark.

**StatusPill** maps case status strings to a glyph + color + label. Extract the mapping into a `getStatusConfig(status)` helper (do not inline it in JSX) so it can be unit-tested and reused by the Dashboard (T22).

**CrisisCard** manages its own `dismissed` state via `useState`. It accepts `isVisible` (from the parent detecting crisis keywords in messages) and `onDismiss` callback. When `isVisible` is false, the component renders `null` — it does not render a hidden or collapsed element. Use a `key` prop at the call site (T25) to reset dismissal when a new user message is sent — the card itself should not know this; it only reads its own internal state.

**Skeleton components**: use shadcn/ui `Skeleton` primitive. Show skeletons after a 300ms `useEffect + setTimeout` delay to avoid a flash on fast loads. Dashboard skeleton is 3 rows at ~72px height. Case detail skeleton is phase-appropriate (full-height column placeholder for chat views, centered card for non-chat phases). Before the 300ms timer fires the component returns `null` (not a placeholder div) — this avoids skeleton flash, not layout reservation.

All components live under `src/components/shared/`. Use named exports. Do not use `dangerouslySetInnerHTML`. All shadcn/ui Dialog and Sheet usage relies on Radix UI's built-in focus trapping — do not reimplement it.

type CaseStatus =

  explanationText: string; // Full privacy explanation shown in Dialog + sr-only

**Queries used**

None — all components are purely presentational.

**Invariants**

- `getStatusConfig` must return a value for every member of `CaseStatus`; it must throw (or return a fallback with a dev warning) for unknown strings.
- `PartyAvatar` always renders exactly 32×32px regardless of `name` length.
- `PrivacyBanner` never renders as collapsed or hidden — `display: none` is not a valid state.
- `PrivacyBanner` trigger button must have `aria-label="Learn more about your privacy"`.
- `PartyAvatar` must have `aria-label="[name] avatar"` when `name` is provided, and `aria-label="[role] avatar"` (role abbreviation, e.g. `"Coach avatar"`) when `name` is absent.
- `CrisisCard` renders `null` when `isVisible` is false — it does not render a hidden or collapsed element.
- `CrisisCard` does not call `onDismiss` until the user explicitly clicks the dismiss button — it does not auto-dismiss on `isVisible` becoming false.
- `CrisisCard` dismiss button must have `aria-label="Dismiss crisis card"`.
- `CoachIntervention` is a wrapper div — it does not render any bubble frame of its own; it wraps whatever is passed as `children`. The ⟡ glyph appears inside the wrapper via a sibling `<span>`, not injected into `children`.
- `PhaseHeader` root element is `<nav>` — it is a navigation landmark.
- `StatusPill` renders both the glyph and the label (not glyph-only) at all viewport sizes.
- All interactive elements (PrivacyBanner trigger, CrisisCard dismiss, PhaseHeader back link) must have a visible focus ring: `2px solid var(--accent)`, `outline-offset: 2px`.
- Skeleton components return `null` before the 300ms timer fires (no placeholder div), then render shadcn/ui Skeleton primitives.

**Non-goals**

- No data fetching inside any of these components.
- `PhaseHeader` does not implement or import `PartyToggle` (T30) — it only renders whatever is passed via `toggleSlot`.
- `CrisisCard` does not detect crisis keywords — detection is the caller's responsibility.
- Typing indicator ('Jordan is typing…') is deferred to v1.1 per planning assumptions — `PhaseHeader` must reserve a 20px visual row below `MessageInput` (a CSS rule, not a component) but no logic is implemented here.
- No animation or transition logic beyond the skeleton delay — motion tokens are defined in T18.

## Stack Practices

- shadcn/ui Dialog for PrivacyBanner modal; Sheet for My Guidance panel and mobile Draft Coach.
- Skeleton components: use shadcn/ui Skeleton primitive with 300ms delay before showing.
- CrisisCard: standalone component with isVisible and onDismiss props, local useState for dismissed.
- Anti-pattern: do NOT use raw `<dialog>` HTML elements — use shadcn/ui Dialog (Radix UI).
- Screen reader text: use sr-only Tailwind class.

## Design Practices

- PrivacyBanner: persistent strip at top of private-scoped views. Background --private-tint (#F0E9E0). Lucide Lock icon (1.5px stroke, 16px) in --text-secondary. Text: --text-secondary, 14px. Clicking the banner opens a shadcn/ui Dialog (shadow-3 elevation) explaining exactly what is private. Include `<span className='sr-only'>` with full explanation for screen readers.
- PartyAvatar: 32×32px circle (rounded-full). Colors per design doc §4.9: Alex/Initiator = soft blue (#BFDBFE), Jordan/Invitee = soft rose (#FECDD3), Coach = --coach-accent (#8B7AB5) at 30% opacity background with --coach-accent text. First initial, 14px Inter 500 weight, centered. Apply aria-label='[Name] avatar'.
- CoachIntervention message (design doc §4.9): left border 2px solid --coach-accent. Background --accent-subtle. ⟡ glyph rendered before coach message text (Unicode U+27C1 or SVG). isIntervention flag adds --warning (#B58B4D) background tint instead of --accent-subtle to signal inflammatory-content response.
- PhaseHeader (design doc §3.2): full-width top bar, bg-surface, border-bottom --border-default. Left: '← Back to Dashboard' (ghost link, --text-secondary). Center: 'Case with [Name] • [Phase Label]' in 500 weight, --text-primary. Solo cases: append 'Solo' badge — text-xs, bg-coach-accent/20, text-coach-accent, px-1.5 py-0.5 rounded.
- StatusPill (design doc §4.4): ● green (--accent #6B8E7F) = Your Turn; ○ gray (--text-secondary) = Waiting; ◐ amber (--warning #B58B4D) = Ready for Joint; ◼ neutral (--text-tertiary) = Closed. Render glyph + short label. 12px --text-secondary typography.
- CrisisCard: bg-danger-subtle (#F2DCD8) border border-danger-subtle, 8px radius, 16px padding. Lucide AlertTriangle in --danger. Heading 'Need immediate help?' in --danger 500 weight. Dismissable via ✕ icon button with aria-label='Dismiss crisis card'. Links styled as --accent underline.
- Skeleton components: use shadcn/ui Skeleton (animated bg-surface-subtle shimmer). Dashboard row skeletons: 3 rows at ~72px height each. Show after 300ms delay (useEffect + setTimeout) to avoid skeleton flash on fast loads.
- Accessibility (all T19b components): shadcn/ui Dialog and Sheet handle focus trapping via Radix UI — do NOT reimplement. All interactive elements need visible focus ring: 2px solid --accent, offset 2px. PrivacyBanner lock icon button: aria-label='Learn more about your privacy'.
