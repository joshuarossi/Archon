# Draft Coach Panel (Side Panel / Bottom Sheet, Draft Ready Card, Send Gate)

The Draft Coach panel is a private, AI-assisted composition surface that lives alongside the joint chat session. One of Conflict Coach's core product bets (Epic §1.3) is that AI-guided drafting, subject to explicit human approval before sending, produces better joint-chat messages and keeps the conversation productive. This ticket implements `DraftCoachPanel.tsx`: a 420px right panel on desktop (a full-screen bottom Sheet on mobile), the Draft Coach conversation, the `DraftReadyCard` gate that presents a finished draft for the user's approval, and all four post-draft actions (Send / Edit / Refine / Discard). This component depends on the Draft Coach backend (T15) and the joint chat view (T27) that hosts it.

## Acceptance Criteria

- Opens on '✨ Draft with Coach' click, calls draftCoach/startSession
- Private label in panel
- ChatWindow connected to draftCoach/sendMessage
- 'Draft it for me' posts canonical message
- DraftReadyCard when finalDraft populated: Send/Edit/Refine/Discard
- 'Send this message': calls sendFinalDraft; panel closes; message in joint chat
- 'Edit before sending': transfers to main input; does NOT auto-send
- Esc closes panel; focus returns to joint-chat input
- Focus moves to draft textarea when panel opens

## Implementation Notes

`DraftCoachPanel.tsx` is rendered as a child of `JointChatView.tsx` (T27). The parent owns the open/closed toggle state and passes a `jointChatInputRef` prop so focus can be returned on panel close or Esc keydown. The backend mutations powering the panel are `draftCoach/startSession`, `draftCoach/sendMessage`, `draftCoach/sendFinalDraft`, and `draftCoach/discardSession` (all implemented in T15). `draftCoach/generateResponse` is a backend-only internal action triggered server-side after `sendMessage`; the client does NOT call it via `useMutation`.

The panel's conversation is a private counterpart to the joint chat — Jordan cannot see any exchange inside the Draft Coach. Only the final drafted message, once the user presses "Send this message", crosses into the joint chat via `sendFinalDraft`, which internally calls `jointChat/sendUserMessage`. This privacy boundary must be respected at the component level by not mixing Draft Coach messages into the joint message list.

**Desktop layout:** The panel is an absolutely/fixed positioned 420px-wide column on the right side of the viewport, rendering at full height. It does not push the message list — it overlaps it. Use `translateX(100% → 0)` with a `200ms ease-out` CSS transition for the slide-in. Apply `shadow-3` (`0 12px 32px rgba(0,0,0,0.10)`) per the design doc elevation system. Z-index must be above the message list but below any Dialog modals.

**Mobile layout:** Use shadcn/ui `Sheet` with `side='bottom'` and `className='h-screen'`. This replaces the desktop slide-in; do not render both simultaneously. Detect mobile with `useMediaQuery('(max-width: 768px)')` — import the hook from `src/hooks/useMediaQuery.ts`, which is created in T27.

**Session lifecycle:** Clicking "✨ Draft with Coach" calls `draftCoach/startSession`. If a session already exists for this case, the backend is idempotent. The panel subscribes to the session's messages via `useQuery(api.draftCoach.session, { caseId })`. When `session.finalDraft` becomes non-null, render `DraftReadyCard` in place of (or below) the conversation.

**"Draft it for me" button:** Posts a canonical message to the Draft Coach (e.g., `"Please draft a message for me based on what we've discussed."`) via `draftCoach/sendMessage`. It is a `variant='outline'` button — not primary, since drafting is an option not a mandate.

**DraftReadyCard:** Displayed when `session.finalDraft` is populated. It presents the draft text and four actions:
1. "Send this message" (primary `--accent` filled): calls `sendFinalDraft`; button must be disabled while the mutation is in-flight (prevent double-send); on success, closes the panel.
2. "Edit before sending" (secondary outlined): transfers `finalDraft` text to the joint chat `MessageInput` via the `onTransferDraft` state setter prop — does NOT auto-send.
3. "Keep refining with Coach" (ghost): dismisses the `DraftReadyCard`, returns focus to draft textarea.
4. "Discard" (ghost, `--danger` color): calls `draftCoach/discardSession`.

**Focus management:** Use `useRef<HTMLTextAreaElement>` for the draft textarea. In a `useEffect`, when `isOpen` transitions to true, call `textareaRef.current?.focus()`. On Esc keydown (caught on the panel container's `onKeyDown`), close the panel and call `jointChatInputRef.current?.focus()`.

**Architecture note:** `DraftCoachPanel` is `<aside>` — complementary content per the accessibility landmark pattern used in T33. It does NOT trap focus globally (unlike a Dialog) so the user can still interact with the joint chat message list while the panel is open.

### Queries used

- `api.draftCoach.session` — `useQuery(api.draftCoach.session, { caseId })` → `DraftSession | null`; includes `finalDraft: string | null` and `messages: DraftMessage[]`
- `api.draftCoach.startSession` — `useMutation`, called on panel open
- `api.draftCoach.sendMessage` — `useMutation`, called on user message submit and on "Draft it for me"
- `api.draftCoach.sendFinalDraft` — `useMutation`, called on "Send this message"
- `api.draftCoach.discardSession` — `useMutation`, called on "Discard"

Note: `api.draftCoach.generateResponse` is a backend-only internal action; do NOT call it from the client.

### Invariants

- Panel is never open on a closed (`CLOSED_*`) or `COST_LIMITED` case — parent hides it entirely.
- "Send this message" must be disabled while `sendFinalDraft` mutation is in-flight (prevent double-send).
- "Edit before sending" transfers the draft text to the joint chat input but never auto-submits it.
- Focus must move to the draft textarea every time `isOpen` transitions from false to true.
- Esc must close the panel and return focus to `jointChatInputRef` every time, including when `DraftReadyCard` is visible.
- The panel `<aside>` must NOT use `aria-modal='true'` or trap focus globally.
- Mobile `<Sheet>` and desktop slide-in panel must never both render simultaneously.
- Draft textarea must have `aria-label='Message to Draft Coach'`.
- DraftReadyCard "Send this message" button must have `aria-describedby` pointing to the draft text block element's id.

## Stack Practices

- Panel layout: absolute/fixed 420px right panel on desktop; shadcn/ui Sheet side='bottom' className='h-screen' on mobile.
- Mobile detection: import `useMediaQuery` from `src/hooks/useMediaQuery.ts` (created in T27); call `useMediaQuery('(max-width: 768px)')`.
- Focus management: useEffect → textarea focus when isOpen. useRef<HTMLTextAreaElement>.
- Esc close + focus return: onKeyDown on panel container. Pass jointChatInputRef as prop.
- 'Edit before sending': onTransferDraft(finalDraft) state setter prop from JointChatView.
- Anti-pattern: do NOT auto-send when AI produces finalDraft — DraftReadyCard is a gate.

## Design Practices

- Panel appearance (design doc §4.10): desktop — slides in from right (translateX 100% → 0, 200ms ease-out). 420px width, full viewport height, bg-surface, shadow-3 ('0 12px 32px rgba(0,0,0,0.10)'). z-index above message list, below Dialog modals.
- Panel header (design doc §4.10): 'Draft Coach' title (500 weight --text-primary), Sparkles icon (Lucide 18px --coach-accent), Lock icon (Lucide 16px --text-secondary) persistent, ✕ close button (ghost, aria-label='Close Draft Coach').
- Privacy label inside panel: small strip below header, --private-tint background, 13px --text-secondary: 'This is private to you. Jordan can't see what you're discussing here.' Hover on lock icon: tooltip 'Only the final message you send goes to the joint chat.'
- Draft conversation: reuses ChatWindow component with COACH bubbles in --coach-accent tint and USER bubbles in --bg-surface. 16px bubble text for readability.
- 'Draft it for me' button: shadcn/ui Button variant='outline' full-width at bottom of conversation, Sparkles icon prefix. --text-primary, border --border-default. NOT primary — it is an option, not the mandate.
- DraftReadyCard (design doc §4.10): bg-surface border --border-default shadow-1, 16px padding, 8px radius. 'Here's a draft based on what we talked about:' in 14px --text-secondary above the draft text block. Draft text block: bg-surface-subtle, 14px --text-primary, 16px padding, border-radius 8px, border --border-default.
- DraftReadyCard action hierarchy (design doc §4.10): 'Send this message' = primary --accent filled, full-width. 'Edit before sending' = secondary outlined. 'Keep refining with Coach' = ghost. 'Discard' = ghost text-danger. Stack vertically. One primary action (design doc §1.4).
- Mobile Sheet: shadcn/ui Sheet side='bottom' h-screen. Same header/content layout. Slides up from bottom (translateY, 200ms ease-out).
- Accessibility: panel is not a Dialog — it does NOT trap focus globally (user can still interact with joint chat behind it). Draft textarea has aria-label='Message to Draft Coach'. DraftReadyCard 'Send this message' has aria-describedby pointing to the draft text block. Esc key closes and returns focus to joint-chat MessageInput.
