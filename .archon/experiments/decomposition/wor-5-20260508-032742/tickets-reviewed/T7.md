# Transcript Compression Module

AI-powered features across private coaching, synthesis, and joint chat all send conversation history to Claude as context. As cases accumulate messages, those contexts can grow past practical token budgets. This module is the shared solution: when a message array exceeds a token budget it compresses the oldest half into a single `SUMMARY` synthetic message via a Haiku API call, then caches the result by content hash so the same segment is never compressed twice. Every AI action in the codebase will call `compressIfNeeded` before building its API request ŌĆö getting this right and testable in isolation is prerequisite to all four AI actions (T12ŌĆōT15).

## Acceptance Criteria

- AC1: `compressIfNeeded` returns original array when under budget without API call
- AC2: Over-budget: oldest 50% replaced by SUMMARY message
- AC3: Content hash caching: same input = one Haiku API call
- AC4: Token counting uses word-count approximation

## Implementation Notes

The tech spec (┬¦6.4) defines the compression algorithm:
1. Compute total token count of all messages.
2. If under budget, return the array unchanged ŌĆö no API call, no allocation.
3. If over budget, take the oldest 50% of messages.
4. Call Haiku with the prompt: "Summarize this conversation segment in 500 tokens or fewer, preserving facts, decisions, emotional tone, and unresolved threads."
5. Replace those messages with a single synthetic `{ role: 'user', content: 'SUMMARY: <haiku output>' }` message.
6. Cache the result by the content hash of the compressed segment (the oldest-50% slice) so re-compression of an unchanged prefix is free.

The Anthropic client is passed in as a dependency-injection parameter ŌĆö this module does not import from `@anthropic-ai/sdk` directly at module scope, so it remains fully unit-testable with a `vi.fn()` mock. The cache is a module-level `Map<string, string>` keyed on a content hash of the **serialized content of the oldest-50% slice** (i.e., `JSON.stringify(oldestHalf.map(m => m.content))`). The same full messages array always produces the same compressed-segment hash; two full arrays whose oldest-50% slices differ produce different hashes.

Because the hash must work in the Convex runtime (no Node.js `crypto` module), compute it with a `TextEncoder` and a manual djb2-style hash loop over the UTF-8 bytes. djb2 is a best-effort approximation ŌĆö collision probability is negligible for realistic conversation segments but is not mathematically zero; this is acceptable (the consequence of a collision is a stale summary, not data loss).

Token counting is intentionally approximate: `Math.ceil(text.split(/\s+/).filter(Boolean).length * 1.3)`. This is consistent with the prototype approach ŌĆö "nothing clever" is a deliberate design choice per the spec.

The module lives at `convex/lib/transcriptCompression.ts` alongside the other pure utility modules (`privacyFilter.ts`, `prompts.ts`, `stateMachine.ts`). It must not import from any Convex-generated paths (`convex/_generated/server`, etc.) so Vitest can run it without a Convex environment.

The default budgets from the spec are: **60 000 tokens** for user-facing generation (private coaching, joint chat Coach, Draft Coach), **10 000 tokens** for classification tasks (Haiku inflammatory gate). Callers pass the appropriate `budgetTokens` value; this module does not hardcode any default.

Calling `compressIfNeeded` with `budgetTokens = 0` is a programmer error and should throw immediately with a descriptive message.

**Test isolation:** The module-level cache persists for the lifetime of the Vitest worker process. Within the same test file, tests that share the same oldest-50% segment content will share a cache entry ŌĆö this is intentional and what the caching test verifies. Tests that need a fresh cache (e.g., a cache-miss test after a cache-hit test using overlapping fixtures) should use clearly distinct message content, or call the exported `clearCacheForTesting()` helper in `beforeEach`. Export `clearCacheForTesting` as a named function that calls `cache.clear()` ŌĆö it exists solely for test isolation and must not be called in production code.

**Signatures**
```typescript

  };

  messages: Message[],
  budgetTokens: number,
```

**Queries used**
- `anthropic.messages.create` ŌĆö called with `model: 'claude-haiku-4-5-20251001'`, `max_tokens: 600`, and the oldest-50%-of-messages slice. Called at most once per unique content hash of that slice.

**Invariants**
- If `budgetTokens <= 0`, throws `Error('budgetTokens must be > 0')` before any other work.
- If total estimated tokens Ōēż `budgetTokens`, `anthropic.messages.create` is never called and the returned array is the same reference (or structurally equal) to `messages`.
- After compression, the returned array always begins with a `{ role: 'user', content: 'SUMMARY: ...' }` message followed by the uncompressed newer half.
- Calling `compressIfNeeded` twice with the same `messages` array that triggered compression results in exactly one `anthropic.messages.create` call (cache hit on second call).
- The module-level cache is keyed on a deterministic djb2 hash of `JSON.stringify(oldestHalf.map(m => m.content))`; two calls whose oldest-50% slice content differs must produce different cache keys (no intentional collision).
- Token estimate: `countTokens(text)` = `Math.ceil(text.split(/\s+/).filter(Boolean).length * 1.3)`. Total = sum of `countTokens(msg.content)` for all messages.
- `clearCacheForTesting()` resets the module-level cache to empty; subsequent calls to `compressIfNeeded` behave as if no prior compression has occurred.

**Non-goals**
- This ticket does NOT implement RAG, vector search, or embedding-based summarization.
- This ticket does NOT integrate `compressIfNeeded` into any Convex action (that is done in T12ŌĆōT15).
- This ticket does NOT implement streaming ŌĆö the Haiku compression call is a single non-streaming request.
- This ticket does NOT expose any Convex query/mutation; it is a pure utility module.
- This ticket does NOT define budget constant values; those are defined in `convex/lib/pricing.ts` (T4) and passed by callers.

## Stack Practices

- Accepts Anthropic client as dependency-injection parameter.
- Cache via module-level `Map<string, string>`; cache key = djb2 hash of `JSON.stringify(oldestHalf.map(m => m.content))`.
- Use `TextEncoder` + manual djb2 hash loop (no Node.js `crypto` module ŌĆö must run in Convex runtime).
- Export `clearCacheForTesting()` for test isolation; never call in production code paths.
- Anti-pattern: do NOT call `compressIfNeeded` with `budgetTokens=0`.

## Design Practices

- Pure TypeScript module: no Convex runtime imports, no SDK imports at module scope. Must pass `tsc --noEmit` and run in Vitest `environment: 'node'` without a Convex environment.
- The "nothing clever" token-counting heuristic (`words * 1.3`) is a deliberate design choice per the spec ŌĆö do not replace it with a tiktoken library or exact byte counting.
