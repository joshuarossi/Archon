The four core utility modules — privacy filter, prompt assembly, transcript compression, and the case state machine — are pure TypeScript with no Convex or SDK runtime dependencies. That purity makes them straightforward to unit-test in isolation, but it also means bugs here propagate silently into every AI action in the system. This ticket establishes the Vitest suite for all four modules, providing fast, deterministic regression coverage that runs before any integration or E2E layer.

## Acceptance Criteria

- privacyFilter: ≥5 adversarial, ≥3 benign, boundary cases
- All 4 assemblePrompt roles tested
- All 9 valid state transitions and ≥8 illegal transitions tested
- All tests pass with `npx vitest run`

## Implementation Notes

All four source modules are pure TypeScript (no Convex runtime imports, no SDK imports) defined in T5–T8:

- **T5 `convex/lib/privacyFilter.ts`** — `filterResponse(response, otherPartyMessages)` tokenizes by whitespace (`text.toLowerCase().split(/\s+/).filter(Boolean)`), pre-computes N-gram windows into a `Set<string>`, and checks for ≥8 consecutive token matches. Tests must cover: exact 8-token window match (passes false), 7-token window (passes true), zero-overlap benign input, empty arrays, single-word inputs, and adversarial cases that vary casing, extra whitespace, or partial punctuation. Note: both `response` and `otherPartyMessages` entries are lower-cased before comparison, so casing-variant adversarial fixtures must verify `passed === false`.

- **T6 `convex/lib/prompts.ts`** — `assemblePrompt({ role, ... })` returns `{ system, messages }` for roles `PRIVATE_COACH`, `SYNTHESIS`, `COACH`, `DRAFT_COACH`. The no-quote rule must appear in every system prompt. `SYNTHESIS` must include an instruction to respond with ONLY a JSON object. Tests should assert these invariants against the string content of `system`. For testing purposes, `{ role }` alone is the minimum valid opts shape; all four roles are callable with only the `role` field.

- **T7 `convex/lib/transcriptCompression.ts`** — `compressIfNeeded(messages, budgetTokens, anthropicClient)` is async and accepts the Anthropic client as a DI parameter. **Token counting approximation** (word-count): `messages.reduce((sum, m) => sum + m.content.split(/\s+/).filter(Boolean).length, 0)`. Use this formula to construct fixtures that are verifiably under or over `budgetTokens`. Under budget: returns original array, zero API calls. Over budget: replaces the oldest 50% of messages with a single `{ role: 'assistant', content: 'SUMMARY: <canned text>' }` message. Content-hash caching (TextEncoder + manual hash loop, no Node `crypto`): same input → one API call regardless of how many times called. Tests use `vi.fn()` to mock the Anthropic client and assert call counts.

- **T8 `convex/lib/stateMachine.ts`** — exports one named function per transition (e.g., `transitionToJointActive`, `transitionToClosedAbandoned`). Each is pure: takes current status, returns new status or throws `ConvexError` (import from `convex/values` — safe, no `_generated` import) with `CONFLICT` code. The full set of 9 valid (source → target) transitions, with the predecessor status required for each function call:

  | # | Function | Valid predecessor | Returns |
  |---|----------|-------------------|---------|
  | 1 | `transitionToReadyForJoint` | `DRAFT_PRIVATE_COACHING` | `READY_FOR_JOINT` |
  | 2 | `transitionToClosedAbandoned` | `DRAFT_PRIVATE_COACHING` | `CLOSED_ABANDONED` (invitee decline) |
  | 3 | `transitionToJointActive` | `READY_FOR_JOINT` | `JOINT_ACTIVE` |
  | 4 | `transitionToClosedResolved` | `JOINT_ACTIVE` | `CLOSED_RESOLVED` |
  | 5 | `transitionToClosedAbandoned` | `JOINT_ACTIVE` | `CLOSED_ABANDONED` (cron 30d) |
  | 6 | `transitionToClosedUnresolved` | `JOINT_ACTIVE` | `CLOSED_UNRESOLVED` |
  | 7 | `transitionToCostLimited` | `JOINT_ACTIVE` | `COST_LIMITED` |
  | 8 | `transitionToClosedResolved` | `COST_LIMITED` | `CLOSED_RESOLVED` |
  | 9 | `transitionToClosedUnresolved` | `COST_LIMITED` | `CLOSED_UNRESOLVED` |

  Actual function names are determined by T8's implementation; the table above uses the naming pattern established in the plan. Tests should use whatever names T8 exports. The decline-invite path (`DRAFT_PRIVATE_COACHING → CLOSED_ABANDONED`) must be explicitly tested as a distinct test case.

Vitest config: set `environment: 'node'` — no DOM needed. Test files live in `convex/lib/__tests__/`. Do NOT import from `convex/_generated/server` in any test file; that directory is generated at runtime by `npx convex dev` and is not available in the Vitest process.

The transcript compression mock should inject a `vi.fn()` that resolves to a canned string so tests remain synchronous-feeling and deterministic.

**Queries used**

None — all four modules are pure TypeScript; no Convex DB or API calls.

**Invariants**

- `filterResponse` is synchronous and always returns an object with at least `passed`.
- `filterResponse` returns `passed: false` if and only if ≥8 consecutive whitespace-tokenized tokens from `response` (lower-cased) appear as a contiguous window in any lower-cased message in `otherPartyMessages`.
- `assemblePrompt` always includes the no-quote rule string in `result.system` for all four roles.
- `assemblePrompt` with `role: 'SYNTHESIS'` instructs the model to reply with ONLY a JSON object — verified by substring check in `result.system`.
- `compressIfNeeded` never calls `client.messages.create` when word-count sum of messages is below `budgetTokens`.
- `compressIfNeeded` calls `client.messages.create` at most once per unique content-hash regardless of invocation count.
- State machine transition functions throw `ConvexError` (not a plain `Error`) with a `CONFLICT` code when the current status does not allow the requested transition.
- `transitionToClosedAbandoned('DRAFT_PRIVATE_COACHING')` returns `'CLOSED_ABANDONED'` (invitee decline path).

**Non-goals**

- No integration tests against a live Convex deployment.
- No testing of AI prompt quality or Claude output format — only structural invariants.
- No tests for T9–T17 backend mutations or actions (those are integration-level).
- No browser environment testing — `environment: 'node'` throughout.
- Transcript compression does NOT test the Haiku model API response format — only call-count and return shape.
- **No invite token generation tests** — that logic (T10 `invites/redeem`) uses Convex runtime DB and is integration-level, not a pure utility. The plan summary mentions it but it has no AC here and is out of scope.

## Stack Practices

- Test files co-located in convex/lib/__tests__/.
- Vitest config: environment: 'node'.
- Mock for transcriptCompression: vi.fn() for Anthropic client (see concrete construction in Signatures above).
- Anti-pattern: do NOT import convex/_generated/server in Vitest tests.
- `ConvexError` import for state machine tests: `import { ConvexError } from 'convex/values'` — this package is available at test time without a running Convex deployment.
- Token budget fixtures: use the word-count approximation formula `messages.reduce((sum, m) => sum + m.content.split(/\s+/).filter(Boolean).length, 0)` to size fixtures reliably above and below `budgetTokens`.
