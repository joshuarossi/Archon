# Prompt assembly module for all four AI roles

Every AI action in Conflict Coach (Private Coach, Synthesis, Joint Coach, Draft Coach) must pass through a single, testable prompt-assembly module rather than building system prompts inline. This task creates `convex/lib/prompts.ts`, the authoritative source for all four role variants, so that prompt wording and structure can be reasoned about in one place and covered by unit tests before any AI action is wired up.

## Acceptance Criteria

- `assemblePrompt({ role }) returns { system, messages }`
- All four prompts contain the no-quote rule
- SYNTHESIS produces strict JSON instruction
- Pure TypeScript, testable in Vitest

## Implementation Notes

The function signature from the tech spec (§6.3):

```ts
function assemblePrompt(opts: {
  role: "PRIVATE_COACH" | "COACH" | "DRAFT_COACH" | "SYNTHESIS";
  caseId: Id<"cases">;
  actingUserId: Id<"users">;
  recentHistory: Message[];
  templateVersion?: TemplateVersion;
}): { system: string; messages: Message[] }
```

Each role has distinct context requirements and system-prompt language:

**PRIVATE_COACH (§6.3.1):** No template applied — uses a hardcoded runtime default. The system prompt coaches one party privately: "You are a calm, curious, non-judgmental listener helping a person articulate their perspective in an interpersonal conflict…" Context injected: the acting party's form fields (mainTopic, description, desiredOutcome) + their prior private message history. The other party's data must never appear in this call's context; the module enforces this by only accepting the acting party's history.

**SYNTHESIS (§6.3.2):** One-shot, non-streaming. The system prompt instructs the model to output a strict JSON object `{ "forInitiator": "...", "forInvitee": "..." }`. The critical no-quote instruction is verbatim: "You have access to both parties' private content for context. In your outputs, NEVER quote, closely paraphrase, or otherwise surface the other party's raw words. Synthesize themes and positions in your own words only. If you cannot make a point without quoting, omit it." The instruction must also tell the model to respond with ONLY the JSON object — no preamble, no markdown fences.

**COACH (§6.3.3):** Category-specific template applied if available, else a baseline that establishes the facilitator role. Context: joint chat history + both parties' synthesis texts (never raw private messages). The no-quote rule applies here identically to synthesis. The Haiku classification gate (INFLAMMATORY / PROGRESS / QUESTION_TO_COACH / NORMAL_EXCHANGE) is handled in the executor, not in this module; this module only assembles the Sonnet call.

**DRAFT_COACH (§6.3.4):** Category-specific template or default. The system prompt describes the conversation loop: ask clarifying questions, surface tone issues, and produce a structured output `{ "draft": "..." }` when the user signals readiness. Privacy constraint: context includes only the drafting user's joint-chat history and their own synthesis — never the other party's synthesis or private content.

All four roles must include an explicit no-quote instruction somewhere in the system prompt. The SYNTHESIS verbatim instruction satisfies the requirement for that role; PRIVATE_COACH, COACH, and DRAFT_COACH should each include a shorter variant appropriate to their framing (e.g., "Do not quote or closely paraphrase the other party's private words in any response.").

Keep this module zero-dependency: no Convex server imports, no Anthropic SDK imports. All types referenced in the signature (`Message`, `TemplateVersion`) should either be inlined or imported from the Convex schema's inferred types — whichever keeps the module side-effect-free and importable in Vitest without a Convex runtime. Use multi-line template literals for all system-prompt strings; no concatenation chains.

## Test-Gen Brief

**Files**
- Implementation: `convex/lib/prompts.ts`
- Tests: `convex/lib/__tests__/prompts.test.ts`

**Exports**
- `assemblePrompt` — named export, value (function)
- `PromptRole` — named export, type (union: `"PRIVATE_COACH" | "COACH" | "DRAFT_COACH" | "SYNTHESIS"`)
- `AssemblePromptOpts` — named export, type (input shape)
- `AssembledPrompt` — named export, type (`{ system: string; messages: Message[] }`)

**Signatures**

```ts
export type PromptRole = "PRIVATE_COACH" | "COACH" | "DRAFT_COACH" | "SYNTHESIS";

export interface AssemblePromptOpts {
  role: PromptRole;
  recentHistory: Message[];          // Anthropic-shaped messages array
  templateVersion?: TemplateVersion; // undefined for PRIVATE_COACH
  // Additional context fields as needed per role (e.g. partyFormFields, bothSyntheses)
}

export interface AssembledPrompt {
  system: string;
  messages: Message[];
}

export function assemblePrompt(opts: AssemblePromptOpts): AssembledPrompt;
```

Note: `Message` should match the shape expected by the Anthropic SDK (`{ role: "user" | "assistant"; content: string }`). If the Convex schema defines a message shape, derive `Message` from it via `z.infer` or Convex's `Infer` — do not duplicate.

**Queries used**
- None — this is a pure function; it receives pre-fetched data as arguments.

**Invariants**
- `assemblePrompt` never throws for any valid `PromptRole` and any (including empty) `recentHistory`.
- The returned `system` string always contains a no-quote instruction substring for every role.
- For `SYNTHESIS`, the returned `system` string contains the verbatim critical instruction: "NEVER quote, closely paraphrase, or otherwise surface the other party's raw words."
- For `SYNTHESIS`, the returned `system` string instructs the model to respond with only the JSON object.
- `recentHistory` is passed through unchanged into `messages`; the function does not mutate the input array.
- When `templateVersion` is `undefined`, `PRIVATE_COACH` uses the hardcoded default system prompt.
- `COACH` and `DRAFT_COACH` incorporate `templateVersion` content when provided; fall back to baseline when `undefined`.

**Non-goals**
- This ticket does NOT implement Haiku classification logic (belongs in T14).
- Does NOT call the Anthropic API.
- Does NOT read from `ctx.db`; all context is injected as arguments.
- Does NOT implement transcript compression (T7).
- Does NOT validate JSON output from SYNTHESIS (validation is in the executor, T13).
- Does NOT enforce the privacy response filter (T5).

**Tested by**

| All four prompts contain the no-quote rule | Unit | `result.system.toLowerCase()` includes `"never quote"` or `"do not quote"` (exact substring per role) for each of the 4 roles | Same fixtures |

All tests run with `npx vitest run` under `environment: 'node'`. No Convex runtime imports in the test file.
