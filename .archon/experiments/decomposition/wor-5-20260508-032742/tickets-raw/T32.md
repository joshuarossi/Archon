# Admin Template UI (List, Edit with Version History, Audit Log)

This ticket delivers the three admin-only routes that allow Riley (the Applied Labs admin persona) to manage coaching templates and review admin activity without touching code. Template management is a P1 goal of the Epic — AI behavior must be fully template-driven so prompt tuning never requires a deployment. The UI surfaces `convex/admin/templates.ts` (built in T11) to a read-write interface and presents `auditLog` entries in a browsable, filterable table. All three pages are gated behind a server-enforced `role === 'ADMIN'` check; a non-admin who navigates to any `/admin/*` route lands on a calm 403 page.

## Acceptance Criteria

- Non-admins get 403 page (server-side role check)
- TemplatesListPage: table with relevant columns; [+ New Template] button
- Pinned Cases Count computed via query
- TemplateEditPage: two-pane form + version history; 'Publish New Version' button
- Version comparison modal: side-by-side text view
- 'Archive Template': confirmation with pinned case count warning
- AuditLogPage: filterable table; row click opens Drawer with metadata JSON
- Audit log read-only

## Implementation Notes

**Route guarding.** A `RequireAdmin` wrapper component reads `useQuery(api.users.getCurrentUser)` and redirects (or renders a 403 page) if `currentUser.role !== 'ADMIN'`. Place this wrapper in the React Router config around all `/admin/*` nested routes. Never rely solely on the client check — every Convex function behind these pages calls `assertAdmin(ctx)` (already implemented in T4), which throws `FORBIDDEN` server-side. Treat the client check as a UX convenience only.

**Templates list.** Query `api.admin.templates.listAll` (returns templates including archived; T11). Pinned Cases Count for each template is derived from a separate query `api.admin.templates.pinnedCasesCount` or included in the `listAll` response — check T11's implementation to see which approach was used. If not included, add a small helper query in `convex/admin/templates.ts`. Filter archived rows into a collapsible section or badge them with an Archived status pill so active templates are visually primary.

**Template edit form.** This is a two-pane layout on desktop (left: live draft form, right: version history timeline). The form fields match the shape of `admin/templates/publishNewVersion`: `globalGuidance`, `coachInstructions`, `draftCoachInstructions`, `notes`. These are all large Textareas with markdown content; label them clearly. The "Publish New Version" action creates an immutable `templateVersion` row and updates `template.currentVersionId` — it does NOT mutate the version that existing cases are pinned to. Cases are pinned at creation time to a specific `templateVersionId`; this is the versioning guarantee T11 enforces.

**Version history timeline.** Right-pane shows all `templateVersions` for the template in reverse-chronological order (query: `api.admin.templateVersions.list`). Each entry shows version number (or date), the admin who published it, and their notes. Each entry has a "View" ghost button that opens the version comparison modal.

**Version comparison modal.** A wide (max-width 800px) shadcn/ui Dialog with two columns side-by-side: the selected historical version's content and the current draft. All three text fields (globalGuidance, coachInstructions, draftCoachInstructions) shown in `<pre>` blocks with JetBrains Mono. This is read-only; it helps admins see exactly what changed before publishing again.

**Archive confirmation.** The "Archive Template" button opens a shadcn/ui AlertDialog (not a custom dialog). The confirmation body must show how many cases are currently pinned to this template (or any of its versions) so Riley understands the blast radius. Archiving sets `archivedAt` and soft-deletes from the `templates/list` query (the query non-admin users use when creating cases), but existing cases continue to resolve their pinned version without issue.

**Audit log.** Query `api.admin.auditLog.list` (or equivalent from T11's `auditLog` table). Display Actor (admin email) | Action (string) | Target (templateId or description) | Timestamp. Timestamp renders in JetBrains Mono for the monospaced alignment look. A filter Input above the table allows filtering by actor email client-side (no server-side filter needed at v1 scale). Row click opens a shadcn/ui Sheet from the right with a pretty-printed `<pre>` of the full `metadata` JSON blob. The table and Sheet are read-only — no mutations from this page.

**New Template creation.** The [+ New Template] button routes to `/admin/templates/new` (or opens a modal — whichever is simpler given the edit page already handles the form). The create mutation is `api.admin.templates.create`; it writes a template row + an initial `templateVersion` row atomically (T11).

**Component colocation.** Admin components live in `src/routes/admin/`. There is no shared admin state store needed — each page uses its own `useQuery` subscriptions. The `RequireAdmin` wrapper lives in `src/components/layout/RequireAdmin.tsx` so it can be reused in the route config.

### Queries used
- `api.users.getCurrentUser` — role check in RequireAdmin
- `api.admin.templates.listAll` — TemplatesListPage (includes archived)
- `api.admin.templates.pinnedCasesCount` (or field on listAll response) — archive modal warning
- `api.admin.templateVersions.list` — TemplateEditPage version timeline
- `api.admin.auditLog.list` — AuditLogPage (or equivalent T11 export)
- `api.admin.templates.create` — mutation, new template
- `api.admin.templates.publishNewVersion` — mutation, publish draft
- `api.admin.templates.archive` — mutation, soft-delete

### Invariants
- `RequireAdmin` must never render its `children` when `currentUser.role !== 'ADMIN'`; it must render the 403 page or redirect
- Clicking "Publish New Version" must NOT mutate any existing `templateVersion` row — only inserts a new one
- "Archive Template" confirmation dialog must display the pinned cases count before the admin can confirm
- The audit log table is purely read-only — no mutation can be triggered from `AuditLogPage`
- Template edit form state is local until "Publish New Version" is explicitly clicked — typing in the textarea does not auto-save

### Non-goals
- This ticket does NOT implement the Convex backend functions (`convex/admin/templates.ts`, `convex/admin/auditLog`) — those are T11's responsibility
- Does NOT implement any non-admin views (templates are read via `templates/list` in case creation — T23)
- Does NOT add email notifications for publish/archive events
- Does NOT add admin user promotion UI (manual Convex Dashboard action per planning assumption)
- Does NOT add per-field diff highlighting in the version comparison modal — plain `<pre>` side-by-side is sufficient

## Stack Practices

- Admin route guard: RequireAdmin wrapper reads currentUser.role === 'ADMIN'.
- Table: shadcn/ui Table, TableHeader, TableRow, TableHead, TableBody, TableCell.
- Version comparison modal: shadcn/ui Dialog with two side-by-side <pre> blocks.
- Drawer for audit log: shadcn/ui Sheet side='right'.
- Anti-pattern: do NOT bypass server-side assertAdmin check.

## Design Practices

- 403 page: centered on --bg-canvas. '403 — Access Restricted' heading 500 weight --text-primary. Body in --text-secondary. 'Back to Dashboard' secondary link. Calm, not alarming — no red banners.
- Templates list table (design doc §4.13): shadcn/ui Table. bg-canvas page, bg-surface table container with shadow-1. Column headers in 13px 500 weight --text-secondary uppercase. Rows in 14px --text-primary. Hover row: bg-surface-subtle, 150ms ease-out. Status badge (Active / Archived): Active = --accent text + bg-accent-subtle; Archived = --text-tertiary + bg-surface-subtle. Pinned Cases Count column: 14px --text-secondary.
- '+ New Template' button: primary --accent, top-right of page. Single primary action (design doc §1.4).
- Template edit page (design doc §4.13): two-pane layout. Left pane (~60% width): form with shadcn/ui Textarea for Global Guidance, Coach Instructions, Draft Coach Instructions (all markdown, 8+ rows). Labels in 14px 500 weight. Right pane (~40%): version history timeline — vertical list of published versions with date, admin name, notes in 13px --text-secondary. Each version has a 'View' ghost button.
- Version history timeline: vertical connector line in --border-default. Each version node: 8px circle --accent, date label --text-tertiary 12px, 'View' ghost button.
- 'Publish New Version' button: primary --accent, full-width at bottom of left pane. 'Archive Template' button: shadcn/ui Button variant with --danger color, at bottom, below a border divider. Archive confirmation Dialog shows pinned case count in --warning color.
- Version comparison modal: shadcn/ui Dialog, wide (max-width 800px), shadow-3. Two columns: 'Version N-1' and 'Version N' headings in 500 weight. Content in <pre> or monospace textarea, JetBrains Mono 13px, --text-secondary. bg-surface-subtle background.
- Audit log page (design doc §4.13): shadcn/ui Table. Filter input (shadcn/ui Input) for actor email above table. Actor | Action | Target | Timestamp columns. Timestamp in JetBrains Mono 13px --text-tertiary. Row click → shadcn/ui Sheet side='right' with JSON metadata in <pre className='text-xs overflow-auto font-mono'>.
- Accessibility: admin tables use proper <th scope='col'> headers. Filter input has <label> or aria-label='Filter by actor email'. Audit log Sheet focus-trapped, Esc closes. All icon-only buttons in version history have aria-label.
