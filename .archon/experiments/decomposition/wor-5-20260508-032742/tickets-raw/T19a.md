# Core chat UI components (ChatWindow, MessageBubble, StreamingIndicator, MessageInput)

The entire conversational experience of Conflict Coach — private coaching, joint chat, and draft coaching — runs on a shared set of chat primitives. This task builds those four foundational components so that every chat surface in the app uses consistent, accessible, well-designed building blocks rather than re-implementing message rendering in each view. Without these components, none of the feature views (T25–T28) can be assembled. They depend on T18's design tokens for color, typography, spacing, and motion.

## Acceptance Criteria

- ChatWindow auto-scrolls to latest unless user has scrolled up >100px
- MessageBubble USER: right-aligned, bg-surface; COACH/AI: left-aligned, accent-subtle, Sparkles icon
- MessageBubble STREAMING: content + blinking cursor; no copy button
- MessageBubble COMPLETE: copy button + timestamp on hover
- MessageBubble ERROR: warning-tint + Retry button
- StreamingIndicator: 500ms blink; aria-hidden
- MessageInput: Enter sends; Shift+Enter newline; disabled while STREAMING
- No accessibility defects: textarea has visible label or aria-label

## Implementation Notes

These components are purely presentational — they receive data and callbacks as props and carry no Convex query hooks themselves. The calling view (e.g., `PrivateCoachingView`) owns data fetching and passes message arrays down.

**ChatWindow** wraps the scrollable message list. It holds a `useRef<HTMLDivElement>` on the scroll container. A `useEffect` watches the messages array length; when it grows, it scrolls to the bottom — but only if the user's current `scrollTop + clientHeight` is within 100px of `scrollHeight`. This prevents jarring scroll-jacking when the user is reading older messages. The container gets `role="log" aria-live="polite" aria-relevant="additions"` per design doc §9.5.

**MessageBubble** renders one of four variants — `USER`, `COACH`, `STREAMING`, or `ERROR` — controlled by a `variant` prop (or derive from the message `status` field). Key implementation points:
- USER: `justify-end`, `bg-[--bg-surface]`, no icon, max-width ~70% of container, `rounded-2xl`, `shadow-0` (border only).
- COACH/AI: `justify-start`, `bg-[--accent-subtle]`, Lucide `Sparkles` icon at top-left (16px, 1.5px stroke). Joint chat coach messages additionally need a `border-l-2 border-[--coach-accent]` — the calling view (T27) applies this by passing an `isCoachMessage` prop.
- STREAMING: same COACH layout with a `StreamingIndicator` appended inline; hide copy button.
- ERROR: `bg-[--danger-subtle]`, `AlertTriangle` icon in `--danger`, Retry `Button variant="ghost"` in danger color.
- Copy button (COMPLETE only): appears on hover via `opacity-0 group-hover:opacity-100 transition-opacity duration-150`. Uses Lucide `Copy` icon, `aria-label="Copy message"`. Timestamp below bubble in `text-xs text-[--text-tertiary]`.
- Render message text via `react-markdown` with `rehype-sanitize` — never `dangerouslySetInnerHTML`.

**StreamingIndicator** is a thin blinking vertical bar rendered inline at the end of streaming text. Implement as a `<span>` with a `@keyframes blink` animation (`0%, 100% { opacity: 1 } 50% { opacity: 0 }`) at 500ms `step-end`. Wrap the `@keyframes` in `@media (prefers-reduced-motion: no-preference)` so it freezes in reduced-motion environments. The element itself is `aria-hidden="true"` — screen readers should not see it.

**MessageInput** is a controlled `<textarea>` with auto-resize. `onKeyDown` intercepts `Enter` (without Shift) to call the `onSubmit` callback; `Shift+Enter` inserts a newline normally. When `disabled` is true (while a STREAMING message is in flight), the textarea and send button become unresponsive — visually `opacity-50 cursor-not-allowed`. The textarea must have either a visible `<label>` or an `aria-label` attribute (e.g., `"Your message"`). Reserve a 20px empty row below the input for the typing indicator placeholder (deferred to v1.1) to prevent layout shift when it eventually appears.

**shadcn/ui primitives** to install/use: `Button` (`@/components/ui/button`). Lucide icons from `lucide-react`: `Sparkles`, `Copy`, `AlertTriangle`. `react-markdown` and `rehype-sanitize` must be added as dependencies.

**Tailwind CSS custom property classes**: use bracket notation (`bg-[--accent-subtle]`, `text-[--text-tertiary]`) for the design token CSS variables from T18. Do not hardcode hex values.

## Stack Practices

- Component files: src/components/chat/ChatWindow.tsx, MessageBubble.tsx, MessageInput.tsx, StreamingIndicator.tsx. Use named exports.
- Auto-scroll: useRef<HTMLDivElement> + useEffect calling scrollTo when messages change, conditional on scroll position check.
- shadcn/ui primitives: Button from @/components/ui/button; Lucide icons (Sparkles, Copy, AlertTriangle) from lucide-react.
- StreamingIndicator animation: @keyframes blink wrapped in @media (prefers-reduced-motion: no-preference).
- MessageInput keyboard: onKeyDown checks e.key === 'Enter' && !e.shiftKey.
- Anti-pattern: do NOT use dangerouslySetInnerHTML — use react-markdown with rehype-sanitize.

## Design Practices

- MessageBubble — USER variant: right-aligned, background --bg-surface, no icon. Spacing: 16px horizontal padding, 12px vertical. Max-width ~70% of ChatWindow. Border-radius 12px. Use shadow-0 (flat + border-default).
- MessageBubble — COACH/AI variant: left-aligned, background --accent-subtle (#DCE7E0), Sparkles icon (Lucide, 1.5px stroke, 16px) at top-left. Coach messages from joint chat additionally get a 2px left border in --coach-accent (#8B7AB5). Same border-radius and padding as USER variant.
- MessageBubble — STREAMING: render blinking cursor (thin 1px vertical bar) at end of streaming text. Blink @keyframe: 0%,100% opacity:1; 50% opacity:0 at 500ms step-end. Wrap in @media (prefers-reduced-motion: no-preference). StreamingIndicator must be aria-hidden='true' so screen readers do not repeat it.
- MessageBubble — COMPLETE: copy button appears on hover (opacity transition 150ms ease-out). Timestamp rendered below bubble in 12px --text-tertiary. Copy button uses Lucide Copy icon at 14px.
- MessageBubble — ERROR: background --danger-subtle (#F2DCD8), Lucide AlertTriangle icon in --danger (#B5594D). Retry button uses shadcn/ui Button variant='ghost' in danger color.
- MessageInput textarea: Inter 15px, --text-primary, border --border-strong, focus-visible ring 2px --accent. Send button: --accent fill, disabled state opacity-50. Reserve visual space below input for 'Jordan is typing…' (design doc §4.9, deferred v1.1) — a 20px empty row prevents layout jump.
- ChatWindow container: role='log' aria-live='polite' aria-relevant='additions' per design doc §9.5 and T33. New message arrival animation: fade-in + translateY(-8px → 0) at 150ms ease-out, under prefers-reduced-motion guard.
- Accessibility: textarea must have a visible label or aria-label. All icon-only buttons need aria-label (e.g., aria-label='Copy message'). Retry button: role='button' with descriptive aria-label.
