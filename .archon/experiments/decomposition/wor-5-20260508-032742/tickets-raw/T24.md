# Case Detail Router and Phase-Appropriate View Scaffold

`/cases/:caseId` serves as the central routing hub of the Conflict Coach application, directing users to the correct phase-specific sub-route based on the live case status. This task creates the layout route — `CaseDetail.tsx` — that wraps all phase views with a persistent `PhaseHeader`, handles the routing logic that maps each case status to its appropriate sub-path, and provides a shared loading skeleton and friendly error boundary. It is the gateway every feature view depends on; getting the routing model right here prevents status-skipping bugs and ensures consistent layout across all phases.

## Acceptance Criteria

- GET /cases/:caseId redirects to correct sub-route by case.status
- Phase-skip redirects with toasts
- PhaseHeader on all sub-routes
- Loading: skeleton while cases/get in flight
- caseId not found / not party: friendly error page

## Implementation Notes

`CaseDetail.tsx` is a React Router v6 **layout route** that renders `<Outlet />` for child routes. The component runs `useQuery(api.cases.get, { caseId })` once and shares the result downstream via `useOutletContext` — child routes must NOT re-fetch this query independently.

**Status → sub-route mapping** (drive this in a `useEffect` + `navigate(..., { replace: true })` pattern):
- `DRAFT_PRIVATE_COACHING` → `../private`
- `READY_FOR_JOINT` → `../ready`
- `JOINT_ACTIVE` → `../joint`
- `COST_LIMITED` → `../joint` (COST_LIMITED is still joint-chat visible)
- `CLOSED_RESOLVED`, `CLOSED_UNRESOLVED`, `CLOSED_ABANDONED` → `../closed`

Phase-skip toasts fire when a user deep-links to a sub-route that doesn't match the live status. For example, navigating directly to `/cases/:id/private` when the case is already `JOINT_ACTIVE` should redirect to `/joint` and show a shadcn/ui toast: "That phase has already passed."

**Loading states**: `useQuery` returns `undefined` while loading and `null` when the document doesn't exist (Convex convention). Treat `undefined` → render `CaseDetailSkeleton`; treat `null` (or a thrown `NOT_FOUND` / `FORBIDDEN` ConvexError) → render the friendly error page.

**Context sharing**: Use `useOutletContext<{ caseData: CaseWithParties }>()` typed hook in child routes. Define the context type in a shared `src/routes/cases/types.ts` file so all sub-routes import from one place.

**PhaseHeader**: rendered directly inside `CaseDetail.tsx` above `<Outlet />`, making it available unconditionally on all case sub-routes. It receives `caseData` as a prop.

The `CaseDetailSkeleton` should be phase-appropriate: for chat-phase cases (status known from URL hint or prior nav state) render a full-height column skeleton; for cases where status is unknown, default to the chat-phase skeleton layout as it's the most common.

## Stack Practices

- React Router v6 nested routes: CaseDetail.tsx is a layout route that renders <Outlet />.
- Phase routing: useEffect + navigate with replace: true after cases/get resolves.
- useQuery loading: undefined = loading → CaseDetailSkeleton; null = not found.
- Anti-pattern: do NOT re-fetch cases/get in each sub-route — use useOutletContext.

## Design Practices

- CaseDetail layout: PhaseHeader rendered above <Outlet /> in the layout — all sub-routes inherit it. Full-height layout with PhaseHeader fixed or sticky at top using bg-surface, border-bottom --border-default.
- Loading skeleton: phase-appropriate placeholder (full-height column for chat views, centered card for ready-for-joint). Use shadcn/ui Skeleton with 300ms delay. --bg-surface-subtle shimmer.
- Error page (not found / not party): centered on --bg-canvas. Friendly --text-secondary message, no technical jargon. Single 'Back to Dashboard' secondary link. No alarming colors.
- Route transition: crossfade 100ms between sub-routes (design doc §2.6). No slide animations.
- Accessibility: route change must move focus to the h1 of the new sub-route (useLocation subscription in App.tsx + h1 tabIndex={-1}). PhaseHeader <nav> landmark wraps the back link and phase title. Toast announcements use shadcn/ui toast's aria-live.
