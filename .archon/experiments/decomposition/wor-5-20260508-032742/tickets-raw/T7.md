# Transcript Compression Module

AI-powered features across private coaching, synthesis, and joint chat all send conversation history to Claude as context. As cases accumulate messages, those contexts can grow past practical token budgets. This module is the shared solution: when a message array exceeds a token budget it compresses the oldest half into a single `SUMMARY` synthetic message via a Haiku API call, then caches the result by content hash so the same segment is never compressed twice. Every AI action in the codebase will call `compressIfNeeded` before building its API request — getting this right and testable in isolation is prerequisite to all four AI actions (T12–T15).

## Acceptance Criteria

- `compressIfNeeded` returns original array when under budget without API call
- Over-budget: oldest 50% replaced by SUMMARY message
- Content hash caching: same input = one Haiku API call
- Token counting uses word-count approximation

## Implementation Notes

The tech spec (§6.4) defines the compression algorithm:
1. Compute total token count of all messages.
2. If under budget, return the array unchanged — no API call, no allocation.
3. If over budget, take the oldest 50% of messages.
4. Call Haiku with the prompt: "Summarize this conversation segment in 500 tokens or fewer, preserving facts, decisions, emotional tone, and unresolved threads."
5. Replace those messages with a single synthetic `{ role: 'user', content: 'SUMMARY: <haiku output>' }` message.
6. Cache the result by the content hash of the compressed segment so re-compression of an unchanged prefix is free.

The Anthropic client is passed in as a dependency-injection parameter — this module does not import from `@anthropic-ai/sdk` directly at module scope, so it remains fully unit-testable with a vi.fn() mock. The cache is a module-level `Map<string, string>` keyed on a content hash. Because the hash must work in the Convex runtime (no Node.js `crypto` module), compute it with a `TextEncoder` and a manual djb2-style hash loop.

Token counting is intentionally approximate: `text.split(/\s+/).filter(Boolean).length` multiplied by 1.3 (words-to-tokens heuristic). This is consistent with the prototype approach — "nothing clever" is a deliberate design choice per the spec.

The module lives at `convex/lib/transcriptCompression.ts` alongside the other pure utility modules (`privacyFilter.ts`, `prompts.ts`, `stateMachine.ts`). It must not import from any Convex-generated paths (`convex/_generated/server`, etc.) so Vitest can run it without a Convex environment.

The default budgets from the spec are: **60 000 tokens** for user-facing generation (private coaching, joint chat Coach, Draft Coach), **10 000 tokens** for classification tasks (Haiku inflammatory gate). Callers pass the appropriate `budgetTokens` value; this module does not hardcode any default.

Calling `compressIfNeeded` with `budgetTokens = 0` is a programmer error and should throw immediately with a descriptive message.

## Test-Gen Brief

**Files**
- Implementation: `convex/lib/transcriptCompression.ts`
- Tests: `convex/lib/__tests__/transcriptCompression.test.ts`

**Exports**
- `compressIfNeeded` — named, async function (value export)
- `Message` — named, TypeScript type (type export), representing a single `{ role: 'user' | 'assistant', content: string }` object

**Signatures**
```typescript
export type Message = { role: 'user' | 'assistant'; content: string };

export type AnthropicClientLike = {
  messages: {
    create(params: {
      model: string;
      max_tokens: number;
      messages: Message[];
    }): Promise<{ content: Array<{ type: string; text?: string }> }>;
  };
};

export async function compressIfNeeded(
  messages: Message[],
  budgetTokens: number,
  anthropic: AnthropicClientLike,
): Promise<Message[]>
```

**Queries used**
- `anthropic.messages.create` — called with `model: 'claude-haiku-4-5-20251001'`, `max_tokens: 600`, and the oldest-50%-of-messages slice. Called at most once per unique content hash.

**Invariants**
- If `budgetTokens <= 0`, throws `Error('budgetTokens must be > 0')` before any other work.
- If total estimated tokens ≤ `budgetTokens`, `anthropic.messages.create` is never called and the returned array is the same reference (or structurally equal) to `messages`.
- After compression, the returned array always begins with a `{ role: 'user', content: 'SUMMARY: ...' }` message followed by the uncompressed newer half.
- Calling `compressIfNeeded` twice with the same `messages` array that triggered compression results in exactly one `anthropic.messages.create` call (cache hit on second call).
- The module-level cache is keyed on a deterministic hash of the compressed segment's content; different segment contents must produce different keys.
- Token estimate: `countTokens(text)` = `Math.ceil(text.split(/\s+/).filter(Boolean).length * 1.3)`. Total = sum of `countTokens(msg.content)` for all messages.

**Non-goals**
- This ticket does NOT implement RAG, vector search, or embedding-based summarization.
- This ticket does NOT integrate `compressIfNeeded` into any Convex action (that is done in T12–T15).
- This ticket does NOT implement streaming — the Haiku compression call is a single non-streaming request.
- This ticket does NOT expose any Convex query/mutation; it is a pure utility module.
- This ticket does NOT define budget constant values; those are defined in `convex/lib/pricing.ts` (T4) and passed by callers.

All tests use `vitest` with `environment: 'node'`. No Convex imports. The `AnthropicClientLike` interface (structural typing) allows passing `{ messages: { create: vi.fn().mockResolvedValue(...) } }` directly without importing the real SDK.

## Stack Practices

- Accepts Anthropic client as dependency-injection parameter.
- Cache via module-level Map<string, string>.
- Use TextEncoder + manual hash loop (no Node crypto module).
- Anti-pattern: do NOT call with budgetTokens=0.
