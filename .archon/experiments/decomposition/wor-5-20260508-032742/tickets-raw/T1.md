# Project scaffolding and cold-start setup

This ticket creates the empty-but-runnable project foundation that every subsequent ticket builds on. Conflict Coach is a React 18 SPA backed by Convex; getting Vite, TypeScript strict mode, Tailwind v3, shadcn/ui, React Router v6, Convex client, ESLint, and Prettier wired together correctly in one clean pass prevents painful retro-fitting later. The success bar is narrow and concrete: a developer who has never seen the repo runs four commands (`git clone`, `npm install`, `npx convex dev`, `npm run dev`) and gets a working React app in the browser with no console errors.

## Acceptance Criteria

- `npm install` completes without errors on a clean checkout
- `npx convex dev` starts without errors (Convex project configured)
- `npm run dev` serves a React app at localhost:5173 with no console errors
- ESLint and Prettier configs present; `npm run lint` and `npm run format:check` pass on the empty project
- `tsc --noEmit` passes with strict mode enabled
- README contains verbatim the cold-start sequence from tech spec §11.3
- .env.example includes all env vars from §11.2: ANTHROPIC_API_KEY, CLAUDE_MOCK, GOOGLE_OAUTH_CLIENT_ID, GOOGLE_OAUTH_CLIENT_SECRET, SITE_URL, MAGIC_LINK_EMAIL_FROM, VITE_CONVEX_URL, VITE_SITE_URL

## Implementation Notes

Tech spec §1.2 locks every layer in this task: Vite + React 18 + TypeScript for the frontend, Tailwind v3 for styling, shadcn/ui for the component system, React Router v6 for routing, and Convex for the backend/realtime layer. These choices are not up for re-evaluation at scaffolding time.

**Bootstrap order matters.** Running `npm create vite@latest -- --template react-ts` gives the correct TypeScript template. Tailwind, shadcn/ui, and Convex each modify `tailwind.config.js`, `src/index.css`, and `main.tsx` in ways that conflict if applied out of order. Recommended sequence: (1) Vite scaffold, (2) Tailwind + PostCSS, (3) shadcn/ui init (which writes `src/components/ui/` and modifies Tailwind config), (4) React Router, (5) Convex init, (6) Convex Auth wrapper in `main.tsx`.

**tsconfig.json must include `"strict": true` and `"noUncheckedIndexedAccess": true`.** These are locked per the stack practices and must not be softened to make the empty scaffold compile — the empty scaffold will already compile cleanly with them on.

**`convex/_generated/`** is created by `npx convex dev` at runtime and must be gitignored. It is not committed. The `.gitignore` should include it from day one so it never accidentally lands in a commit.

**shadcn/ui init** (`npx shadcn-ui@latest init`) writes opinionated CSS variable names to `src/index.css` (e.g., `--background`, `--foreground`, `--primary`). T18 will map the project's 16 design tokens onto these; at T1 the defaults from shadcn/ui init are fine — do not pre-fill design tokens here.

**`main.tsx` wiring.** The final wrapper order must be `<ConvexAuthProvider><ConvexProvider client={convex}><BrowserRouter><App /></BrowserRouter></ConvexProvider></ConvexAuthProvider>`. T3 configures the actual auth providers; at T1 this just needs the structural wiring present so T3 can slot in without restructuring `main.tsx`.

**.env.example** carries all 8 env vars from tech spec §11.2 with placeholder values and inline comments explaining each one. The README's cold-start section (tech spec §11.3) references this file explicitly: `cp .env.example .env.local`. The verbatim sequence from §11.3 is:
```
git clone <repo>
cd conflict-coach
npm install
cp .env.example .env.local
# fill in VITE_CONVEX_URL after convex dev starts
npx convex dev
npm run dev
```

PRD §7.1 launch criterion #6 ("Clean-env setup: `git clone && npm install && npx convex dev && npm run dev` runs the full app") applies to the finished product, not just this ticket — but T1 establishes the baseline that makes that possible.

## Stack Practices

- Bootstrap with `npm create vite@latest -- --template react-ts`; do NOT use CRA.
- Install Convex client: `npm install convex`; init project: `npx convex dev` (first run prompts project creation). The generated `convex/_generated/` directory must be gitignored — it is recreated on every `npx convex dev` run.
- Tailwind v3 setup: `npm install -D tailwindcss postcss autoprefixer && npx tailwindcss init -p`. Content glob must include `./src/**/*.{ts,tsx}` and `./convex/**/*.ts`.
- shadcn/ui init: `npx shadcn-ui@latest init`. Choose 'New York' style, CSS variables color strategy. This writes `src/components/ui/` and modifies `tailwind.config.js` — commit both.
- React Router v6: `npm install react-router-dom`. Wrap app in `<BrowserRouter>` in main.tsx. Never use React Router v5 patterns (`<Switch>`, `component=` prop).
- Wire Convex in main.tsx: `<ConvexProvider client={convex}><App /></ConvexProvider>` — Convex Auth adds `<ConvexAuthProvider>` wrapping `<ConvexProvider>`.
- tsconfig.json: enable `strict: true`, `noUncheckedIndexedAccess: true`. These are locked — do not soften.
- ESLint: use `eslint-plugin-react-hooks` and `eslint-plugin-react-refresh`. Prettier: use `prettier-plugin-tailwindcss` for class sorting.
- Anti-pattern: do NOT call `npx convex deploy` during dev scaffolding — `npx convex dev` is the development command; deploy is for production.
