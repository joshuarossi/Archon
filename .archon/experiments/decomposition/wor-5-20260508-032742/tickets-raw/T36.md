# CI Pipeline (GitHub Actions: lint, typecheck, unit, E2E)

This task configures the GitHub Actions CI pipeline that enforces all quality gates on every pull request targeting `main`. It is the final integration point of the project: CI runs lint, type-check, and Vitest unit tests as independent parallel jobs, then runs the full Playwright E2E suite against a live Convex dev deployment with `CLAUDE_MOCK=true`. Without this pipeline, none of the quality work in T33–T35 is enforceable at merge time. This task exists to ensure the repo's default branch is always in a green, deployable state.

## Acceptance Criteria

- Four jobs on push to PR branches targeting main
- lint, typecheck, unit pass independently
- e2e: deploy to Convex dev; seed; Playwright with CLAUDE_MOCK=true; multi-browser
- node_modules cached

## Implementation Notes

The CI workflow lives at `.github/workflows/ci.yml`. It triggers on `push` events to any branch that is part of a pull request targeting `main` (use `on: pull_request: branches: [main]`).

**Job structure:**

1. **lint** — runs `npm ci` (cached), then `npm run lint` and `npm run format:check`.
2. **typecheck** — runs `npm ci` (cached), then `tsc --noEmit`.
3. **unit** — runs `npm ci` (cached), then `npx vitest run`. Sets `CI=true` to disable watch mode.
4. **e2e** — runs `npm ci` (cached), deploys schema to a Convex dev project with `npx convex deploy`, seeds the database with `npx convex run seed`, installs Playwright browsers with `npx playwright install --with-deps`, then runs `npx playwright test` with `CLAUDE_MOCK=true`. Must run against all three browser projects (chromium, firefox, webkit).

**Caching strategy:** Use `actions/cache@v4` keyed on `${{ hashFiles('package-lock.json') }}` with `path: ~/.npm`. Each job restores the cache before `npm ci`.

**Convex deployment in CI:** Requires the `CONVEX_DEPLOY_KEY` secret to be set in the repository. Use `npx convex deploy --cmd 'echo deployed'` (or `npx convex deploy` with the key in env). Do NOT use `npx convex dev` — it is an interactive watch process, not suitable for CI.

**Secrets required:**
- `CONVEX_DEPLOY_KEY` — Convex project deploy key (set in GitHub repo secrets)
- `ANTHROPIC_API_KEY` — only needed if a non-mock integration test is ever added; not needed for the `CLAUDE_MOCK=true` suite

**Environment variables for e2e job:**
- `CLAUDE_MOCK=true` — activates the deterministic AI stub in all four AI actions
- `CONVEX_DEPLOY_KEY` — from secrets
- `VITE_CONVEX_URL` — set from the Convex deploy output (use `npx convex deploy --print-url` or capture from deployment step)

The `lint` and `typecheck` jobs have no dependency on each other or on `unit`; all three can run in parallel. The `e2e` job may be configured with `needs: [lint, typecheck, unit]` to fail fast if static checks are broken before spending E2E runtime, but this is optional — both strategies are valid.

Playwright config (from T34) already defines `chromium`, `firefox`, and `webkit` projects. The CI invocation should not limit projects; running `npx playwright test` with no `--project` flag picks up all three from `playwright.config.ts`.

The `npx convex run seed` step seeds an admin user and three templates as established in T11's `convex/seed.ts`. This must run after `npx convex deploy` and before `npx playwright test`.

## Test-Gen Brief

### Files
- Implementation: `.github/workflows/ci.yml`
- No Vitest tests for this task (YAML-only, no TypeScript)
- Validation: manual inspection of GitHub Actions run log + green status badge

### Exports
- No TypeScript exports. Artifact is the YAML workflow file.

### Signatures
- N/A (YAML configuration, not TypeScript)

### Queries used
- `npx convex deploy` — pushes schema and functions to Convex cloud
- `npx convex run seed` — executes `convex/seed.ts` on the deployed backend
- `npx playwright test` — runs all Playwright specs from `playwright.config.ts`
- `npx vitest run` — runs all Vitest unit tests in non-watch mode

### Invariants
- `npm ci` (not `npm install`) must be used in all jobs to ensure reproducible installs from `package-lock.json`.
- `CLAUDE_MOCK=true` must be set for the e2e job — the suite must never make live Anthropic API calls in CI.
- `npx convex dev` must NOT appear in any CI job — it is a foreground watch process.
- Cache key must be based on `package-lock.json` hash; a lock-file change invalidates the cache automatically.
- Playwright browser installation uses `--with-deps` to install system dependencies in the GitHub-hosted runner environment.

### Non-goals
- This ticket does NOT configure a deployment/release pipeline (no `npx convex deploy` to production).
- This ticket does NOT add additional test jobs beyond the four (lint, typecheck, unit, e2e).
- This ticket does NOT set up branch protection rules in GitHub — that is a manual repo settings step.
- This ticket does NOT configure Dependabot or security scanning.
- This ticket does NOT set up notifications (Slack, email) on failure.

### Tested by
- **AC: Four jobs on push to PR branches targeting main** — Integration: open a test PR and verify all four jobs appear in the GitHub Actions UI. Assert `on: pull_request: branches: [main]` in the YAML.
- **AC: lint, typecheck, unit pass independently** — Integration: each of these three jobs must appear as a separate job in the workflow run. Assert no `needs:` dependency between them (or only explicit sequencing). Verify each job uses `npm ci` with cache restore.
- **AC: e2e: deploy to Convex dev; seed; Playwright with CLAUDE_MOCK=true; multi-browser** — Integration: verify e2e job log shows `npx convex deploy` completing, `npx convex run seed` completing, and Playwright running against chromium/firefox/webkit with `CLAUDE_MOCK=true` in env. All three browser projects must appear in the test report.
- **AC: node_modules cached** — Integration: on a second run with no lock-file changes, verify the cache restore step shows "Cache hit" in all four job logs.

No unit-testable TypeScript in this task. All validation is integration-level via live CI runs.

## Stack Practices

- GitHub Actions cache: actions/cache@v4 with npm ci (not npm install).
- Convex deploy in CI: npx convex deploy using CONVEX_DEPLOY_KEY secret.
- Playwright in CI: npx playwright install --with-deps.
- Anti-pattern: do NOT run npx convex dev in CI — use npx convex deploy.
