## Implementation Notes

All four source modules are pure TypeScript (no Convex runtime imports, no SDK imports) defined in T5â€“T8:

- **T5 `convex/lib/privacyFilter.ts`** â€” `filterResponse(response, otherPartyMessages)` tokenizes by whitespace, pre-computes N-gram windows into a `Set<string>`, and checks for â‰¥8 consecutive token matches. Tests must cover: exact 8-token window match (passes false), 7-token window (passes true), zero-overlap benign input, empty arrays, single-word inputs, and adversarial cases that vary casing, extra whitespace, or partial punctuation.

- **T6 `convex/lib/prompts.ts`** â€” `assemblePrompt({ role, ... })` returns `{ system, messages }` for roles `PRIVATE_COACH`, `SYNTHESIS`, `COACH`, `DRAFT_COACH`. The no-quote rule must appear in every system prompt. `SYNTHESIS` must include an instruction to respond with ONLY a JSON object. Tests should assert these invariants against the string content of `system`.

- **T7 `convex/lib/transcriptCompression.ts`** â€” `compressIfNeeded(messages, budgetTokens, anthropicClient)` is async and accepts the Anthropic client as a DI parameter. Under budget: returns original array, zero API calls. Over budget: replaces the oldest 50% of messages with a single `SUMMARY` message. Content-hash caching: same input â†’ one API call regardless of how many times called. Tests use `vi.fn()` to mock the Anthropic client and assert call counts.

- **T8 `convex/lib/stateMachine.ts`** â€” exports one named function per transition (e.g., `transitionToJointActive`, `transitionToClosedAbandoned`). Each is pure: takes current status, returns new status or throws `ConvexError` with `CONFLICT` code. The decline-invite path (`DRAFT_PRIVATE_COACHING â†’ CLOSED_ABANDONED`) must be explicitly tested. All 9 valid transitions must have a passing test; â‰¥8 illegal transitions must verify the thrown error.

Vitest config: set `environment: 'node'` â€” no DOM needed. Test files live in `convex/lib/__tests__/`. Do NOT import from `convex/_generated/server` in any test file; that directory is generated at runtime by `npx convex dev` and is not available in the Vitest process.

The transcript compression mock should inject a `vi.fn()` that resolves to a canned string so tests remain synchronous-feeling and deterministic.

## Test-Gen Brief

**Files**

- Source: `convex/lib/privacyFilter.ts`, `convex/lib/prompts.ts`, `convex/lib/transcriptCompression.ts`, `convex/lib/stateMachine.ts`
- Tests: `convex/lib/__tests__/privacyFilter.test.ts`, `convex/lib/__tests__/prompts.test.ts`, `convex/lib/__tests__/transcriptCompression.test.ts`, `convex/lib/__tests__/stateMachine.test.ts`

**Exports**

- `filterResponse` â€” named, value, `(response: string, otherPartyMessages: string[]) => { passed: boolean; matchedSubstring?: string }`
- `assemblePrompt` â€” named, value, `(opts: AssemblePromptOpts) => { system: string; messages: AnthropicMessage[] }`
- `compressIfNeeded` â€” named, value, `async (messages: AnthropicMessage[], budgetTokens: number, client: AnthropicClient) => AnthropicMessage[]`
- Per-transition functions from `stateMachine.ts` â€” named, values, each with signature `(currentStatus: CaseStatus) => CaseStatus` (throws `ConvexError` on illegal transition)

**Signatures**

```typescript
// privacyFilter.ts
type FilterResult = { passed: boolean; matchedSubstring?: string };
function filterResponse(response: string, otherPartyMessages: string[]): FilterResult;

// prompts.ts

  messages: AnthropicMessage[],
  budgetTokens: number,
  client: AnthropicClient
): Promise<AnthropicMessage[]>;

function transitionToJointActive(status: CaseStatus): CaseStatus;
function transitionToClosedAbandoned(status: CaseStatus): CaseStatus;
// ... one per valid transition
```

**Queries used**

None â€” all four modules are pure TypeScript; no Convex DB or API calls.

**Invariants**

- `filterResponse` is synchronous and always returns an object with at least `passed`.
- `filterResponse` returns `passed: false` if and only if â‰¥8 consecutive whitespace tokens from `response` appear as a contiguous window in any message in `otherPartyMessages`.
- `assemblePrompt` always includes the no-quote rule string in `result.system` for all four roles.
- `assemblePrompt` with `role: 'SYNTHESIS'` instructs the model to reply with ONLY a JSON object â€” verified by substring check in `result.system`.
- `compressIfNeeded` never calls `client.messages.create` when `messages` is under `budgetTokens`.
- `compressIfNeeded` calls `client.messages.create` at most once per unique content-hash regardless of invocation count.
- State machine transition functions throw `ConvexError` (not a plain `Error`) with a `CONFLICT` code when the current status does not allow the requested transition.

**Non-goals**

- No integration tests against a live Convex deployment.
- No testing of AI prompt quality or Claude output format â€” only structural invariants.
- No tests for T9â€“T17 backend mutations or actions (those are integration-level).
- No browser environment testing â€” `environment: 'node'` throughout.
- Transcript compression does NOT test the Haiku model API response format â€” only call-count and return shape.
