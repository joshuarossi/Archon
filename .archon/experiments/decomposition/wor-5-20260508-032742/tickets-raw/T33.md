## Implementation Notes

T33 is a cross-cutting remediation task, not a feature. Every route (landing, login, invite, dashboard, case creation, case detail, private coaching, ready-for-joint, joint chat, draft coach, closure, solo mode, profile, admin templates, audit log) must be reviewed.

**axe-core integration:** Add `@axe-core/react` in `main.tsx` under a `process.env.NODE_ENV === 'development'` guard. Run it against every page in dev to produce a machine-readable violations list before manually remediating.

**Focus management pattern:** React Router v6 does not automatically move focus on navigation. Install a `useLocation` subscription in `App.tsx` that calls `document.querySelector('h1')?.focus()` after each route transition. Every page `<h1>` must have `tabIndex={-1}` so it receives programmatic focus without appearing in the tab order.

**Modal / Sheet focus traps:** shadcn/ui `Dialog` and `Sheet` use Radix UI `FocusScope` internally â€” do not reimplement. Verify correct trigger-to-focus-return behavior for: PrivacyBanner dialog, MarkComplete `AlertDialog`, CrisisCard (not a modal â€” no focus trap), `DraftCoachPanel` (Sheet), closure modal, version comparison modal, audit log drawer. The Esc key must close each and return focus to the trigger button.

**Chat announcements (role='log'):** `ChatWindow` must have `role='log' aria-live='polite' aria-relevant='additions'`. The STREAMING Coach bubble adds `role='status' aria-label='Coach is replying'` to announce the reply once; when the message reaches COMPLETE, remove `role='status'`. User-authored messages must NOT be announced (they originate from the user).

**Draft Coach panel (T28 aside):** The panel is `<aside>` (complementary content), not a `Dialog`. It must NOT globally trap focus â€” users should be able to tab back to the joint chat behind it. When the panel opens, `useEffect` programmatically focuses the draft `<textarea>`. Esc closes and returns focus to the joint-chat `MessageInput`.

**Color contrast audit:** Check every `--text-*` / `--bg-*` combination defined in T18. Critical pairs: `--text-secondary` (#5C5952) on `--bg-canvas` (#FAF8F5), `--text-tertiary` (#8A8680) on `--bg-surface` (#FFFFFF), `--accent` (#6B8E7F) on `--bg-surface`, `--danger` (#B5594D) on `--bg-canvas`. Use the `are-you-ok` CLI or browser DevTools contrast picker. Both light and dark mode palettes must pass. Any token failing 4.5:1 for normal text (or 3:1 for large text â‰Ą18px or â‰Ą14px bold) must be adjusted.

**Focus ring:** `src/index.css` must include a global `:focus-visible { outline: 2px solid var(--accent); outline-offset: 2px; }` rule. `outline: none` is only permissible on elements that supply a custom `:focus-visible` replacement.

**prefers-reduced-motion:** T18 set transition durations to 0ms under the `@media (prefers-reduced-motion: reduce)` guard. T33 verifies consistent application: message arrival animation, streaming cursor blink, panel slide-in, route crossfade, skeleton shimmer, DraftCoachPanel slide-in. Run the audit with the OS reduce-motion setting active.

**Keyboard critical path:** Tab through landing â†’ login â†’ dashboard â†’ new case (all 5 steps) â†’ invite page â†’ private coaching (send a message, mark complete) â†’ ready for joint (enter) â†’ joint chat (send message, open Draft Coach, send draft, propose closure). Every action must be completable without a mouse.

**Landmark structure:**
- `<main>`: exactly one per page, wrapping primary content
- `<nav>`: PhaseHeader navigation (back link + phase label)
- `<aside>`: DraftCoachPanel (complementary content)
- Admin tables: `<th scope='col'>` on all column headers
- Filter input on audit log page: `aria-label='Filter by actor email'`
