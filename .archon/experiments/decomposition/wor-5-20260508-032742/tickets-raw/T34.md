# Playwright E2E Test Suite

The Conflict Coach application requires a comprehensive end-to-end test suite that validates all critical user journeys in a multi-browser environment without hitting live external APIs. This task implements all 7 Playwright spec files against a Convex dev deployment, using a `CLAUDE_MOCK=true` stub to replace Anthropic calls with deterministic responses. The suite provides the last safety gate before CI deploys to production and is the definitive proof that the full mediation flow works end-to-end for real users.

## Acceptance Criteria

- CLAUDE_MOCK=true: deterministic stub with configurable delays; no live Anthropic calls
- All 7 spec files pass 100% with CLAUDE_MOCK=true
- Three browser projects; axe-core scan in solo-full-flow

## Implementation Notes

The test suite lives under `e2e/` at the repo root. Playwright config (`playwright.config.ts`) defines three browser projects — chromium, firefox, webkit — all running against a Convex dev deployment seeded with an admin user and baseline templates (via `convex/seed.ts` from T11).

**CLAUDE_MOCK stub**: Each Convex AI action (`generateAIResponse`, `synthesis/generate`, `generateCoachResponse`, `generateDraftResponse`) checks `process.env.CLAUDE_MOCK === 'true'` at the top of the function body and returns a canned response object instead of calling the Anthropic SDK. The stub must support configurable delays (via `CLAUDE_MOCK_DELAY_MS` env var, defaulting to 100ms) to simulate streaming latency without flaking. The stub must produce structurally correct output: for SYNTHESIS, a valid `{ forInitiator: "...", forInvitee: "..." }` JSON; for DRAFT_COACH, a `{ "draft": "..." }` JSON; for PRIVATE_COACH and COACH, plain prose strings that do not trigger the privacy filter against themselves.

**The 7 spec files:**

1. **`auth.spec.ts`** — magic link registration (mock email capture via Mailhog or Convex-captured token), logout, re-login. Google OAuth uses Playwright's route interception to mock the OAuth redirect. Covers US-01, US-02.

2. **`solo-full-flow.spec.ts`** (highest-value) — creates a solo case, toggles to Initiator view, completes private coaching, marks complete, switches to Invitee view, completes private coaching, marks complete, waits for synthesis, enters joint chat, exchanges messages with Draft Coach assistance, proposes and confirms closure, verifies the case appears in Closed section. This single spec validates the entire mediation lifecycle. Includes an `@axe-core/playwright` scan after each major navigation step.

3. **`invite-flow.spec.ts`** — two separate `browser.newContext()` instances (User A and User B). User A creates a case, copies the invite link. User B opens the link in their context, registers (magic-link mocked), accepts the invite. Both contexts verify the case appears in their respective dashboards. Tests the two-context concurrency path.

4. **`draft-coach.spec.ts`** — with a joint chat in progress, opens Draft Coach panel, exchanges messages, clicks "Generate Draft" (posts canonical user message), verifies that clicking "Generate Draft" does NOT send to joint chat, verifies that only "Send this message" posts to joint chat, discards a draft and verifies it does not appear in joint chat.

5. **`privacy.spec.ts`** (security test) — attempts to read another user's `privateMessages` via a direct `convex dev` function call (using Playwright's page.evaluate to invoke `convex.query(api.privateCoaching.myMessages, { caseId })`). Expects FORBIDDEN. Tests same as admin user. Also verifies the privacy filter: inserts known private message text, calls synthesis, verifies Coach response does not contain an 8-token substring match.

6. **`admin-templates.spec.ts`** — logs in as admin (seeded from T11 seed), creates a template, publishes v2, verifies a case pinned to v1 still uses the old version, archives the template and verifies it no longer appears in the case creation category picker while a pinned case still functions.

7. **`state-machine.spec.ts`** — attempts to send a joint message when the case is in DRAFT_PRIVATE_COACHING status (expects CONFLICT error surfaced in the UI), marks private coaching complete twice (idempotent — no error or duplicate state change), redeems a consumed invite token (expects TOKEN_INVALID message).

**Test setup helpers**: `e2e/fixtures/` exports reusable Playwright fixtures for authenticated pages, seeded cases, and solo-case state at various phases. `e2e/helpers/mockClaude.ts` exports utilities to assert that no live Anthropic calls were made (by monitoring network requests to `api.anthropic.com`).

**Anti-patterns**: never use `page.waitForTimeout` — use `page.waitForSelector`, `expect(locator).toBeVisible()`, or Playwright's `waitForResponse`. Never couple tests by shared Convex state — each test that mutates data should create its own case.

## Test-Gen Brief

### Files
- `playwright.config.ts` — project root, three browser projects, base URL pointing to dev server
- `e2e/auth.spec.ts`
- `e2e/solo-full-flow.spec.ts`
- `e2e/invite-flow.spec.ts`
- `e2e/draft-coach.spec.ts`
- `e2e/privacy.spec.ts`
- `e2e/admin-templates.spec.ts`
- `e2e/state-machine.spec.ts`
- `e2e/fixtures/index.ts` — shared Playwright fixtures (authenticatedPage, soloCase, etc.)
- `e2e/helpers/mockClaude.ts` — helper to assert no live Anthropic API calls
- `convex/lib/mockClaudeStub.ts` — stub logic used inside CLAUDE_MOCK branch of each AI action

### Exports
- `playwright.config.ts`: `export default defineConfig({ ... })` (default)
- `e2e/fixtures/index.ts`:
  - `test` — extended Playwright test object with fixtures (named)
  - `expect` — re-export from base (named)
  - `AuthenticatedPage` — fixture type (type)
  - `SeedFixtures` — fixture type (type)
- `convex/lib/mockClaudeStub.ts`:
  - `getMockPrivateCoachResponse(): string` (named)
  - `getMockSynthesisResponse(): { forInitiator: string; forInvitee: string }` (named)
  - `getMockCoachResponse(): string` (named)
  - `getMockDraftResponse(): { draft: string }` (named)
  - `MOCK_DELAY_MS: number` (named, reads `process.env.CLAUDE_MOCK_DELAY_MS ?? '100'`)

### Signatures
```typescript
// convex/lib/mockClaudeStub.ts

```

### Queries used
- `api.privateCoaching.myMessages` — tested in privacy.spec.ts via page.evaluate
- `api.cases.list` — asserted in invite-flow.spec.ts for both contexts
- `api.cases.get` — asserted in state-machine.spec.ts
- `api.invites.redeem` — asserted in state-machine.spec.ts (consumed token)
- `api.jointChat.messages` — asserted in draft-coach.spec.ts and solo-full-flow.spec.ts

### Invariants
- `CLAUDE_MOCK=true` must produce zero outbound requests to `api.anthropic.com` — verified by monitoring network in solo-full-flow.spec.ts
- Each spec file must be independently runnable (no cross-spec state dependency)
- Stub responses must not trigger the privacy filter against themselves (8-token substring match must not fire)
- The axe-core scan in solo-full-flow must produce zero violations with impact `critical` or `serious`
- `page.waitForTimeout` must not appear anywhere in the spec files

### Non-goals
- Performance benchmarks or load testing
- Live Anthropic API calls in any spec (always mocked)
- Testing UI visual regressions (no screenshot comparison)
- Testing mobile viewports (Playwright projects are desktop only in v1)
- Testing the Python/node Convex CLI behavior — only the web app UI

## Stack Practices

- Playwright config: three projects (chromium, firefox, webkit).
- CLAUDE_MOCK stub: check process.env.CLAUDE_MOCK === 'true' at top of each AI action.
- Two-context invite test: browser.newContext() per user.
- axe-core in Playwright: @axe-core/playwright after each major navigation.
- Anti-pattern: do NOT use page.waitForTimeout — use page.waitForSelector or expect(locator).toBeVisible().
