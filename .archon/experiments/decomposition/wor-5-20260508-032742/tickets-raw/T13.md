# AI Synthesis Backend (generate action, privacy filter, state transition)

Once both parties have marked private coaching complete, the system must bridge their separate experiences into a shared starting point for the joint session — without ever surfacing one party's raw words to the other. This ticket implements the `synthesis/generate` internal action that reads both parties' private messages, calls Claude Sonnet to produce two independent synthesis texts, runs the privacy filter on each, and writes the results to the database while advancing the case to `READY_FOR_JOINT`. This is the only moment in the product where both parties' private content is visible in the same execution context, making correct isolation and filter enforcement load-bearing for the privacy guarantee.

## Acceptance Criteria

- Calls claude-sonnet-4-5; validates { forInitiator, forInvitee } JSON
- Runs filterResponse on each half; retries up to 2x on failure
- On success: writes synthesisText to both partyStates; sets case READY_FOR_JOINT
- Idempotent: if synthesisText already populated, returns without re-generating

## Implementation Notes

`synthesis/generate` must be declared as an `internalAction` — it is only ever called from `privateCoaching/markComplete` (via `ctx.scheduler.runAfter` or `ctx.runAction`) when both parties' `privateCoachingCompletedAt` fields are set. It must never be a public action; the client has no direct path to invoke it.

**Reading both parties' context.** The action calls `ctx.runQuery` on an internal helper that returns each party's full private message history and their form fields (mainTopic, description, desiredOutcome). Both are passed to `assemblePrompt({ role: "SYNTHESIS", ... })` from `convex/lib/prompts.ts` (T6). The assembled prompt includes the verbatim instruction: *"You have access to both parties' private content for context. In your outputs, NEVER quote, closely paraphrase, or otherwise surface the other party's raw words."*

**Calling Claude.** Use the `@anthropic-ai/sdk` Anthropic client with a non-streaming call (`messages.create` without `stream: true`). Model: `claude-sonnet-4-5`. The prompt instructs the model to respond with ONLY the JSON object — no surrounding prose. Parse the response text as JSON; if JSON.parse throws, treat it as a format failure and retry.

**Privacy filter.** After parsing, call `filterResponse(forInitiator, inviteePrivateMessages)` and `filterResponse(forInvitee, initiatorPrivateMessages)` from `convex/lib/privacyFilter.ts` (T5). Each returns `{ passed, matchedSubstring? }`. Both must pass. If either fails, increment the retry counter and regenerate — do not mix a passing half with a failing half across retries.

**Retry loop.** Use a `for (let attempt = 0; attempt < 3; attempt++)` loop (0, 1, 2 = up to 3 attempts total, 2 retries after the first). Never recurse. On all three failures, write a generic fallback synthesis to both partyStates: `"Our coach encountered a brief issue preparing your guidance. You can still proceed to the joint session."` Flag the case for admin review by writing an `auditLog` row with `action: "SYNTHESIS_FALLBACK"`.

**CLAUDE_MOCK guard.** If `process.env.CLAUDE_MOCK === 'true'`, skip the Anthropic call and return canned synthesis strings immediately, same as T12.

**Persistence.** Write results via `ctx.runMutation(internal.synthesis._persistSynthesis, { caseId, forInitiator, forInvitee })`. This internal mutation atomically writes `synthesisText` + `synthesisGeneratedAt` to both `partyStates` rows and sets `cases.status = "READY_FOR_JOINT"`. Keeping this in a mutation preserves Convex's transactional guarantee — if the write fails partway, neither partyState is updated.

**Idempotency.** At the top of `synthesis/generate`, call a query that checks whether both partyStates already have `synthesisText` populated. If so, return immediately without calling Claude. This prevents double-billing if the action is accidentally re-scheduled.

**File locations.** All synthesis functions live in `convex/synthesis.ts`. The internal persistence mutation is `internal.synthesis._persistSynthesis`.

**Exports**

- `internal.synthesis._persistSynthesis` — internalMutation, registered via `internalMutation()` wrapper
- `internal.synthesis._isSynthesisComplete` — internalQuery checking both partyStates for existing synthesisText

**Signatures**
```ts
// internalAction
export const generate = internalAction({
  args: { caseId: v.id("cases") },
  handler: async (ctx, { caseId }): Promise<void>
});

// internalMutation
export const _persistSynthesis = internalMutation({
  args: {
    caseId: v.id("cases"),
    forInitiator: v.string(),
    forInvitee: v.string(),
  },
  handler: async (ctx, args): Promise<void>
});

// internalQuery
export const _isSynthesisComplete = internalQuery({
  args: { caseId: v.id("cases") },
  handler: async (ctx, { caseId }): Promise<boolean>
});
```

**Queries used**
- `ctx.runQuery(internal.synthesis._isSynthesisComplete, { caseId })` — idempotency check
- `ctx.runMutation(internal.synthesis._persistSynthesis, { caseId, forInitiator, forInvitee })` — persist results
- `ctx.runQuery(internal.cases._getBothPartiesContext, { caseId })` — fetch both parties' private messages and form fields (internal query, may be defined in `convex/cases.ts` or `convex/synthesis.ts`)
- `filterResponse(text, otherPartyMessages)` from `convex/lib/privacyFilter.ts`
- `assemblePrompt({ role: "SYNTHESIS", ... })` from `convex/lib/prompts.ts`
- Anthropic SDK: `client.messages.create({ model: "claude-sonnet-4-5", ... })`

**Invariants**
- `forInitiator` is never filtered against initiator's own messages — only against invitee's messages, and vice versa
- The retry counter never exceeds 3 total attempts (loop index 0..2)
- `_persistSynthesis` is never called unless both halves passed the privacy filter (or we are writing the fallback)
- `cases.status` is set to `"READY_FOR_JOINT"` atomically with `partyStates.synthesisText` writes — they never diverge
- If `_isSynthesisComplete` returns true, no Anthropic call is made and no mutations are run
- `synthesis/generate` is never a public action — it is only callable via `ctx.scheduler` or `ctx.runAction` from server-side code

**Non-goals**
- This ticket does NOT implement streaming for synthesis (tech spec TQ3: one-shot, non-streaming)
- This ticket does NOT implement admin review UI for flagged fallback cases
- This ticket does NOT implement cost tracking on the synthesis call — that is part of T17
- This ticket does NOT implement the `privateCoaching/markComplete` scheduling logic — that belongs to T12; T13 assumes the action is already being invoked correctly
- This ticket does NOT define the `assemblePrompt` or `filterResponse` implementations — those are T6 and T5 respectively

## Stack Practices

- synthesis/generate is an internalAction only.
- Retry loop: for loop 0..2, never use recursion.
- Anti-pattern: do NOT call the Anthropic API inside a Convex mutation.
