# Private coaching AI action ‚Äî convex/privateCoaching/generateAIResponse.ts (US-06, US-07, FR-04)

Phase 1 establishes the core private coaching loop: a user sends a message, the AI coach responds in real time, and the response is visible only to that user. This task implements the Convex action that orchestrates the full AI call pipeline ‚Äî cost check ‚Üí compression ‚Üí prompt assembly ‚Üí streaming ‚Üí spend recording ‚Äî and wires the `PrivateCoachingView` React page to display the conversation using the shared `ChatWindow` component. It is the first live Claude integration in the codebase and sets the pattern every subsequent AI action will follow.

## Acceptance Criteria

- generateAIResponse action: before calling Claude, calls `checkCostLimit(ctx, caseId)` from `convex/lib/costTracking.ts`; if `{ limited: true }` is returned, throws `RATE_LIMITED` immediately without invoking Claude or inserting a new STREAMING row.
- Before constructing the message array, calls `compressIfNeeded(history, 60000)` from `convex/lib/compression.ts`; the returned (possibly compressed) array is what gets passed to `assemblePrompt` and onward to the SDK.
- Inserts a `privateMessages` row with `status: STREAMING`, calls `assemblePrompt` with role=PRIVATE_COACH on the (possibly compressed) history, calls Claude Sonnet (`claude-sonnet-4-5`) with streaming, batches token updates to the row every ~50ms, sets `status: COMPLETE` with `tokens` count on completion.
- After streaming completes successfully, calls `recordSpend(ctx, caseId, tokensUsed)` from `convex/lib/costTracking.ts` to accumulate token usage.
- PrivateCoachingView renders the party's private chat using the shared ChatWindow component (T9); only the calling user's private messages are fetched and rendered.
- On Claude API 429, the action retries once after 2 seconds; on second failure, sets message `status: ERROR`.
- On network timeout (>30s) or other non-retryable error, sets message `status: ERROR` with the applicable error code.
- A direct query for another user's private messages by any authenticated user returns FORBIDDEN (privacy enforcement at query layer).
- Vitest unit test: the Convex query `privateCoaching/messages` called by a user who is not the owner of the requested messages returns FORBIDDEN.
- Playwright (`privacy.spec.ts`, PRD ¬ß7.1 launch criterion #2, Test 1 ‚Äî private message isolation): using two browser contexts, user B directly calls the `privateCoaching/messages` query for user A's caseId+userId; asserts the response is FORBIDDEN (hard access denial, not an empty array).
- Playwright: in solo mode, messages sent as initiator do not appear when toggled to invitee's private coaching view.

## Implementation Notes

`generateAIResponse` is a **Convex action** ‚Äî the only function type that may call external APIs. It never writes to the DB directly; all DB mutations happen via `ctx.runMutation()`. The action follows a strict ordered pipeline:

1. **Cost guard** ‚Äî call `checkCostLimit(ctx, caseId)` from `convex/lib/costTracking.ts`. If `limited: true`, throw `createError('RATE_LIMITED', ...)` immediately. No STREAMING row should be inserted yet; failing early saves a wasted DB write.
2. **User message persistence** ‚Äî insert the user's `privateMessages` row (`role: 'user'`, `status: 'COMPLETE'`) via mutation.
3. **Compression** ‚Äî call `compressIfNeeded(history, 60000)` from `convex/lib/compression.ts` on the full message history. Pass the returned array (possibly containing a leading summary message) to `assemblePrompt`. Do not pass the raw history.
4. **Prompt assembly** ‚Äî call `assemblePrompt({ role: 'PRIVATE_COACH', caseId, actingUserId, recentHistory })` from `convex/lib/prompts.ts`. The returned `{ system, messages }` is the complete Claude payload.
5. **STREAMING row insertion** ‚Äî insert a `privateMessages` row with `status: 'STREAMING'` before the `client.messages.stream()` call. This row's `_id` is the target for all subsequent patch mutations during streaming.
6. **Claude streaming** ‚Äî use `client.messages.stream()` with model `claude-sonnet-4-5`. Accumulate token deltas; batch-write content to the STREAMING row every ~50ms via `ctx.runMutation()`. On stream end, set `status: 'COMPLETE'` and write the final `tokens` count from `usage`.
7. **Spend recording** ‚Äî call `recordSpend(ctx, caseId, tokensUsed)` from `convex/lib/costTracking.ts` after the Claude call completes successfully.
8. **Error handling** ‚Äî on 429, wait 2 seconds (use `await new Promise(r => setTimeout(r, 2000))`) and retry once. On second 429, or on any 400/401/timeout, call `ctx.runMutation()` to set the STREAMING row's `status: 'ERROR'`. Never retry 401 (invalid key) or 400 (bad request).

**Privacy enforcement lives at the query layer.** The `privateCoaching/messages` query must resolve the caller's identity via `requireAuth(ctx)` and then compare the authenticated userId against the `partyStates` owner for the requested `caseId`. Any mismatch throws `createError('FORBIDDEN', ...)`. This must be enforced server-side even if the client never calls it wrong ‚Äî it is the PRD ¬ß7.1 launch criterion #2 boundary.

**PrivateCoachingView** is a React route component that wraps the shared `ChatWindow` from T9. It subscribes to `useQuery(api.privateCoaching.messages, { caseId })` ‚Äî which already scopes to the calling user by design ‚Äî and passes the result as the `messages` prop. The `onRetry` callback re-invokes `useMutation(api.privateCoaching.generateAIResponse)` with the failed message's id. It does not manage any streaming state itself; the reactive query handles updates automatically as the STREAMING row is patched.

**Solo mode isolation**: in solo mode, both `partyStates` rows belong to the same userId, but are distinguished by `role: 'INITIATOR'` vs `role: 'INVITEE'`. The `privateCoaching/messages` query must filter by both `caseId` AND the acting `partyStatesId` (or equivalent discriminator), not just `userId`, so toggling the `?as=` param changes which party's messages are fetched.

Model constants should live in `convex/lib/models.ts` (create if not already present): `export const SONNET = 'claude-sonnet-4-5'`. Import from there ‚Äî never inline the string.

**Queries used**
- `api.privateCoaching.generateAIResponse` ‚Äî called via `useMutation` in PrivateCoachingView
- `api.privateCoaching.messages` ‚Äî called via `useQuery` in PrivateCoachingView
- `checkCostLimit(ctx, caseId)` from `convex/lib/costTracking.ts` ‚Äî called at action start
- `recordSpend(ctx, caseId, tokensUsed)` from `convex/lib/costTracking.ts` ‚Äî called after streaming completes
- `compressIfNeeded(history, 60000)` from `convex/lib/compression.ts` ‚Äî called before `assemblePrompt`
- `assemblePrompt({ role: 'PRIVATE_COACH', caseId, actingUserId, recentHistory })` from `convex/lib/prompts.ts` ‚Äî called after compression
- `client.messages.stream(...)` from `@anthropic-ai/sdk` ‚Äî only inside the action, model `claude-sonnet-4-5`

**Invariants**
- `checkCostLimit` is always called before any Claude invocation; if `limited: true`, no STREAMING row is created and no Anthropic call is made.
- `compressIfNeeded` is always called before `assemblePrompt`; the compressed (or original) history is what reaches the SDK.
- The STREAMING row is inserted before `client.messages.stream()` starts; it is always resolved to `COMPLETE` or `ERROR` ‚Äî never left in `STREAMING` state.
- `recordSpend` is called if and only if the Claude call completes without error.
- `messages` query returns only rows where the authenticated caller is the party owner of the requested `caseId`; any other caller receives `FORBIDDEN`, not an empty array.
- In solo mode, toggling `?as=` changes the acting partyState; each party's messages are fetched and displayed independently.

**Non-goals**
- No joint messages (`jointMessages` table is untouched by this task).
- No synthesis triggering (that is T11's `markComplete` mutation).
- No soft-cap detection or boilerplate response (that is T17b/T17c in Phase 2; `checkCostLimit` in Phase 1 returns only `limited` boolean).
- No compression implementation (T10c owns `compressIfNeeded`).
- No prompt assembly changes (T10a owns `assemblePrompt`).
- No privacy filter (`privacyFilter.ts` is not called by private coaching ‚Äî only by synthesis and Coach in Phase 2).
- No audit log writes for normal coaching (audit log is written on auth errors per tech spec ¬ß6.5, but not for normal 429 retries).
