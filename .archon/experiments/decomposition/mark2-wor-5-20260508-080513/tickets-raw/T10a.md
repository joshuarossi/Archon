# Shared prompt assembly module — convex/lib/prompts.ts (all 4 AI roles)

`convex/lib/prompts.ts` is the privacy-critical backbone of every AI interaction in Conflict Coach. It consolidates the four distinct role configurations—PRIVATE_COACH, COACH, DRAFT_COACH, and SYNTHESIS—into one auditable module with deterministic context injection rules. Phase 1 needs this module first because T10b (private coaching action), T14 (synthesis), T17b (Coach action), and T19 (Draft Coach action) all delegate system prompt and message-array construction to it. Centralizing these rules prevents the fragmentation and drift that would arise if each action hand-rolled its own context assembly.

## Acceptance Criteria

- `prompts.ts` exports `assemblePrompt({ role, caseId, actingUserId, recentHistory, templateVersion? }): { system: string, messages: MessageParam[] }`; all four roles are handled with correct context injection rules per tech spec §6.3.
- PRIVATE_COACH role: system prompt matches the verbatim text from tech spec §6.3.1; context includes only the acting party's form fields and message history — never the other party's privateMessages, partyStates form fields, or synthesisText.
- SYNTHESIS role: context includes both parties' form fields and private coaching histories per tech spec §6.3.2; system prompt contains the verbatim anti-quotation critical instruction.
- COACH role: context includes joint chat history + both parties' synthesisText only; does NOT include any raw privateMessages.
- DRAFT_COACH role: context includes the drafting user's joint-chat history and their own synthesisText only; does NOT include the other party's synthesisText or any privateMessages.
- Vitest unit test: `assemblePrompt` with role=PRIVATE_COACH and a populated partyStates does not include the string 'otherParty' or any other-party identifier in the returned context.
- Vitest unit test: `assemblePrompt` with role=DRAFT_COACH does not include the other party's synthesisText in the returned context.
- TypeScript strict mode: all returned message arrays are typed as `MessageParam[]` compatible with `@anthropic-ai/sdk`.

## Implementation Notes

`convex/lib/` is the home for business logic with zero Convex runtime dependency. This module receives all data it needs as plain arguments—no `ctx`, no DB access—which makes it purely unit-testable in Vitest without `convex-test`. The four calling actions are responsible for fetching data from the DB and for calling `compressIfNeeded` (T10c) before passing `recentHistory` in; this module just consumes what it receives.

**Context injection summary per role (tech spec §6.3):**

| Role | System prompt source | Messages may include | Messages must never include |
|------|---------------------|---------------------|-----------------------------|
| PRIVATE_COACH | §6.3.1 verbatim | Acting party's form fields + their privateMessages | Other party's form fields, privateMessages, synthesisText |
| SYNTHESIS | §6.3.2 verbatim (with anti-quotation block) | Both parties' form fields + both privateMessage histories | Nothing beyond those |
| COACH | §6.3.x | Joint chat history + both parties' synthesisText | Any raw privateMessages |
| DRAFT_COACH | §6.3.x | Drafting user's joint chat history + their own synthesisText | Other party's synthesisText, any privateMessages |

The optional `templateVersion?` parameter allows callers to splice template-specific instructions into the system prompt (e.g., `globalGuidance`, `coachInstructions`, `draftCoachInstructions` fields from the pinned `templateVersions` row). When absent, fall back to the baseline system prompt verbatim from the tech spec.

Use `MessageParam[]` from `@anthropic-ai/sdk/resources/messages` directly for the return type—do not define a parallel interface. Use `Doc<"partyStates">`, `Doc<"privateMessages">`, etc. from `convex/_generated/dataModel` for input types to keep everything in sync with the schema from T1.

## Test-Gen Brief

### Contract

**Files:**
- Implementation: `convex/lib/prompts.ts`
- Tests: `convex/lib/prompts.test.ts`

**Exports:**
- `assemblePrompt` — named export, function value (not async; no I/O)

**Signatures:**
```typescript
import type { MessageParam } from "@anthropic-ai/sdk/resources/messages";
import type { Doc, Id } from "../_generated/dataModel";

type AssemblePromptRole = "PRIVATE_COACH" | "COACH" | "DRAFT_COACH" | "SYNTHESIS";

type AssemblePromptArgs = {
  role: AssemblePromptRole;
  caseId: Id<"cases">;
  actingUserId: Id<"users">;
  recentHistory: MessageParam[];
  templateVersion?: Doc<"templateVersions">;
  selfPartyState?: Doc<"partyStates">;
  otherPartyState?: Pick<Doc<"partyStates">, "synthesisText" | "mainTopic" | "description" | "desiredOutcome">;
  otherPartyHistory?: MessageParam[]; // SYNTHESIS only
};

export function assemblePrompt(args: AssemblePromptArgs): {
  system: string;
  messages: MessageParam[];
};
```

**Queries used:** None — pure function; no Convex DB access, no Anthropic API calls.

**Invariants:**
- PRIVATE_COACH: serialized output never contains the other party's form fields or privateMessages content
- DRAFT_COACH: serialized output never contains `otherPartyState.synthesisText`
- COACH: serialized output contains zero raw privateMessages content
- SYNTHESIS: `system` string always contains the verbatim anti-quotation critical instruction from tech spec §6.3.2
- Return type is always `{ system: string; messages: MessageParam[] }` — no `any` casts

**Non-goals:**
- This module does NOT call the Anthropic SDK
- This module does NOT access the Convex DB (no `ctx` parameter)
- This module does NOT handle authorization (callers own auth via `requireAuth`)
- This module does NOT compress message history (T10c's `compressIfNeeded` is called by the action before passing `recentHistory`)
- This module does NOT validate that the `caseId` or `actingUserId` exist
