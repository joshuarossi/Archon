# Cost-tracking stub — convex/lib/costTracking.ts (Phase 1 interface for all AI actions)

`costTracking.ts` establishes the cost-tracking contract that every AI action in the app calls before and after invoking Claude. In Phase 1, the enforcement logic is intentionally minimal — `checkCostLimit` reads the `costLimited` flag set by a future Phase 2 task (T17c) and returns `{ limited: true }` if it is set, `{ limited: false }` otherwise; `recordSpend` is a real accumulator that writes token and USD totals to the `cases` row. This stub-first approach means T10b, T14, T17b, and T19 can all wire their cost-tracking calls now — against stable, typed signatures — without any Phase 2 concern bleeding into Phase 1. T17c later replaces the threshold logic inside `checkCostLimit` without changing any callers. The three cost fields this module writes to (`costUsd`, `costTokens`, `costLimited`) were added to the `cases` table in T1 precisely to support this module.

## Acceptance Criteria

- `costTracking.ts` exports `checkCostLimit(ctx: ActionCtx, caseId: Id<'cases'>): Promise<{ softCapReached?: boolean, limited: boolean }>` — in Phase 1 implementation, reads `cases.costLimited` from the DB; returns `{ limited: true }` if the flag is set, `{ limited: false }` otherwise. The `softCapReached` field is absent in the Phase 1 stub; T17c adds it without changing the callers.
- `costTracking.ts` exports `recordSpend(ctx: ActionCtx, caseId: Id<'cases'>, tokensUsed: number): Promise<void>` — atomically increments `cases.costTokens` by `tokensUsed` and increments `cases.costUsd` by the corresponding USD amount (Sonnet pricing per token, defined as a constant in the module).
- Both functions are callable from Convex actions; they internally run a mutation to read/update the `cases` row atomically.
- TypeScript strict mode: all function signatures use the exact types from `convex/schema.ts`; no `any` casts.
- Vitest unit test: calling `recordSpend` twice accumulates the sum in `cases.costTokens` and `cases.costUsd`.
- Vitest unit test: `checkCostLimit` returns `{ limited: true }` when `cases.costLimited` is set to `true`, and `{ limited: false }` when the field is absent or false.

## Implementation Notes

**Convex function type.** Both exported functions are helpers called from Convex actions. They must not be bare async functions — they must schedule their DB reads and writes through `ctx.runQuery()` and `ctx.runMutation()` respectively, since actions cannot directly touch the database. Define an internal mutation (e.g., `internal.costTracking.accumulateSpend`) that receives `caseId`, `deltaTokens`, and `deltaUsd` and performs a `ctx.db.patch()`. Define an internal query (e.g., `internal.costTracking.getCostLimited`) that reads `cases.costLimited`. Export the action-level helpers as plain TypeScript functions that accept `ActionCtx` and call these internals — do not register them as top-level Convex queries/mutations themselves.

**Atomic accumulation.** `recordSpend` increments two fields: `cases.costTokens` and `cases.costUsd`. The increment must be atomic — read the current values and patch in one mutation call. Use `ctx.db.get(caseId)` inside the mutation, add the deltas, and `ctx.db.patch(caseId, { costTokens: current + delta, costUsd: currentUsd + deltaUsd })`. Both fields are declared `v.optional(v.number())` in the schema, so treat `undefined` as `0` when reading.

**Sonnet pricing constant.** Define `SONNET_USD_PER_TOKEN` as a module-level constant. Use Sonnet 4.5 pricing (input + output blended average is acceptable for the stub; the exact value is less important than having a single, named constant that T17c can later recalibrate). A reasonable placeholder: `3e-6` (USD per token at $3/1M tokens). Document the source pricing in a comment.

**Type imports.** Use `Id<'cases'>` from `convex/schema.ts` generated types. Use `ActionCtx` from `convex/server`. Never define parallel interfaces for what the generated types already provide.

**Phase 1 `checkCostLimit` behavior.** The function reads `cases.costLimited` only. It does NOT read `cases.costUsd` or compare against any threshold — that logic belongs to T17c. Return `{ limited: cases.costLimited ?? false }`. The `softCapReached` key must be absent (not `softCapReached: false`) so T17c can add it to the return type without triggering TypeScript errors in callers that use property-access narrowing.

**Testing with `convex-test`.** Both unit tests require the in-memory Convex runtime because they exercise mutations against the real schema and indexes. Use `convex-test` (not mocked internals). Seed a `cases` row in each test before calling the helpers.

## Test-Gen Brief

### Contract

**Files**
- Implementation: `convex/lib/costTracking.ts`
- Internal mutation (read/write): `convex/lib/costTracking.ts` (internal helpers co-located, registered via `internalMutation`/`internalQuery`)
- Tests: `convex/lib/costTracking.test.ts`

**Exports**
- `checkCostLimit` — named export, async function value (not a Convex query/mutation registration)
- `recordSpend` — named export, async function value (not a Convex query/mutation registration)
- `SONNET_USD_PER_TOKEN` — named export, `number` constant

**Signatures**
```typescript
import type { ActionCtx } from 'convex/server';
import type { Id } from '../_generated/dataModel';

export async function checkCostLimit(
  ctx: ActionCtx,
  caseId: Id<'cases'>,
): Promise<{ softCapReached?: boolean; limited: boolean }>;

export async function recordSpend(
  ctx: ActionCtx,
  caseId: Id<'cases'>,
  tokensUsed: number,
): Promise<void>;

export const SONNET_USD_PER_TOKEN: number;
```

**Queries used**
- `internal.costTracking.getCostLimited` — reads `cases.costLimited` for the given caseId; used by `checkCostLimit`
- `internal.costTracking.accumulateSpend` — patches `cases.costTokens` and `cases.costUsd`; used by `recordSpend`

**Invariants**
- `recordSpend` must never reduce `cases.costTokens` or `cases.costUsd` — only add.
- `checkCostLimit` must not write to the database in Phase 1; it is read-only.
- `cases.costLimited = undefined` and `cases.costLimited = false` are both treated as `{ limited: false }`.
- `SONNET_USD_PER_TOKEN` must be a positive finite number.
- Both functions must be callable from a Convex action context (they use `ctx.runQuery`/`ctx.runMutation` internally).

**Non-goals**
- Soft-cap threshold enforcement (`softCapReached: true` logic) — deferred to T17c.
- Hard-cap write (`cases.costLimited = true`) on crossing $10 — deferred to T17c.
- Any React UI for displaying cost — not part of this ticket.
- Per-model pricing granularity (Haiku vs Sonnet) — single constant for now; T17c may refine.
