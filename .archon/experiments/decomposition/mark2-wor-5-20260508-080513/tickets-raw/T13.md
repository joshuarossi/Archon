# Privacy Response Filter Module â€” `convex/lib/privacyFilter.ts`

This task implements the standalone privacy leak detector that guards both synthesis generation and the Coach AI action from inadvertently echoing one party's private messages back to the other. It lives in `convex/lib/` with no Convex runtime dependency, making it a pure TypeScript utility that can be fully exercised in Vitest without spinning up any backend. It is a prerequisite for T14 (synthesis) and T17b (Coach), which both call it before committing an AI response to the database.

## Acceptance Criteria

- `privacyFilter.ts` exports `checkPrivacyLeak(otherPartyMessages: string[], candidateResponse: string, minTokens?: number): { passed: boolean, matchedSubstrings: string[] }` with default `minTokens = 8`.
- Returns `passed: false` when any 8-token window from the other party's messages appears verbatim in the candidate response.
- Returns `passed: true` when the candidate response shares no 8-token window with the other party's messages.
- Handles edge cases: empty other-party messages (always passes), single-word messages (always passes since < 8 tokens), candidate response shorter than 8 tokens (always passes).
- Vitest unit tests: (1) exact substring match detects leak; (2) paraphrase with different word order passes; (3) boundary condition at exactly 7 tokens passes; (4) boundary at exactly 8 tokens fails; (5) empty other-party array passes.

## Implementation Notes

This module tokenizes on whitespace â€” no NLP library required. For each message in `otherPartyMessages`, slide a window of `minTokens` consecutive tokens across it; for each window, check whether that token sequence (re-joined with a single space) appears as a substring of the `candidateResponse`. If any window matches, accumulate it into `matchedSubstrings` and set `passed: false`.

The check is intentionally literal: paraphrase and semantic similarity are out of scope. Claude's own anti-quotation system prompt (injected by `assemblePrompt` for SYNTHESIS and COACH roles) handles semantic drift; this filter is the hard structural backstop for verbatim leakage.

**File placement:** `convex/lib/` per the Convex architectural convention for business logic that has no Convex dependency. This module does not import from `convex/server`, `convex/values`, or `@anthropic-ai/sdk`.

**TypeScript strictness:** `strict: true` and `noUncheckedIndexedAccess: true` are in effect. Any array indexing inside the sliding-window logic must be bounds-checked or use safe iteration patterns to satisfy `noUncheckedIndexedAccess`.

**Callers:** T14 (`convex/synthesis/generate.ts`) calls `checkPrivacyLeak` against the *other* party's raw private messages before writing `synthesisText`. T17b (`convex/jointChat/generateCoachResponse.ts`) calls it against *both* parties' private messages before committing the Coach response. Both callers retry up to 2 times on `passed: false` and fall back to generic text on the third failure.

**Tokenization detail:** split on `/\s+/` and filter empty strings so that leading/trailing whitespace in source messages does not produce phantom empty tokens that inflate the window size.

**Queries used**
None â€” this module has zero Convex, SDK, or network dependencies.

**Invariants**
- `checkPrivacyLeak([], anyString)` always returns `{ passed: true, matchedSubstrings: [] }`.
- Any message whose token count is strictly less than `minTokens` never contributes a window; it is silently skipped.
- A `candidateResponse` whose total token count is strictly less than `minTokens` always returns `{ passed: true, matchedSubstrings: [] }` (no window can match).
- `matchedSubstrings` contains exactly the token-window strings (space-joined) that matched, in the order they were found. Duplicates are allowed if the same window appears multiple times.
- `passed` is `false` if and only if `matchedSubstrings.length > 0`.
- The default value of `minTokens` is `8`; callers may override to any positive integer.

**Non-goals**
- Does not check semantic similarity or paraphrase â€” verbatim token windows only.
- Does not call Claude, Convex, or any external service.
- Does not modify any database row.
- Does not handle multilingual tokenization (whitespace split is the contract).
- Does not deduplicate `matchedSubstrings` entries.
