# Coach invocation decision â€” Haiku classification gate, @-mention detection, exchange timer

Rather than injecting the Coach AI into every user exchange, this module acts as a decision gate that determines _when_ the Coach should speak. It classifies each new message using Haiku (cheap and fast), applies an @-mention regex override, and enforces a 5-exchange periodic check â€” returning a single `{ shouldRespond, classification, mentionsCoach }` result that the full Coach action (T17b) consumes without duplicating any decision logic. This task exists in Phase 2 because it depends on joint chat messages being live (T16) and is deliberately extracted from the Sonnet-powered response action so the classification logic can be unit-tested in full isolation.

## Acceptance Criteria

- `coachShouldRespond.ts` exports `checkShouldCoachRespond({ recentMessages, latestMessage, lastCoachMessageIndex }): Promise<{ shouldRespond: boolean, classification: 'INFLAMMATORY' | 'PROGRESS' | 'QUESTION_TO_COACH' | 'NORMAL_EXCHANGE', mentionsCoach: boolean }>`.
- Haiku classification (`claude-haiku-4-5-20251001`) is called with a 10k token budget on every invocation; result maps to one of the 4 classification values.
- Returns `shouldRespond: true` when classification is INFLAMMATORY, PROGRESS, or QUESTION_TO_COACH.
- Returns `shouldRespond: true` when the latest message contains '@coach' or '@Coach' (case-insensitive), regardless of classification.
- Returns `shouldRespond: true` when 5 or more user exchanges have occurred since the last Coach message.
- Returns `shouldRespond: false` only when classification is NORMAL_EXCHANGE AND no @-mention AND fewer than 5 exchanges since last Coach message.
- Vitest unit test: given NORMAL_EXCHANGE classification, no @-mention, and 4 prior user exchanges, `shouldRespond` is false.
- Vitest unit test: given NORMAL_EXCHANGE classification, no @-mention, and exactly 5 prior user exchanges, `shouldRespond` is true.
- Vitest unit test: @-mention in message forces `shouldRespond: true` regardless of classification.

## Implementation Notes

This module is a pure decision function with no Convex DB writes and no Sonnet dependency. It lives in `convex/jointChat/coachShouldRespond.ts` and is called from within the `generateCoachResponse` action (T17b) â€” not directly from a Convex query or mutation.

**Function type:** utility called from a Convex action. No `query()` / `mutation()` wrapper is needed; it is a plain async function imported by T17b's action file.

**Haiku classification:** Per the `anthropic-sdk` stack rule, Haiku (`claude-haiku-4-5-20251001`) is the correct model for cheap classification gates â€” it is ~10Ă— cheaper than Sonnet and fast enough for per-message decisions. Call `client.messages.create()` (non-streaming) with a system prompt that instructs the model to return exactly one of the four classification strings, and a `max_tokens: 10000` budget. Parse the response text and map it to the `ClassificationValue` union; treat any unrecognized string as `NORMAL_EXCHANGE` to fail safe.

**Three independent override conditions (evaluated in order, short-circuit on first true):**
1. Classification is INFLAMMATORY, PROGRESS, or QUESTION_TO_COACH â†’ `shouldRespond: true`
2. `latestMessage` matches `/@coach/i` â†’ `shouldRespond: true`, `mentionsCoach: true`
3. `(recentMessages.length - lastCoachMessageIndex) >= 5` â†’ `shouldRespond: true`

Only if all three conditions are false does the function return `shouldRespond: false`.

**Exchange counting:** `lastCoachMessageIndex` is the index (in the `recentMessages` array) of the last Coach-authored message. Messages after that index are user exchanges. A `lastCoachMessageIndex` of `-1` (no prior Coach message) means all messages are user exchanges.

**Testing:** `vi.mock('@anthropic-ai/sdk')` at the module level to control classification responses. Do not use `convex-test` for this module â€” it has no Convex schema dependency. All nine ACs can be satisfied with pure Vitest unit tests.

**TypeScript:** Use `MessageParam[]` from `@anthropic-ai/sdk` for `recentMessages`. Strict mode must pass; no `any` casts. Define `ClassificationValue` as a union type in the same file and export it for T17b to import.

## Test-Gen Brief

### Contract

**Files**
- `convex/jointChat/coachShouldRespond.ts` â€” implementation (exports `checkShouldCoachRespond` and `ClassificationValue`)
- `convex/jointChat/coachShouldRespond.test.ts` â€” Vitest unit tests (all 9 ACs)

**Exports**
- `checkShouldCoachRespond` â€” named, async function, value export
- `ClassificationValue` â€” named, TypeScript union type export (for T17b consumer)

**Signatures**
```typescript
export type ClassificationValue =
  | 'INFLAMMATORY'
  | 'PROGRESS'
  | 'QUESTION_TO_COACH'
  | 'NORMAL_EXCHANGE';

export interface CoachShouldRespondResult {
  shouldRespond: boolean;
  classification: ClassificationValue;
  mentionsCoach: boolean;
}

export interface CoachShouldRespondArgs {
  recentMessages: MessageParam[];   // from @anthropic-ai/sdk
  latestMessage: string;
  lastCoachMessageIndex: number;    // -1 if no prior Coach message
}

export async function checkShouldCoachRespond(
  args: CoachShouldRespondArgs,
): Promise<CoachShouldRespondResult>
```

**Queries used**
- None â€” this module makes no Convex DB calls. It calls `@anthropic-ai/sdk` directly (the Anthropic client is passed in or constructed internally; T17b holds the Convex action context).

**Invariants**
- If `classification` is INFLAMMATORY, PROGRESS, or QUESTION_TO_COACH â†’ `shouldRespond` is always `true`.
- If `latestMessage` matches `/@coach/i` â†’ `shouldRespond` is always `true` and `mentionsCoach` is `true`.
- If `recentMessages.length - lastCoachMessageIndex >= 5` â†’ `shouldRespond` is always `true`.
- `shouldRespond` is `false` only when all three conditions above evaluate to false simultaneously.
- `mentionsCoach` is `false` when no `/@coach/i` match exists, regardless of `shouldRespond`.
- Unrecognized Haiku output strings are treated as `NORMAL_EXCHANGE` (fail-safe default).

**Non-goals**
- Does not generate the Coach response text (T17b owns that).
- Does not write to the `jointMessages` or any other Convex table.
- Does not check cost limits (T10d / T17c).
- Does not apply the privacy filter (T13).
- Does not handle the streaming pattern â€” this is a non-streaming Haiku call.
- Does not validate that the caller is a party to the case (T16's `sendUserMessage` mutation owns that guard).
