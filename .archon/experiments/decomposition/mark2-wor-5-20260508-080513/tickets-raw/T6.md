# Case creation form + invite token generation (US-03)

`NewCasePage.tsx` is the first meaningful action an authenticated user takes after logging in: they describe their conflict, pick a category, and submit a structured form. Phase 1 needs this page because every downstream task — private coaching, invite acceptance, dashboard display, and solo mode — depends on a case existing with a valid `partyStates` row and an invite token. The `cases/create` Convex mutation atomically creates three rows (case, partyStates for the initiator, inviteToken) in a single transaction and hands back a 32-character token the user can share. This ticket covers the form UI, the mutation, and the post-submit invite-link display.

## Acceptance Criteria

- NewCasePage form has: category select (workplace, family, personal, contractual, other), main topic text input (single line), description textarea (multi-paragraph), desired outcome text input.
- All four fields are required; submitting with any empty field shows inline validation errors and does not call the mutation.
- `cases/create` mutation creates: a `cases` row with status `DRAFT_PRIVATE_COACHING`, `isSolo: false`, `templateVersionId` set to the current version of the default template for the selected category, `initiatorUserId` set; a `partyStates` row for the initiator with the form fields populated.
- Invite token is generated using `crypto.randomUUID().replace(/-/g,'')` (two UUIDs concatenated, trimmed to 32 chars) and stored as an `inviteTokens` row with status `ACTIVE`.
- After creation, user sees the invite URL (`{SITE_URL}/invite/{token}`) with a one-click copy button and explanatory text.
- Attempting to create a case while unauthenticated returns `UNAUTHENTICATED` error.
- Playwright test: authenticated user fills the form, submits, and sees the invite link; invite URL contains a 32-character token.

## Implementation Notes

**Form layout and validation**

`NewCasePage.tsx` uses shadcn/ui `Select`, `Input`, and `Textarea` primitives composed inside a `Card`. Max-width `720px` centered (`max-w-[720px] mx-auto`). All four fields carry the `"Private to you."` label convention from the tone-and-copy rules (bold label text + explanatory sentence). Inline validation errors appear below the offending field in `var(--danger)` at 13px — not a banner, not a toast. Validation fires on submit (not on blur) to avoid premature error noise. The form does not call the Convex mutation if any field is empty; the guard lives entirely in the UI before any network request is made.

**Convex mutation design**

`cases/create` is a `mutation()` (never an `action()` — no external I/O needed). It starts with `requireAuth(ctx)` from `convex/lib/errors.ts` to resolve the caller. Template lookup: query the `templates` table for the matching category, then `templateVersions` for the latest published version — this is the value written into `cases.templateVersionId`. The three inserts (cases, partyStates, inviteTokens) happen within the same mutation and are therefore atomic. The token generation formula is exactly `crypto.randomUUID().replace(/-/g,'') + crypto.randomUUID().replace(/-/g,'')` trimmed to 32 characters — producing a 32-hex-char token. Store it in `inviteTokens` with `status: 'ACTIVE'` and the `caseId` FK.

**Post-submit invite display**

After the mutation resolves, the page transitions from the form view to an invite-link display (same route, conditional render — no navigation). The display shows the full invite URL (`${import.meta.env.VITE_SITE_URL}/invite/${token}`) in a read-only Input or code block plus a copy button (`navigator.clipboard.writeText`). The explanatory text follows the tone rules: plain language explaining the other person should click the link to participate, without legal-sounding phrasing.

**Type safety**

Use `Doc<"cases">`, `Doc<"partyStates">`, `Doc<"inviteTokens">` from Convex codegen for all DB interactions. The category field must use a `v.union(v.literal(...))` validator matching the exact five values in the select. Never write a parallel TypeScript interface — derive from the Convex schema directly.

**Copy conventions**

Form field labels follow the privacy labeling convention: the description textarea and desired-outcome input show **"Private to you."** before the helper sentence. The submit button label: "Create case" (describes the outcome). The invite display button: "Copy invite link".

## Test-Gen Brief

### Contract

**Files**
- `src/pages/NewCasePage.tsx` — route component (form + post-submit invite display)
- `convex/cases/create.ts` — `cases/create` mutation
- `convex/lib/errors.ts` — `requireAuth`, `createError` (already exists from T1)
- `convex/schema.ts` — read-only reference; no changes in this ticket
- `playwright/case-creation.spec.ts` — E2E assertions for ACs 1, 2, 5, 7

**Exports**
- `NewCasePage` — default export, React route component
- `create` — named export from `convex/cases/create.ts`, wrapped with `mutation()`; registered as `api.cases.create` via Convex codegen

**Signatures**
```typescript
// convex/cases/create.ts
export const create = mutation({
  args: {
    category: v.union(
      v.literal('workplace'),
      v.literal('family'),
      v.literal('personal'),
      v.literal('contractual'),
      v.literal('other'),
    ),
    mainTopic: v.string(),
    description: v.string(),
    desiredOutcome: v.string(),
  },
  returns: v.object({
    caseId: v.id('cases'),
    inviteToken: v.string(),
  }),
  handler: async (ctx, args): Promise<{ caseId: Id<'cases'>; inviteToken: string }> => { ... },
});
```

**Queries used**
- `ctx.db.query('templates').withIndex('by_category', q => q.eq('category', args.category)).first()` — resolve default template for category
- `ctx.db.query('templateVersions').withIndex('by_template', q => q.eq('templateId', template._id)).order('desc').first()` — resolve latest version (index TBD based on schema; use whatever index T1 defined)
- `ctx.db.insert('cases', { ... })` — create case row
- `ctx.db.insert('partyStates', { ... })` — create initiator party state
- `ctx.db.insert('inviteTokens', { ... })` — store token
- `requireAuth(ctx)` — from `convex/lib/errors.ts`

**Invariants**
- Token returned from mutation is exactly 32 alphanumeric characters (hex, no hyphens)
- `cases.status` is always `'DRAFT_PRIVATE_COACHING'` on creation via this mutation
- `cases.isSolo` is always `false` for cases created via this mutation
- `partyStates` row for the initiator is inserted in the same mutation call as the case row — never deferred
- `inviteTokens.status` is `'ACTIVE'` immediately after insertion
- `cases.templateVersionId` is never null; if no template exists for the category, the mutation should throw `INTERNAL` rather than insert a null FK

**Non-goals**
- Solo case creation (`isSolo: true`) — T12
- Email delivery of the invite link — not in v1 scope
- Form field draft/auto-save — not in scope
- Invite token expiry or rate-limiting — not in scope for this ticket
