# Synthesis generation action — both-party context, response filter, state transition

Phase 2 begins the joint session, and synthesis is the bridge: after both parties have completed private coaching, a Convex action reads both coaching transcripts, calls Claude Sonnet with the combined context, and produces personalized guidance for each party that must never echo the other's private words. This task implements `convex/synthesis/generate.ts` — the action that orchestrates cost checking, prompt assembly, privacy filtering with retry logic, fallback handling, result persistence, and the state-machine transition to `READY_FOR_JOINT`. It also locks the `jointChat/mySynthesis` query behind a per-party ownership check, delivering the second of five PRD §7.1 privacy tests.

## Acceptance Criteria

- Synthesis action is only callable when both parties have `privateCoachingCompletedAt` set; otherwise throws `CONFLICT`.
- Before calling Claude, calls `checkCostLimit(ctx, caseId)` from `convex/lib/costTracking.ts`; if `{ limited: true }`, throws `RATE_LIMITED` without calling Claude.
- The Claude call uses `assemblePrompt(role=SYNTHESIS)` which includes the verbatim anti-quotation critical instruction from tech spec §6.3.2 in the system prompt.
- Response is parsed as JSON `{ forInitiator: string, forInvitee: string }`; if JSON parse fails, the action logs the error and retries once; on second failure sets a generic fallback and flags in audit log.
- Each synthesis text is passed through `privacyFilter.checkPrivacyLeak` against the OTHER party's raw private messages; if leak detected, regenerates up to 2 times.
- If all 3 attempts fail the privacy filter, writes a generic fallback synthesis ('The system was unable to generate personalized guidance at this time.') and writes a `SYNTHESIS_PRIVACY_FALLBACK` entry to `auditLog`.
- On success: both `partyStates.synthesisText` and `synthesisGeneratedAt` are set, then `stateMachine.transitionCase` is called to move to `READY_FOR_JOINT` in the same logical step.
- After the Claude call completes (success or fallback), calls `recordSpend(ctx, caseId, tokensUsed)` from `convex/lib/costTracking.ts`.
- A query for one party's synthesis by the other party returns FORBIDDEN.
- Vitest unit test: synthesis action with mocked Claude returning a response containing 8+ token verbatim match triggers retry and eventually uses fallback.
- Playwright (`privacy.spec.ts`, PRD §7.1 launch criterion #2, Test 2 — synthesis isolation): using two browser contexts, user B directly calls the synthesis query (`jointChat/mySynthesis`) for user A's partyState in this case; asserts the response is FORBIDDEN.

## Implementation Notes

**Convex action type.** All Claude API calls must live in Convex actions (`internalAction` or `action`), never in mutations. The action uses `ctx.runQuery()` to fetch party states and private message histories, calls Claude, then uses `ctx.runMutation()` to write results and trigger the state transition. Each `ctx.runMutation()` is an independent round-trip — no cross-call atomicity, so design writes to be safe if the action retries (idempotent result writes keyed on `partyStateId`).

**Scheduling vs. direct call.** T11's `privateCoaching/markComplete` schedules this action via `ctx.scheduler.runAfter(0, internal.synthesis.generate, { caseId })`. The scheduler is not transactional: if `markComplete` later rolls back, the action may still fire. The action must re-validate that both `privateCoachingCompletedAt` values are set before proceeding; if not, it should exit silently (idempotent guard).

**Non-streaming call.** Unlike private coaching and Coach, synthesis uses a single non-streaming call because it produces structured JSON. Use `client.messages.create()` with `response_format: { type: "json_object" }` per the Anthropic SDK. Capture `usage.input_tokens + usage.output_tokens` from the response for `recordSpend`.

**Retry budget.** The task has a 3-attempt budget that covers both the privacy-filter retry loop and the JSON parse retry:
- Attempt 1: call Claude, parse JSON, run privacy filter on both texts.
- If JSON parse fails: retry the Claude call once (attempt 2).
- If either text fails the privacy filter: retry the full generation (up to 2 additional attempts, so attempts 2 and 3).
- After all attempts exhausted (either by JSON failure on both tries or privacy filter failure on all 3): use the generic fallback text and write `SYNTHESIS_PRIVACY_FALLBACK` to `auditLog`.

**Privacy filter context.** `privacyFilter.checkPrivacyLeak` takes `otherPartyMessages: string[]`. For each party's synthesis text, "other party" means the OPPOSITE party's raw `privateMessages` content strings. Fetch both parties' private message rows before the first Claude call so the check doesn't require additional round-trips inside the retry loop.

**`mySynthesis` query ownership.** The query `convex/jointChat/mySynthesis.ts` (or co-located in `convex/synthesis/`) must call `requireAuth(ctx)`, resolve the caller's `partyStates` row for the requested case, and throw `FORBIDDEN` via `createError('FORBIDDEN', ...)` if the caller is not the owner of that row. Never return the other party's `synthesisText` under any code path.

**State transition.** `stateMachine.transitionCase(ctx, caseId, 'READY_FOR_JOINT', options)` must be called within the same `ctx.runMutation()` that writes `synthesisText` and `synthesisGeneratedAt` to both `partyStates` rows — or in an immediately following mutation if a single mutation writing both rows plus the case status is too large. Either way, the state must not transition before both texts are persisted.

**Error code flow.** The action should bubble `RATE_LIMITED` (from `checkCostLimit`) and `CONFLICT` (from the precondition check) as `ConvexError` with the matching code from `convex/lib/errors.ts`. These propagate to the scheduler caller and are visible in the Convex dashboard logs.

## Test-Gen Brief

### Contract

**Files**
- Implementation: `convex/synthesis/generate.ts`
- Synthesis query: `convex/jointChat/mySynthesis.ts` (or `convex/synthesis/mySynthesis.ts`)
- Dependencies (read, not modified): `convex/lib/prompts.ts`, `convex/lib/costTracking.ts`, `convex/lib/privacyFilter.ts`, `convex/stateMachine.ts`, `convex/lib/errors.ts`, `convex/schema.ts`
- Vitest unit test: `convex/synthesis/generate.test.ts`
- Playwright E2E: `playwright/privacy.spec.ts`

**Exports**
- `generate` — named Convex internal action export (`internalAction`), scheduled by `privateCoaching/markComplete`
- `mySynthesis` — named Convex query export (`query`), called from the React client

**Signatures**
```typescript
// convex/synthesis/generate.ts
export const generate = internalAction({
  args: { caseId: v.id('cases') },
  handler: async (ctx: ActionCtx, args: { caseId: Id<'cases'> }): Promise<void> => { ... },
});

// convex/jointChat/mySynthesis.ts (or convex/synthesis/mySynthesis.ts)
export const mySynthesis = query({
  args: { caseId: v.id('cases') },
  returns: v.union(v.null(), v.string()),
  handler: async (ctx: QueryCtx, args: { caseId: Id<'cases'> }): Promise<string | null> => { ... },
});
```

**Queries used**
- `ctx.runQuery(internal.synthesis.getPartyStatesAndMessages, { caseId })` — TBD name; fetches both `partyStates` rows + their raw `privateMessages` content arrays for the case
- `ctx.runMutation(internal.synthesis.writeSynthesisResults, { caseId, initiatorText, inviteeText })` — persists both `synthesisText` + `synthesisGeneratedAt`, then calls `transitionCase`
- `ctx.runMutation(internal.auditLog.write, { action: 'SYNTHESIS_PRIVACY_FALLBACK', targetType: 'cases', targetId: caseId })` — on full fallback path
- `checkCostLimit(ctx, caseId)` from `convex/lib/costTracking.ts`
- `recordSpend(ctx, caseId, tokensUsed)` from `convex/lib/costTracking.ts`
- `assemblePrompt({ role: 'SYNTHESIS', caseId, actingUserId, recentHistory })` from `convex/lib/prompts.ts`
- `checkPrivacyLeak(otherPartyMessages, candidateResponse)` from `convex/lib/privacyFilter.ts`

**Invariants**
- `generate` never calls the Anthropic SDK when `checkCostLimit` returns `{ limited: true }`.
- `generate` never sets `partyStates.synthesisText` to any string that failed `checkPrivacyLeak` against the other party's messages.
- `stateMachine.transitionCase` to `READY_FOR_JOINT` is only called after both `synthesisText` values are written.
- `mySynthesis` never returns the calling user's synthesis if the caller is not the owner of the `partyStates` row for that case.
- `recordSpend` is called exactly once per `generate` invocation (success or fallback), using the token count from the final successful Claude call (or 0 if Claude was never called due to hard cap or both-parties guard).

**Non-goals**
- This ticket does NOT build `ReadyForJointView` (that is T15).
- This ticket does NOT implement per-party streaming for synthesis (it is a single non-streaming JSON call).
- This ticket does NOT add new fields to `convex/schema.ts` (all fields — `synthesisText`, `synthesisGeneratedAt` on `partyStates`; `costUsd`, `costTokens`, `costLimited` on `cases`; `auditLog` table — are already defined in T1).
- This ticket does NOT modify the cost enforcement threshold logic (that is T17c; T10d's stub is in effect during Phase 2 until T17c lands).
