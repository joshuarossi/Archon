# Transcript compression utility — convex/lib/compression.ts

`compression.ts` is the module that keeps private coaching sessions within Claude's context window. As conversations grow, the accumulated `MessageParam[]` array can exceed the Sonnet input limit; this utility detects that condition and calls Haiku to summarize the oldest half before the array is handed to `assemblePrompt`. Phase 1 introduces this module because `T10b` calls `compressIfNeeded` before every AI invocation — it must exist and be correct before private coaching works under extended use. The module lives in `convex/lib/` alongside other pure helpers (prompts, privacy filter) that have no Convex dependency and are fully unit-testable in isolation.

## Acceptance Criteria

- `compression.ts` exports `compressIfNeeded(messages: MessageParam[], tokenBudget: number): Promise<MessageParam[]>`; returns the original array if total tokens are under budget.
- When token count exceeds 60k, the oldest 50% of messages are summarized using `claude-haiku-4-5-20251001` with the compression prompt from tech spec §6.4.
- The summary replaces the compressed messages with a single `{ role: 'user', content: '[Summary of earlier conversation]: ...' }` entry.
- Compression results are cached by a SHA-256 hash of the input messages; calling with the same messages twice makes only one Haiku API call.
- Vitest unit test: given a message array with a total token count exceeding the budget, `compressIfNeeded` returns a shorter array with a leading summary message.
- Vitest unit test: calling `compressIfNeeded` twice with identical input makes exactly one external API call (cache hit on second call).
- Vitest unit test: given a message array within budget, the original array is returned unchanged.

## Implementation Notes

`compressIfNeeded` is a standalone async function — no Convex imports, no `ctx`, no DB. It is called directly from the `generateAIResponse` action in `T10b` before the message array reaches `assemblePrompt`. Because it sits outside the Convex function model, it can import `@anthropic-ai/sdk` and use `node:crypto` directly.

**Token counting:** The function must estimate the total token budget of the input array. Per the `anthropic-sdk` stack rule, the SDK does not expose a synchronous token counter — use a character-based heuristic (e.g., `Math.ceil(totalChars / 4)`) or count by splitting on whitespace. The important invariant is that the heuristic is consistent: the same array always produces the same count, so caching by content hash is sound.

**Compression split:** When over budget, take the first `Math.floor(messages.length / 2)` messages (the oldest), call Haiku with the compression prompt from tech spec §6.4, and replace them with the summary entry. The remaining (newer) half is preserved verbatim and appended after the summary.

**Cache:** Use a `Map<string, MessageParam[]>` module-level singleton. The key is the hex-encoded SHA-256 digest of `JSON.stringify(messages)`. On a cache hit, return the cached result without calling Haiku. The cache is process-scoped and does not need to survive restarts; this is a pure performance optimisation.

**Haiku model constant:** This task creates `convex/lib/models.ts` (it does not yet exist — T10a has no model constants, and T10b depends on T10c so cannot create it first). Export `HAIKU` as the constant:

```typescript
// convex/lib/models.ts
export const HAIKU = 'claude-haiku-4-5-20251001';
// SONNET will be added by T10b (which depends on this task)
```

Import `HAIKU` from `convex/lib/models.ts` in `compression.ts` — do not inline the string. The `anthropic-sdk` stack rule requires model strings to be defined as constants in one file and imported everywhere.

**Cross-ticket note (T10b):** T10b's Contract declares `export const HAIKU: string` in `models.ts`. Use the name `HAIKU` (not `HAIKU_MODEL`) so T10b can import it without a rename. T10b will extend the file with `SONNET`; no conflicts arise because T10b depends on T10c.

**Error handling:** If the Haiku call fails, let the error propagate — the calling action (`T10b`) handles all Claude API errors uniformly. Do not swallow or wrap errors inside `compressIfNeeded`.

**TypeScript:** All parameters and the return type must use `MessageParam[]` from `@anthropic-ai/sdk` directly, per the project rule to never duplicate SDK types. No `any` casts.

## Test-Gen Brief

### Contract

**Files**
- Implementation: `convex/lib/compression.ts`
- Model constants (created by this task): `convex/lib/models.ts`
- Tests: `convex/lib/compression.test.ts`

**Exports**

| Symbol | Kind | Shape |
|--------|------|-------|
| `compressIfNeeded` | named export, async function | `(messages: MessageParam[], tokenBudget: number) => Promise<MessageParam[]>` |
| `HAIKU` | named export, string constant (`convex/lib/models.ts`) | `'claude-haiku-4-5-20251001'` |

**Signatures**

```typescript
// convex/lib/compression.ts
import type { MessageParam } from '@anthropic-ai/sdk';

export async function compressIfNeeded(
  messages: MessageParam[],
  tokenBudget: number,
): Promise<MessageParam[]>;

// convex/lib/models.ts
export const HAIKU: string;  // 'claude-haiku-4-5-20251001' — Haiku tier for transcript compression and classification
// Note: SONNET will be added by T10b
```

**Queries used**

- Anthropic SDK: `client.messages.create({ model: HAIKU, ... })` — called only when total estimated token count exceeds `tokenBudget`. Not a Convex query; called directly from the module.

**Invariants**

1. If the estimated token count of `messages` is ≤ `tokenBudget`, the function returns the original array (no Haiku call, no mutation of content).
2. When compression occurs, the returned array is strictly shorter than the input array.
3. The first element of any compressed result always has `role: 'user'` and `content` starting with `'[Summary of earlier conversation]: '`.
4. Identical `messages` inputs (by JSON serialization) never trigger more than one Haiku API call across the lifetime of the module (cache guarantee).
5. The second half of the message array (the newer messages) is appended after the summary entry, preserving their exact content and order.

**Non-goals**

- Does not interact with Convex DB or `ctx` in any form.
- Does not enforce or validate the `tokenBudget` value — callers pass 60000 by convention; the function respects whatever value is given.
- Does not handle Haiku API errors — propagates them to the caller.
- Does not recursively compress (i.e., does not re-check budget after compression; one pass only).
- Does not modify the `convex/schema.ts` or any Convex table.
- Does not touch the compression prompt text itself — that is defined in tech spec §6.4 and hardcoded in this module.
